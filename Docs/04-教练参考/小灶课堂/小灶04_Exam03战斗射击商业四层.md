## 一、考试版 vs 商业版 · 核心差距

| 问题 | 考试极简（你 Exam03 · 能得分） | 商业上线（射击/塔防/MOBA 通用） |
|------|--------------------------------|----------------------------------|
| 刷怪 | `GameController` Start 里 for+Random+Instantiate | **SpawnDirector** · 波次表 SO · 区域/NavMesh 采样 |
| 开火 | 按钮 → Instantiate 子弹 → AddForce → Destroy | **WeaponSystem** → **BulletPool.Get** → 还池 |
| 命中 | `EnemyController.OnTriggerEnter` 判 Bullet Tag | **子弹** 上 `HitProcessor` · 或 **统一 HitResolver** |
| 加血 | 打敌人 **给自己** +10 · `FindObjectOfType<PlayerHealth>` | **HealSystem.ApplyHeal(target, amount)** · 逻辑不知 UI |
| UI | `PlayerHealth` 直接改 Slider/Text/Panel | **BattleHudView** 只监听 **OnHpChanged / OnVictory** |
| 移动 | `TrankController` Update 读 Input | **PlayerMotor** + InputSystem · 与 **CameraRig** 解耦 |
| 相机 | `CameraFollow` 手写 offset+Lerp (**L2 ✅**) | **Cinemachine** VCam + Collision Extension (**L3/M1**) |
| 敌人 | 击中可选 Destroy 敌人 | **不销毁** · 无敌帧/重复击中 · 或 **对象池敌人** |
| 稳定 | 能 Play 闭环 | 池上限 · 事件退订 · **不在 Update GetComponent** |

**阅卷 10 分**：命名空间 + 注释 + Trigger 闭环 + 胜利判定 → **不必 W2 写对象池**；**面试要能讲「若上线会怎么拆」**。


**API**：`[Serializable]` · 可进 **ScriptableObject** 给策划改数值而不改代码。

## 四、逻辑层 · 核心四件（不碰 UI）

### 4.1 WaveSpawner（≈ 你的 GameController）

```csharp
/// <summary>只负责按 SpawnData 刷怪，不写 Slider。</summary>
public class WaveSpawner : MonoBehaviour
{
    [SerializeField] private SpawnData spawn;
    [SerializeField] private GameObject enemyPrefab;

    void Start()
    {
        for (int i = 0; i < spawn.enemyCount; i++)
        {
            float x = Random.Range(-spawn.rangeX, spawn.rangeX);
            float z = Random.Range(-spawn.rangeZ, spawn.rangeZ);
            Instantiate(enemyPrefab, new Vector3(x, spawn.spawnY, z), Quaternion.identity);
        }
    }
}
```

### 4.2 WeaponSystem + BulletPool（≈ FireController 商业版）

```csharp
/// <summary>开火只借子弹、设速度；不 Destroy。</summary>
public class WeaponSystem : MonoBehaviour
{
    [SerializeField] private BulletPool pool;
    [SerializeField] private WeaponData weapon;
    [SerializeField] private Transform firePoint;

    public void Fire()
    {
        var go = pool.Get(firePoint.position, firePoint.rotation);
        var rb = go.GetComponent<Rigidbody>();
        if (rb) rb.velocity = firePoint.forward * weapon.bulletSpeed;
    }
}
```

### 4.3 HitProcessor（建议挂 **子弹** · 你挂在敌人也能 L1）

```csharp
/// <summary>命中敌人 → 走 HealSystem，还子弹到池。</summary>
public class HitProcessor : MonoBehaviour
{
    [SerializeField] private WeaponData weapon;

    void OnTriggerEnter(Collider other)
    {
        if (!other.CompareTag("Enemy")) return;
        HealSystem.ApplyToPlayer(weapon.healOnEnemyHit);
        GetComponent<PooledBullet>()?.Return();
    }
}
```

### 4.4 HealthComponent + HealSystem（≈ PlayerHealth 商业版）

```csharp
public static class HealSystem
{
    static HealthComponent _player;

    public static void RegisterPlayer(HealthComponent hp) => _player = hp;

    public static void ApplyToPlayer(int amount)
    {
        _player?.Heal(amount);
    }
}

public class HealthComponent : MonoBehaviour
{
    public HealthData data;
    public event Action<int,int> OnHpChanged;
    public event Action OnVictory;

    int _cur;
    void Awake() { _cur = data.initHp; HealSystem.RegisterPlayer(this); }

    public void Heal(int v)
    {
        _cur = Mathf.Min(_cur + v, data.maxHp);
        OnHpChanged?.Invoke(_cur, data.maxHp);
        if (_cur >= data.maxHp) OnVictory?.Invoke();
    }
}
```

**与你的 `PlayerHealth` 映射**：`AddHealth` ≈ `Heal` · 直接改 Slider ≈ 缺 **事件 → View**。

## 五、表现层 · BattleHudView（纯 UI）

```csharp
public class BattleHudView : MonoBehaviour
{
    [SerializeField] HealthComponent target;
    [SerializeField] Slider slider;
    [SerializeField] Text label;
    [SerializeField] GameObject victoryPanel;

    void OnEnable()
    {
        target.OnHpChanged += Refresh;
        target.OnVictory += ShowWin;
    }
    void OnDisable()
    {
        target.OnHpChanged -= Refresh;
        target.OnVictory -= ShowWin;
    }
    void Refresh(int cur, int max)
    {
        slider.maxValue = max;
        slider.value = cur;  // §11.20：只 value
        label.text = $"主角血量  {cur}";
    }
    void ShowWin() => victoryPanel.SetActive(true);
}
```

## 十一、自检题（简讲师课后）

1. 为什么商业更愿意把 `OnTriggerEnter` 写在 **子弹** 而不是 **敌人**？  
2. Exam03「打敌人给自己加血」若要做成 **Healer 技能**，改 **数据层** 还是 **改 UI**？  
3. 对象池 `Return` 时为什么要 **`rb.velocity = Vector3.zero`**？  
4. 你的 `CameraFollow` 为什么用 **LateUpdate** 而不是 **Update**？  
5. 只加 **一个** 商业特性且不超 P-L1，你选 **BulletPool** 还是 **BattleHudView 事件拆层**？为什么？

## §11-A · 自检标准答案

1. **Trigger 在子弹** → 一颗子弹只处理一次命中逻辑 · 敌人不用关心子弹类型 · 新增穿甲/爆炸只改子弹组件。  
2. **改数据层** `WeaponData.healOnEnemyHit` 或 **IHitPolicy** · UI 不动。  
3. **池复用** 上一帧速度残留会导致 **还池再取出瞬飞** · 必须清零。  
4. **坦克 Update 先动** · 相机 **LateUpdate** 读 **本帧最终 Transform** · 减抖（L2 标准答法）。  
5. **W2 交卷优先 BulletPool 或补 Inspector 引用** · **HudView 事件拆层** 留阶段 02（改动面大）· 若 Console 已稳、想练 **M2 性能** 选 **池**。
