## 一、考试版 vs 商业版 · 核心差距（豆包要点 · 教练校准）

| 问题 | 考试极简写法（能得分） | 商业上线写法（MOBA/手游通用） |
|------|------------------------|-------------------------------|
| 数值 | `const 100` / Awake 赋值 | **HealthData** 序列化 · 策划 Inspector 配 |
| 结构 | 一个 Mono：算血 + Slider + 按钮 | **数据 / 逻辑 / 表现 / 伤害入口** 四层 |
| UI | 脚本直接 `hpSlider.value` | **HpView 只监听事件** · 逻辑不知 Slider 存在 |
| 伤害 | 按钮 → `TakeDamage(20)` | 按钮 → **DamageSystem.SendDamage** → HealthComponent |
| 扩展 | 无 | **OnHpChanged / OnTakeDamage / OnDeath** 挂音效/动画/飘字/任务 |
| 稳定 | Clamp 有则加分 | **_isDead 锁** · **无敌帧** · 事件 OnDisable 退订 |
| 多角色 | 复制整脚本 | 同一 **HealthComponent** · 换 HealthData |
| 联机 | 不要求 | HealthData 可序列化 · 服务端校验 |

**阅卷 10 分代码规范**：分层注释 + 序列化 + 边界 + 事件退订 → **比单脚本多写 30 行注释** 往往比硬套四层更能拿分；**整包四层不是 W2 硬性交付**。

## 二、商业四层架构（王者荣耀 / MOBA 同款思想）

```
[ 表现层 ]  HpView / GameOverView     ← 只监听事件，改 Slider/Text/Panel
      ↑ OnHpChanged / OnDeath
[ 逻辑层 ]  HealthComponent           ← 受伤/回血/死亡/无敌，不引用 UI
      ↑ 读写
[ 数据层 ]  HealthData (struct)       ← 纯数值，无 MonoBehaviour
      ↑
[ 辅助层 ]  DamageSystem (static)     ← 全游戏伤害统一入口（技能/普攻/Buff）
      ↑
[ 输入层 ]  DamageTestBtn / 技能系统  ← 只调 DamageSystem，不摸 Slider
```

**设计模式对应（面试能讲）**

| 模式 | 在本模块里是什么 |
|------|------------------|
| **观察者** | `OnHpChanged` / `OnDeath` · HpView 订阅 |
| **单一职责** | HealthComponent 不算 UI · HpView 不算伤害 |
| **数据驱动** | HealthData 给策划配表 |
| **门面 Facade** | DamageSystem 统一入口 |
| **状态锁** | `_isDead` 防重复死亡 |

## 三、数据层 · HealthData（无 Unity 依赖）

```csharp
using System;
using UnityEngine;

/// <summary>
/// 纯血量配置：可序列化 · 可进 ScriptableObject · 可网络传 struct。
/// 禁止：在这里写 Slider、Button、TakeDamage。
/// </summary>
[Serializable]
public struct HealthData
{
    public int maxHp;           // 策划：英雄 100 / 小兵 30 / BOSS 5000
    public int initHp;          // 开局血量，通常 = maxHp
    [Range(0f, 1f)] public float damageReduce; // 减伤 0~1
    public float invincibleTime; // 受伤后无敌秒数
}
```

**API**：`[Serializable]` · `[Range]` · `struct` 值类型适合小包同步（阶段 04+）。

## 四、逻辑层 · HealthComponent（角色实体挂载）

```csharp
using UnityEngine;
using System;

/// <summary>
/// 职责：算伤害、Clamp、无敌、死亡判定；只抛事件，不碰 UI。
/// </summary>
public class HealthComponent : MonoBehaviour
{
  public HealthData healthData;

  private int _currentHp;
  private float _invincibleTimer;
  private bool _isDead; // 死亡锁：行业防重复 Die()

  public event Action<int, int> OnHpChanged;   // (current, max)
  public event Action OnDeath;
  public event Action<int, int> OnTakeDamage; // (finalDamage, currentAfter)

  private void Awake()
  {
    _currentHp = healthData.initHp;
    _isDead = false;
  }

  private void Update()
  {
    if (_invincibleTimer > 0) _invincibleTimer -= Time.deltaTime;
  }

  /// <summary>行业统一受伤入口；外部禁止直接改 _currentHp。</summary>
  public void TakeDamage(int rawDamage)
  {
    if (_isDead || _invincibleTimer > 0) return;

    int final = Mathf.RoundToInt(rawDamage * (1f - healthData.damageReduce));
    final = Mathf.Max(final, 1);

    _currentHp = Mathf.Max(_currentHp - final, 0); // ② 数据边界
    _invincibleTimer = healthData.invincibleTime;

    OnTakeDamage?.Invoke(final, _currentHp);
    OnHpChanged?.Invoke(_currentHp, healthData.maxHp);

    if (_currentHp <= 0 && !_isDead) Die(); // ⑤ 状态锁
  }

  public void Heal(int value)
  {
    if (_isDead) return;
    _currentHp = Mathf.Min(_currentHp + value, healthData.maxHp);
    OnHpChanged?.Invoke(_currentHp, healthData.maxHp);
  }

  private void Die()
  {
    _isDead = true;
    OnDeath?.Invoke();
  }

  public int GetCurrentHp() => _currentHp;
  public bool IsDead() => _isDead;
}
```

**与 Exam05 BattleController 映射**：你的 `TakeDamage` + `isGameOver` ≈ 逻辑层 **子集**（缺事件、无敌、Heal）。

## 五、表现层 · HpView（纯 UI）

```csharp
using UnityEngine;
using UnityEngine.UI;

/// <summary>
/// 只监听 HealthComponent 事件；换皮、换 Slider 样式不动逻辑层。
/// </summary>
public class HpView : MonoBehaviour
{
  [SerializeField] private HealthComponent target;
  [SerializeField] private Slider hpSlider;
  [SerializeField] private Text hpText;

  private void OnEnable()
  {
    target.OnHpChanged += RefreshHpUI;  // ⑥ 成对订阅
    target.OnDeath += OnUnitDeath;
    RefreshHpUI(target.GetCurrentHp(), target.healthData.maxHp); // 初刷
  }

  private void OnDisable()
  {
    target.OnHpChanged -= RefreshHpUI;  // ⑥ 必退订 · 防泄漏
    target.OnDeath -= OnUnitDeath;
  }

  private void RefreshHpUI(int cur, int max)
  {
    hpSlider.maxValue = max;
    hpSlider.value = cur; // ③ 只 value · §11.20
    hpText.text = $"{cur}/{max}";
  }

  private void OnUnitDeath()
  {
    // 商业：藏条或弹结算；Exam05：还可交给 GameOverPanel 另一脚本
    gameObject.SetActive(false);
  }
}
```

**API**：`Action` 事件 · `OnEnable/OnDisable` 退订 · `Slider.maxValue/value`。

## 六、辅助层 · DamageSystem + 按钮（统一伤害入口）

```csharp
/// <summary>
/// 全游戏伤害门面：后期加暴击/护盾/统计只改这里。
/// </summary>
public static class DamageSystem
{
  public static void SendDamage(HealthComponent target, int rawDamage)
  {
    if (target == null) return;
    target.TakeDamage(rawDamage);
  }
}

public class DamageTestBtn : MonoBehaviour
{
  [SerializeField] private HealthComponent playerHealth;

  public void Dmg20() => DamageSystem.SendDamage(playerHealth, 20);
  public void Dmg30() => DamageSystem.SendDamage(playerHealth, 30);
  public void Dmg40() => DamageSystem.SendDamage(playerHealth, 40);
  public void Dmg50() => DamageSystem.SendDamage(playerHealth, 50);
}
```

**Unity 按钮接线**：Inspector OnClick → `Dmg20()` · **不要** OnClick 里拖 Slider。

## 七、Exam05 交卷版 · 与四层的「最小映射」（你本周敲这个）

| 商业层 | Exam05 P-L1 等价 | 你仓库现状 |
|--------|------------------|------------|
| 数据 | `MaxHp` 常量或字段 | Awake `maxHp=100` |
| 逻辑 | `BattleController.TakeDamage` | ✅ 有 Clamp、门闩 |
| 表现 | `RefreshHpUI` + Slider | ✅ value · 可抽方法 |
| 伤害入口 | 四钮 → 命名方法 → TakeDamage | ⚠️ lambda |
| 事件 | 不要求 | 无（用直接调用） |
| GameOver | `EnterGameOver` + Panel | ✅ 内联可抽 |

**P-L1 不必写 HealthData 四文件**；**小灶要求你能说清「若上商业会拆成哪四层」**。

## §11-A · 自检标准答案（2026-06-24 · lixiaotong）

1. **为何不让 TakeDamage 里直接改 Slider？**  
   → 逻辑与表现耦合 · 换皮/多 UI/联机回放都要改战斗脚本 · 商业用 **event → HpView** 只刷 UI。

2. **RemoveAllListeners vs `-=` 事件？**  
   → UI 按钮用 **RemoveListener(命名方法)**（E1）· **C# event** 用 **`-=` 对称退订**（HpView OnDisable）· **RemoveAllListeners 误清他人监听**。

3. **Exam06 只加一项商业特性不超 P-L1，选哪个？**  
   → **EnterGameOver() 抽方法** + **isGameOver 门闩**（你已在 TakeDamage 里）· **HealthData SO** 留阶段 02 · W2 交卷优先 **可读+对称解绑+Clamp**。
