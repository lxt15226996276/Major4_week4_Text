### 6.2 F05-BATTLE · BattleController（Read 仓库 2026-06-24）

**行业依据**：Game Developer UI 架构（模态挡射线 · 过渡 blocking）· Unity MVVM/数据绑定方向 · Unity Discussions 监听 · §11.20 血条

| 维 | 试卷 L1 | **行业** | **lixiaotong 现状**（`BattleController.cs`） | **B 建议**（P-L1+1 · 本步可选） |
|:--:|---------|----------|---------------------------------------------|----------------------------------|
| ① 职责 | 一个脚本能扣血 | 战斗逻辑 + **RefreshUI()** 分离 · 结束 **EnterGameOver()** 单入口 | TakeDamage 内联 Panel/Slider/Text | 抽 `RefreshUI()` · `EnterGameOver()` |
| ② 数据 | currentHp 能减 | **一处改 HP** + Clamp · UI 只读数据 | ✅ Clamp 在 TakeDamage · 先减后 Clamp 可合并为一行 | `currentHp = Clamp(currentHp-dmg,0,max)` |
| ③ UI 表现 | Slider 联动 | **只 value** · Fill@0 Active · 文字格式统一 | ✅ value · 文字 `"生命值  "` 双空格 | 统一 `"生命值：" + currentHp` |
| ④ 输入 | 四钮能点 | 门闩 + **可选 interactable=false** · GameOver 时禁攻击钮 | ✅ isGameOver 在 TakeDamage  return | 结束态四钮 `interactable=false` |
| ⑤ 状态 | Panel + Log | bool/enum · **单入口** 进 GameOver · 不 Hide Fill | ✅ bool · Panel SetActive 在 TakeDamage | 抽 `EnterGameOver()` |
| ⑥ 生命周期 | AddListener 能玩 | 命名方法 · OnEnable/Disable · **禁 RemoveAllListeners** | ⚠️ **lambda** · **RemoveAllListeners** | 见 §6.2 对照代码 |
| ⑦ 性能 | 无要求 | 血条 **分 Canvas**（§6.3） | 与背景同 Canvas | 见 F05-BUI |
| ⑧ 可维护 | 能跑 | §11.9 五层 · namespace · 无魔法数 | ⚠️ 缺类顶/分区/多数方法思路 | 对齐 §11.19 范本 |

**B 对照代码（⑥ 生命周期 · 行业）**

```csharp
private void OnEnable() {
    buttonAttack20.onClick.AddListener(OnAttack20);
    // … 30/40/50
}
private void OnDisable() {
    buttonAttack20.onClick.RemoveListener(OnAttack20);
    // … 禁止 RemoveAllListeners
}
private void EnterGameOver() {
    if (isGameOver) return;
    isGameOver = true;
    SetAttackButtonsInteractable(false);
    gameOverPanel.SetActive(true);
    Debug.Log("游戏结束");
}
```
