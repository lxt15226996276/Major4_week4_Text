## 第一周

### 第一单元：画布和基础布局

**1. 什么是游戏 UI？**
- 原答：游戏中玩家与系统交互的视觉界面，如菜单、血条、按钮等。
- 精简：玩家与系统交互的界面，如菜单、血条、按钮。

**2. 游戏 UI 和普通应用 UI 的核心区别？**
- 原答：游戏 UI 需贴合游戏风格，动态效果更强，适配多输入设备，性能要求更高。
- 精简：贴合游戏风格，动效强，适配多输入，性能要求高。

**3. Canvas 的主要作用？**
- 原答：是所有 UI 元素的容器，所有 UI 组件必须依附于 Canvas 存在。
- 精简：UI 容器，所有 UI 须挂在 Canvas 下。

**4. Canvas 有哪几种渲染模式？**
- 原答：屏幕覆盖模式、屏幕相机模式、世界空间模式；
- 精简：Screen Space-Overlay / Camera、World Space。

**5. Rect Transform 的特点？**
- 原答：专为 UI 设计，支持锚点和轴心，可随父级自动调整大小和位置。
- 精简：UI 专用，支持锚点/轴心，随父级自适应大小位置。

**6. 普通 Transform 和 Rect Transform 的适用对象？**
- 原答：普通 Transform 用于 3D/2D 对象；Rect Transform 用于 UI 元素。
- 精简：Transform→3D/2D 对象；RectTransform→UI。

**7. 实现 UI 屏幕自适应的关键组件？**
- 原答：Canvas Scaler 组件是实现 UI 屏幕自适应的核心。
- 精简：Canvas Scaler 是屏幕自适应核心。

**8. 锚点（Anchors）的作用？**
- 原答：决定 UI 元素与父级边缘的连接方式，影响元素随父级变化的调整方式。
- 精简：决定 UI 与父级边缘的连接及缩放方式。

**9. Pivot（轴心）对 UI 元素的影响？**
- 原答：是元素旋转、缩放的中心点，也影响位置计算。
- 精简：旋转/缩放中心，影响位置计算。

**10. Text 控件的核心属性？**
- 原答：包括字体、字号、颜色、对齐方式、文本内容等。
- 精简：字体、字号、颜色、对齐、文本等。

**11. EventSystem 的主要功能？**
- 原答：负责检测和处理用户输入，如点击、滑动等交互操作。
- 精简：检测处理点击、滑动等输入。

**12. “Scale With Screen Size” 模式的参考是什么？**
- 原答：按设定的参考分辨率缩放 UI，保持元素比例。
- 精简：按参考分辨率缩放，保持比例。

**13. 为什么要设置合理的 UI 锚点？**
- 原答：确保 UI 元素在屏幕尺寸变化时能正确显示在预期位置。
- 精简：屏幕变化时 UI 仍显示在正确位置。

- 精简：模板测试，只显示遮罩内内容。

**36. Slider 的 “On Value Changed” 事件触发时机？**

- 精简：Value 变化时（拖动或代码改）。

**37. Scroll Rect 的 “Movement Type” 有哪些？**

- 精简：Unrestricted / Elastic / Clamped。

**38. Mask 组件需要配合什么使用？**
- 原答：通常需要配合 Image 等具有图形的组件使用。
- 精简：需配合 Image 等图形组件。

**39. Scrollbar 的 “Size” 属性作用？**
- 原答：控制滚动条滑块的大小，反映内容占可视区域的比例。
- 精简：控制滑块大小，反映内容/可视区比例。

**40. Slider 的 “Direction” 属性作用？**
- 原答：控制滑块的滑动方向。
- 精简：控制滑动方向。

**41. RectMask2d 的优势？**
