## 【全文总述】

**Canvas（画布）** 是 UGUI 的「舞台」：所有 UI 控件都必须放在 Canvas 之下，否则无法显示。  
**RectTransform（矩形变换）** 是 UI 控件的「位置管理器」：用锚点（Anchor）和偏移量（Offset）来控制 UI 在屏幕上的位置和大小，与普通 Transform 完全不同。  
**Layout Groups（布局组）** 是 UGUI 的「自动排版引擎」：一行代码不用写，就能让按钮自动排整齐、列表自动滚动、面板自适应大小。  
本文从画布创建到 RectTransform 原理，再到布局组和适配方案，逐层展开。

# 第一阶段：零基础篇

## 【阶段总述】

零基础阶段建立**正确认知**：Canvas 是什么、为什么所有 UI 都要在 Canvas 下、三种渲染模式有什么区别。  
同时学会**读懂** `GetComponent<Canvas>()` 和 `rectTransform.anchoredPosition`，并认识 Canvas 三件套。

---

## 一、定义 — Canvas 是什么？

| 项目 | 说明 |
|------|------|
| **类型** | `UnityEngine.UI` 命名空间下的 **组件（Component）** |
| **挂载** | 添加在 GameObject 上（自动创建的 Canvas 已带） |
| **官方定义** | The Canvas is the area that all UI elements should be inside |
| **一句话** | UI 元素的「舞台」，所有按钮、文字、图片都必须放在它下面才能显示 |

```csharp
// 获取 Canvas 组件
Canvas canvas = GetComponent<Canvas>();

// 三种渲染模式
canvas.renderMode = RenderMode.ScreenSpaceOverlay;    // 覆盖屏幕
canvas.renderMode = RenderMode.ScreenSpaceCamera;     // 相机前方
canvas.renderMode = RenderMode.WorldSpace;            // 3D 世界中
```

---

## 二、本质 — 画布如何「画」UI？

```
UI 元素（Text/Image/Button）
       │
       ▼
  Canvas 组件（收集所有子 UI）
       │
       ▼
  Canvas 渲染管线（合并绘制）
       │
       ▼
  屏幕 或 RenderTexture（最终显示）
```

| 概念 | 说明 |
|------|------|
| **Canvas 三件套** | Canvas（渲染）+ CanvasScaler（适配）+ GraphicRaycaster（点击检测） |
| **自动批处理** | Unity 会自动合并同一 Canvas 下的 UI 元素，减少 Draw Call |
| **EventSystem** | 创建 Canvas 时自动附带，处理 UI 输入事件（点击、拖拽等） |

---

## 三、三种 Render Mode — UI 怎么显示？

| 对比 | Screen Space - Overlay | Screen Space - Camera | World Space |
|------|------------------------|-----------------------|-------------|
| **视觉效果** | UI 直接覆盖屏幕，永远在最上层 | UI 在指定 Camera 前方，可被 3D 物体遮挡 | UI 作为 3D 物体存在，可被相机远近缩放 |
| **Inspector** | 不需要 Camera | 需指定 Target Camera | 像普通 3D 物体一样有位置旋转 |
| **适用** | 菜单、HUD、弹窗（最常用） | 需要 UI 被 3D 物体遮挡的场景 | 3D 世界中的广告牌、全息 UI |
| **优点** | 最简单，不用管相机 | 可控制深度关系 | 真实 3D 空间感 |
| **缺点** | 不能被 3D 物体遮挡 | 需配置相机 | 复杂度高，需处理透视 |

> **小白必记**：90% 的游戏 UI 用 **Screen Space - Overlay** 就够了！只有特殊需求才用另外两种。

---

## 四、Canvas 三件套 — 缺一不可？

| 组件 | 作用 | 是否必须 |
|------|------|----------|
| **Canvas** | 核心渲染组件 | ✅ 必须 |
| **CanvasScaler** | 控制 UI 在不同分辨率下的缩放 | ✅ 建议必加 |
| **GraphicRaycaster** | UI 射线检测（点击按钮、拖拽等） | ✅ 需要交互时必加 |

```
Canvas（根）
├── CanvasScaler（适配分辨率）
├── GraphicRaycaster（处理点击）
└── EventSystem（同级，处理输入事件）
    ├── EventSystem
    └── StandaloneInputModule
```

> **为什么要 EventSystem？** 没有它，按钮点不动、滑块拖不了、输入框输不了字。创建 Canvas 时 Unity 会自动帮你创建。

---

## 五、核心一课：如何读懂获取 Canvas

```csharp
Canvas canvas = GetComponent<Canvas>();
```

| 部分 | 含义 |
|------|------|
| `Canvas` | 类名，画布组件类型 |
| `.GetComponent<Canvas>()` | 获取当前 GameObject 上的 Canvas 组件 |
| `canvas` | 变量名，用来存引用，后续写 `canvas.renderMode = ...` 等 |

**整行人话**：从当前物体上拿到画布组件，方便后续修改渲染模式等属性。

---

| 维度 | 要点 |
|------|------|
| **定义** | UI 的舞台，所有 UI 元素必须在其下 |
| **本质** | 渲染管线 + 自动批处理 + 输入事件 |
| **渲染模式** | Overlay（覆盖）/ Camera（相机前）/ World（3D） |
| **三件套** | Canvas + CanvasScaler + GraphicRaycaster |

# 第二阶段：入门篇

## 【阶段总述】

入门阶段掌握 **RectTransform（矩形变换）** 的核心概念——锚点（Anchor）和偏移（Offset），以及 **CanvasScaler（画布缩放器）** 的分辨率适配策略。  
重点：**锚点决定相对位置，偏移决定具体尺寸；CanvasScaler 决定 UI 在不同屏幕上的大小**。

---

## 一、RectTransform — UI 的位置系统（逐词读懂）

> 普通 3D 物体用 `Transform`（position/rotation/scale），UI 元素用 `RectTransform`（锚点+偏移）。

### 1.1 锚点 Anchor — UI 绑在哪里？

```csharp
rectTransform.anchorMin = new Vector2(0.5f, 0.5f);
rectTransform.anchorMax = new Vector2(0.5f, 0.5f);
```

| 部分 | 含义 |
|------|------|
| `anchorMin` | 锚点左下角在父物体上的归一化位置（0~1） |
| `anchorMax` | 锚点右上角在父物体上的归一化位置（0~1） |
| `(0.5, 0.5)` | 父物体正中心 |

**整行人话**：告诉 Unity「这个 UI 要绑在父物体的哪个位置」——绑中心就是居中，绑左下角就是靠左下角。

| 锚点配置 | 效果 |
|----------|------|
| `anchorMin = anchorMax = (0, 0)` | 绑定左下角，固定位置 |
| `anchorMin = anchorMax = (0.5, 0.5)` | 绑定中心，居中显示 |
| `anchorMin = (0, 0), anchorMax = (1, 1)` | 绑定四边，自适应拉伸填满父物体 |
| `anchorMin = (0.2, 0), anchorMax = (0.8, 0)` | 绑定底部中间一段，左右拉伸 |

### 1.2 偏移量 Offset — 离锚点多远？

```csharp
rectTransform.offsetMin = new Vector2(10f, 10f);
rectTransform.offsetMax = new Vector2(-10f, -10f);
```

| 部分 | 含义 |
|------|------|
| `offsetMin` | 相对于锚点左下角的偏移（x=左边距，y=下边距） |
| `offsetMax` | 相对于锚点右上角的偏移（x=负的右边距，y=负的上边距） |
| `(10, 10)` | 左边距 10px，下边距 10px |
| `(-10, -10)` | 右边距 10px，上边距 10px（负号表示从右边/上边向内） |

**整行人话**：锚点定了基准位置，偏移量定了具体的大小和边距。

### 1.3 Pivot — 旋转缩放中心

```csharp
rectTransform.pivot = new Vector2(0.5f, 0.5f);
```

| 部分 | 含义 |
|------|------|
| `pivot` | 旋转和缩放的中心点（0~1 归一化） |
| `(0.5, 0.5)` | 中心（默认值） |
| `(0, 0)` | 左下角 |

**整行人话**：旋转 UI 时围绕哪个点转——默认围绕中心转。

### 1.4 anchoredPosition — 锚点位置

```csharp
rectTransform.anchoredPosition = new Vector2(0f, 0f);
```

| 部分 | 含义 |
|------|------|
| `anchoredPosition` | UI 的 Pivot 相对于锚点的位置（像素或世界单位） |
| `(0, 0)` | 正好在锚点位置上 |

**整行人话**：锚点是基准线，anchoredPosition 是 UI 中心点离基准线的距离。

---

## 二、CanvasScaler — 分辨率适配（逐词读懂）

### 2.1 Scale With Screen Size — 最常用的适配方式

```csharp
CanvasScaler scaler = GetComponent<CanvasScaler>();
scaler.uiScaleMode = CanvasScaler.ScaleMode.ScaleWithScreenSize;
scaler.referenceResolution = new Vector2(1920, 1080);
scaler.screenMatchMode = CanvasScaler.ScreenMatchMode.MatchWidthOrHeight;
scaler.matchWidthOrHeight = 0.5f;
```

| 部分 | 含义 |
|------|------|
| `uiScaleMode = ScaleWithScreenSize` | 按屏幕大小缩放 UI |
| `referenceResolution = (1920, 1080)` | 参考分辨率（设计稿尺寸） |
| `screenMatchMode = MatchWidthOrHeight` | 匹配宽或高 |
| `matchWidthOrHeight = 0.5f` | 0=匹配宽，1=匹配高，0.5=折中 |

**整行人话**：告诉 Unity「我按 1920×1080 设计的，实际屏幕不一样就按比例缩放」。

### 2.2 三种适配模式对比

| 模式 | 效果 | 适用 |
|------|------|------|
| **Constant Pixel Size** | UI 像素大小固定，屏幕越大 UI 越小 | 极少用 |
| **Scale With Screen Size** | 按参考分辨率缩放，保持比例 | **最常用** |
| **Constant Physical Size** | UI 物理尺寸固定（毫米） | VR/AR 场景 |

> **小白必记**：新项目默认选 **Scale With Screen Size**，参考分辨率设为你的设计稿尺寸（如 1920×1080 或 1080×1920）。

### 案例 1：锚点固定位置 — 血条始终在左上角

**功能**：血条 UI 始终固定在屏幕左上角，不管屏幕怎么拉伸都不动。

```csharp
using UnityEngine;
using UnityEngine.UI;

public class HealthBarPosition : MonoBehaviour
{
    private RectTransform _rectTransform;

    void Awake()
    {
        _rectTransform = GetComponent<RectTransform>();
        
        // 设置锚点到左上角
        _rectTransform.anchorMin = new Vector2(0f, 1f);
        _rectTransform.anchorMax = new Vector2(0f, 1f);
        
        // 设置偏移：左边距 20px，上边距 -50px（负号表示从上方向内）
        _rectTransform.offsetMin = new Vector2(20f, -50f);
        _rectTransform.offsetMax = new Vector2(220f, -20f);
        
        // 设置中心点在左上角（旋转缩放以左上角为中心）
        _rectTransform.pivot = new Vector2(0f, 1f);
    }
}
```

#### 语法拆解

##### `_rectTransform.anchorMin = new Vector2(0f, 1f);` 是什么？

```csharp
_rectTransform.anchorMin = new Vector2(0f, 1f);
_rectTransform.anchorMax = new Vector2(0f, 1f);
```

| 部分 | 含义 |
|------|------|
| `anchorMin` | 锚点左下角坐标 |
| `anchorMax` | 锚点右上角坐标 |
| `(0, 1)` | x=0（最左边），y=1（最上边）= 左上角 |

**整行人话**：把锚点定在父物体（Canvas）的左上角，这样 UI 就绑在左上角了。

---

##### `offsetMin = (20, -50)` 和 `offsetMax = (220, -20)` 是什么？

```csharp
_rectTransform.offsetMin = new Vector2(20f, -50f);
_rectTransform.offsetMax = new Vector2(220f, -20f);
```

| 部分 | 含义 |
|------|------|
| `offsetMin.x = 20` | 左边距：离锚点左边 20px |
| `offsetMin.y = -50` | 下边距：离锚点上边 50px（负号表示向上偏移） |
| `offsetMax.x = 220` | 右边距：离锚点左边 220px → 宽度 = 220 - 20 = 200px |
| `offsetMax.y = -20` | 上边距：离锚点上边 20px → 高度 = -20 - (-50) = 30px |

**整行人话**：左上角为基准，左边空 20px，上边空 20px，宽 200px，高 30px。

### 案例 2：自适应面板 — 设置界面填满屏幕

**功能**：设置面板自适应屏幕大小，左右边距 100px，上下边距 50px，不管屏幕怎么变都保持这个比例。

```csharp
using UnityEngine;

public class SettingsPanel : MonoBehaviour
{
    private RectTransform _rectTransform;

    void Awake()
    {
        _rectTransform = GetComponent<RectTransform>();
        
        // 锚点绑四边：左下角(0,0)到右上角(1,1)
        _rectTransform.anchorMin = new Vector2(0f, 0f);
        _rectTransform.anchorMax = new Vector2(1f, 1f);
        
        // 四边偏移：左右各 100px，上下各 50px
        _rectTransform.offsetMin = new Vector2(100f, 50f);
        _rectTransform.offsetMax = new Vector2(-100f, -50f);
        
        // 中心点在中心（默认值，可不写）
        _rectTransform.pivot = new Vector2(0.5f, 0.5f);
    }
}
```

#### 语法拆解

##### `anchorMin = (0, 0), anchorMax = (1, 1)` 是什么？

```csharp
_rectTransform.anchorMin = new Vector2(0f, 0f);
_rectTransform.anchorMax = new Vector2(1f, 1f);
```

| 部分 | 含义 |
|------|------|
| `(0, 0)` | 左下角 |
| `(1, 1)` | 右上角 |
| **效果** | 锚点变成一个覆盖整个父物体的矩形，UI 会随父物体拉伸 |

**整行人话**：把 UI 的四个边分别绑到父物体的四个边上，屏幕拉伸时 UI 跟着拉伸。

---

##### `offsetMin = (100, 50), offsetMax = (-100, -50)` 是什么？

```csharp
_rectTransform.offsetMin = new Vector2(100f, 50f);
_rectTransform.offsetMax = new Vector2(-100f, -50f);
```

| 部分 | 含义 |
|------|------|
| `offsetMin.x = 100` | 左边距 100px |
| `offsetMin.y = 50` | 下边距 50px |
| `offsetMax.x = -100` | 右边距 100px（负号） |
| `offsetMax.y = -50` | 上边距 50px（负号） |

**整行人话**：虽然绑了四边，但不是贴边，而是留了 100px/50px 的边距。

---

#### 逐行详解

| 行 | 代码 | 含义 |
|----|------|------|
| 9 | `GetComponent<RectTransform>()` | 获取矩形变换组件 |
| 12~13 | `anchorMin/Max = (0,0)/(1,1)` | 锚点绑四边，自适应 |
| 16~17 | `offsetMin/Max` | 四边留边距 |
| 20 | `pivot = (0.5, 0.5)` | 中心旋转 |

1. Canvas 下创建 Panel（带 Image 的空物体）  
2. 挂此脚本

### 案例 3：代码动态创建按钮并定位

**功能**：用代码在屏幕底部居中创建一排按钮，每个按钮间距相等。

```csharp
using UnityEngine;
using UnityEngine.UI;

public class CreateButtons : MonoBehaviour
{
    public Button buttonPrefab;          // 按钮预制体，Inspector 拖入
    public int buttonCount = 4;          // 按钮数量
    public float spacing = 20f;          // 按钮间距
    public float bottomMargin = 50f;     // 底部边距
    public float buttonWidth = 120f;     // 按钮宽度
    public float buttonHeight = 50f;     // 按钮高度

    void Start()
    {
        // 计算总宽度：按钮宽度×数量 + 间距×(数量-1)
        float totalWidth = buttonWidth * buttonCount + spacing * (buttonCount - 1);
        
        // 创建每个按钮
        for (int i = 0; i < buttonCount; i++)
        {
            // 实例化按钮
            Button btn = Instantiate(buttonPrefab, transform);
            RectTransform rt = btn.GetComponent<RectTransform>();
            
            // 设置按钮尺寸
            rt.sizeDelta = new Vector2(buttonWidth, buttonHeight);
            
            // 设置锚点到底部中心
            rt.anchorMin = new Vector2(0.5f, 0f);
            rt.anchorMax = new Vector2(0.5f, 0f);
            
            // 计算每个按钮的 x 位置：从左到右排列
            float startX = -totalWidth / 2 + buttonWidth / 2;
            float x = startX + i * (buttonWidth + spacing);
            float y = bottomMargin + buttonHeight / 2;
            
            // 设置位置
            rt.anchoredPosition = new Vector2(x, y);
            
            // 设置按钮文字
            btn.GetComponentInChildren<Text>().text = "按钮 " + (i + 1);
        }
    }
}
```

#### 语法拆解

##### `rt.sizeDelta = new Vector2(width, height);` 是什么？

```csharp
rt.sizeDelta = new Vector2(buttonWidth, buttonHeight);
```

| 部分 | 含义 |
|------|------|
| `sizeDelta` | 当锚点重合时，直接设置 UI 的宽度和高度 |
| `(120, 50)` | 宽 120px，高 50px |

**整行人话**：锚点重合时，用 sizeDelta 直接定大小，最方便。

---

##### `Instantiate(prefab, transform)` 是什么？

```csharp
Button btn = Instantiate(buttonPrefab, transform);
```

| 部分 | 含义 |
|------|------|
| `Instantiate` | 创建物体实例 |
| `buttonPrefab` | 要创建的预制体 |
| `transform` | 父物体（Canvas 的子物体） |

**整行人话**：从预制体克隆一个按钮，挂到当前物体（Canvas）下面。

---

#### 逐行详解

| 行 | 代码 | 含义 |
|----|------|------|
| 5~9 | 公开参数 | 按钮预制体、数量、间距、尺寸等 |
| 13 | `totalWidth` | 计算所有按钮加间距的总宽度 |
| 16 | `for` 循环 | 创建 buttonCount 个按钮 |
| 19 | `Instantiate` | 克隆按钮预制体 |
| 21 | `sizeDelta` | 设置按钮尺寸 |
| 24~25 | `anchorMin/Max = (0.5, 0)` | 锚点绑底部中心 |
| 28~30 | 计算 x 位置 | 从左到右均匀排列 |
| 33 | `anchoredPosition` | 设置按钮位置 |
| 36 | 设置文字 | 按钮显示 "按钮 1" "按钮 2" 等 |

#### 操作提示

2. 把脚本挂到 Canvas 上

## 一、Layout Groups — 自动排版引擎

### 1.1 三种布局组

| 组件 | 效果 | 适用 |
|------|------|------|
| **Horizontal Layout Group** | 子物体水平排列 | 一排按钮、工具栏 |
| **Vertical Layout Group** | 子物体垂直排列 | 一列选项、列表项 |
| **Grid Layout Group** | 子物体网格排列 | 背包格子、图标墙 |

### 1.2 核心属性

```csharp
HorizontalLayoutGroup hlg = GetComponent<HorizontalLayoutGroup>();
hlg.spacing = 10f;                    // 子物体间距
hlg.padding.left = 20f;               // 左边距
hlg.padding.right = 20f;              // 右边距
hlg.childAlignment = TextAnchor.MiddleCenter;  // 子物体对齐方式
hlg.childForceExpandWidth = true;     // 子物体是否强制展开宽度
hlg.childControlWidth = true;         // 是否控制子物体宽度
```

| 属性 | 含义 |
|------|------|
| `spacing` | 子物体之间的距离 |
| `padding` | 布局组本身的边距（left/right/top/bottom） |
| `childAlignment` | 子物体在布局中的对齐方式 |
| `childForceExpandWidth/Height` | 是否让子物体撑满可用空间 |
| `childControlWidth/Height` | 是否覆盖子物体自己的尺寸 |

**整行人话**：加一个 Layout Group 组件，子物体自动排整齐，不用手动算位置。

## 二、ContentSizeFitter — 内容自适应

```csharp
ContentSizeFitter csf = GetComponent<ContentSizeFitter>();
csf.horizontalFit = ContentSizeFitter.FitMode.PreferredSize;
csf.verticalFit = ContentSizeFitter.FitMode.PreferredSize;
```

| 属性 | 效果 |
|------|------|
| `HorizontalFit = PreferredSize` | 宽度自适应子物体内容 |
| `VerticalFit = PreferredSize` | 高度自适应子物体内容 |
| `HorizontalFit = Unconstrained` | 宽度不受限制 |

**整行人话**：面板大小跟着内容走——内容多面板就长，内容少面板就短。

## 三、Canvas 性能优化

### 3.1 Canvas 分层策略

```
主 Canvas（Overlay）→ 游戏 HUD、血条等高频更新的 UI
弹窗 Canvas（Overlay）→ 弹窗、菜单等低频更新的 UI
3D UI Canvas（World Space）→ 场景中的 3D UI
```

| 原则 | 说明 |
|------|------|
| **高频更新和低频更新分开** | 避免刷新整个 Canvas |
| **遮挡剔除** | 被挡住的 UI 不渲染 |
| **Canvas Group 控制显隐** | 比 SetActive 更高效 |

### 3.2 性能影响因素

| 因素 | 影响 | 优化 |
|------|------|------|
| Canvas 数量 | 越多 Draw Call 越多 | 合理分层，不要过多 |
| UI 嵌套深度 | 太深影响布局计算 | 保持扁平结构 |
| Layout Group 动态更新 | 每帧计算开销大 | 内容不变时禁用 |
| Mask 组件 | 裁剪开销大 | 能用 RectMask2D 就不用 Mask |

## 四、常见误区与最佳实践

| 误区 | 正确做法 |
|------|----------|
| UI 不在 Canvas 下 | 所有 UI 必须放在 Canvas 子物体中 |
| 用 Transform 控制 UI 位置 | 用 RectTransform 的 anchoredPosition |
| 锚点重合却用 offsetMin/Max | 锚点重合时用 sizeDelta 更简单 |
| CanvasScaler 没设参考分辨率 | 设为设计稿尺寸（如 1920×1080） |
| 按钮点不动 | 检查 EventSystem 是否存在、GraphicRaycaster 是否添加 |
| 频繁 Instantiate/Destroy UI | 用对象池（Object Pooling） |
| 多层 Mask 嵌套 | 减少嵌套，能用 RectMask2D 就不用 Mask |
| 每帧修改 Layout Group | 只在内容变化时更新 |

| 性能 | Canvas 分层、避免过度嵌套、对象池 |
