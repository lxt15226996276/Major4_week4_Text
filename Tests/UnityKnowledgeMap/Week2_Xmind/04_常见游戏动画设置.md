## 【阶段总述】

零基础阶段掌握 **「让动画动起来」三件事**：导入 FBX、设循环、挂 Animator。  
按步骤一步步走，保证第一次就能让角色在场景里动起来。

---

## 一、FBX 模型动画导入三步法

### 1.1 把 FBX 拖进 Unity

- 从文件管理器把 `xxx.fbx` 直接拖到 Unity Project 窗口的 `Assets/Art/Models/` 文件夹
- Unity 会自动导入并生成对应资源

### 1.2 Model 选项卡（模型）

选中 FBX，Inspector 里第一个选项卡是 **Model**：

| 设置项 | 推荐值 | 说明 |
|--------|--------|------|
| **Scale Factor** | 1 或 0.01 | FBX 单位是厘米就设 0.01（1cm = 0.01m） |
| **Use File Scale** | 看情况 | 勾上用文件里的缩放值 |
| **Mesh Compression** | Off / Low / Medium / High | 压缩网格减少文件，太高会变形 |
| **Read/Write Enabled** | 默认关 | 运行时要改网格才开 |
| **Normals** | Import | 导入模型自带的法线 |
| **Tangents** | Import | 导入切线（法线贴图需要） |

**小白小贴士**：
- 模型太大或太小 → 调 **Scale Factor**
- 模型发黑 → 检查 Normals 是否正确
- 模型有闪烁面 → 可能是法线反了，调 `Normals > Calculate` 试试

### 1.3 Rig 选项卡（骨骼绑定）

第二个选项卡 **Rig**：

| 选项 | 说明 | 什么时候用 |
|------|------|-----------|
| **None** | 没有骨骼动画 | 纯静态道具 |
| **Legacy** | 旧版动画系统 | 老项目兼容，新项目别用 |
| **Generic** | 通用骨骼，不需要 Avatar | 怪物、机械、非人形 |
| **Humanoid** | 人形骨骼，自动生成 Avatar | 人、类人生物 |

**怎么选（一句话）**：
- 两个胳膊两条腿 → **Humanoid**（动画可复用）
- 其他 → **Generic**（不能复用但性能稍好）

**设置 Humanoid 时**：
- 选了 Humanoid 后点 **Configure...**
- 会进入 Avatar 配置界面，检查骨骼映射
- 绿色勾勾 = 没问题
- 红色问号 = 没识别到，手动拖对应骨骼进去
- 配置完点 **Done** 返回

### 1.4 Animation 选项卡（动画片段）

第三个选项卡 **Animation**：

| 设置项 | 说明 |
|--------|------|
| **Import Animation** | 勾选才导入动画 |
| **Bake Animation** | 烘焙 IK 到 FK（一般默认） |
| **Resample Curves** | 重采样曲线（Euler 转 Quaternion） |
| **Clips 列表** | FBX 里的所有动画片段 |

**分割动画片段**（一个 FBX 里多个动画时）：

```
步骤1  在 Clips 列表点 + 号，新增一个 Clip
步骤2  改名字（如 Idle / Walk / Jump）
步骤3  拖 Start / End 滑块，或直接输入帧数
步骤4  预览窗口看动作对不对
步骤5  所有 Clip 都弄完 → 点 Apply
```

**怎么知道哪帧到哪帧？**
- 问美术要动画表（推荐，最准）
- 自己在预览窗口里一帧帧拖，看动作起止

## 二、循环动画设置（Loop Time / Loop Pose）

### 2.1 两个关键勾选

选中某个 Clip 后往下拉：

| 选项 | 作用 | 什么时候勾 |
|------|------|-----------|
| **Loop Time** | 时间循环播放 | 走路/跑步/待机 等需要循环的动作 |
| **Loop Pose** | 首尾姿势无缝衔接 | 勾了 Loop Time 后最好也勾上 |

### 2.2 Cycle Offset（循环偏移）

- 范围 0~1
- 让循环从动画的某个百分比开始
- 比如走路动画第一帧刚好脚在地上，但你想从抬脚开始 → 调偏移

### 2.3 Root Transform 三个设置（循环相关）

| 设置 | 作用 |
|------|------|
| **Root Transform Rotation** | 根骨骼旋转是否参与循环 |
| **Root Transform Position (Y)** | 根骨骼 Y 轴位移是否参与循环 |
| **Root Transform Position (XZ)** | 根骨骼 XZ 轴位移是否参与循环 |

- **Bake Into Pose**：勾了 = 这部分动画算到姿势里，不算 Root Motion
- 小白先不用管，等学 Root Motion 时再回头看

### 2.4 走路动画「滑步」怎么办？

**症状**：角色走路时脚在地上滑来滑去，像踩在冰上。

**常见原因 + 解决**：

| 原因 | 解决 |
|------|------|
| 动画本身做的就有滑步 | 让美术调，或开 Root Motion 用动画驱动位移 |
| 代码移动速度和动画速度不匹配 | 调 `animator.speed` 或代码移动速度 |
| Loop Pose 没勾上，首尾帧脚位置不一样 | 勾上 Loop Pose；或让美术修 |
| 用代码位移但动画是 Root Motion 的 | 二选一：要么关 Root Motion 用代码，要么开 Root Motion |

## 三、让角色动起来（Animator 五步法）

```
步骤1  创建 Animator Controller
       右键 → Create → Animator Controller
       放 Assets/Animations/Controllers/ 里

步骤2  打开 Animator 窗口
       Window → Animation → Animator
       或双击 Controller 资产

步骤3  拖动画建状态
       把 Animation Clip 从 Project 拖到 Animator 画布
       第一个拖进去的会是橙色（默认状态）

步骤4  挂 Animator 组件
       选中模型 → Add Component → Animator
       Controller 字段拖刚才的 Controller 资产
       Humanoid 还需要指定 Avatar

步骤5  Play 运行
       角色会播放默认状态的动画
```

> **新手最常见的坑**：模型拖进去是 T-Pose 不动  
> 检查清单：① 有没有 Animator 组件？② Controller 字段有没有赋值？③ Avatar 有没有？④ 默认状态是不是有动画的？

| 维度 | 要点 |
|------|------|
| **导入** | Model / Rig / Animation 三个选项卡 |
| **Rig 选型** | 人形选 Humanoid，其他选 Generic |
| **分割 Clip** | Animation 选项卡 Clips 列表里加减 |
| **循环** | Loop Time 勾上，走路跑步要循环 |

## 一、动画事件 Animation Event

### 1.1 什么是动画事件？

在动画的**某一帧**上挂一个函数名，动画播到这一帧时，自动调用挂在同一物体上的脚本里的同名方法。

**典型用途**：
- 脚步声（脚落地那一帧播放音效）
- 攻击判定（挥刀到中间那一帧开启碰撞）
- 生成特效（抬手那一帧生成火球）
- 武器切换（换弹动画某一帧换模型）

### 1.2 添加事件的两种方式

#### 方式 A：Animation 窗口（推荐，运行时动态加）

```
步骤1  选中带 Animator 的物体
步骤2  Window → Animation → Animation（Ctrl+6）
步骤3  选中要加事件的 Clip
步骤4  时间轴拖到目标帧
步骤5  点时间轴上方的「+事件」按钮（小旗子图标）
步骤6  在 Inspector 的事件里选函数名
```

#### 方式 B：导入设置里加（永久的，FBX 级）

在 FBX 的 Animation 选项卡 → 选中 Clip → Events 展开 → 添加事件

### 1.3 支持的参数类型

| 参数类型 | 说明 |
|----------|------|
| `float` | 浮点数 |
| `int` | 整数 |
| `string` | 字符串 |
| `Object` | Unity Object 引用（比如音频、预制体） |

> **注意**：一个事件只能传 **一个参数**（或不传）。要传多个就封装成类，或者用 string 传 JSON。

### 1.4 代码示例

```csharp
using UnityEngine;

public class AnimationEventTest : MonoBehaviour
{
    // 脚落地事件（动画里配好函数名 = "OnFootStep"）
    void OnFootStep()
    {
        Debug.Log("啪！脚踩地了");
        // 播放脚步声、生成尘土特效等
    }

    // 带 float 参数的事件
    void OnAttackFrame(float damage)
    {
        Debug.Log("攻击判定帧，伤害=" + damage);
    }

    // 带 Object 参数（拖个 AudioClip 进去）
    void OnPlaySound(Object audioObj)
    {
        AudioClip clip = audioObj as AudioClip;
        if (clip != null)
            AudioSource.PlayClipAtPoint(clip, transform.position);
    }
}
```

### 1.5 常见坑

| 问题 | 原因 / 解决 |
|------|------------|
| 事件不触发 | 检查函数名是否完全一致（大小写）；检查脚本是否挂在同一物体上 |
| 报 "Function not found" 警告 | 函数名拼错了；或者参数类型不匹配 |
| 触发了好几次 | 动画循环播放，每次到那帧都会触发，正常 |
| 事件时有时无 | 动画过渡（Transition）期间可能跳过事件帧 |

## 二、Root Motion 根运动

### 2.1 什么是 Root Motion？

**Root Motion（根运动）**：让**动画本身**驱动角色位移和旋转，而不是用代码 `transform.Translate`。

| 对比 | 代码控制位移 | Root Motion |
|------|-------------|-------------|
| 位移来源 | C# 代码算的 | 动画师 K 的 |
| 手感 | 比较飘/滑 | 自然，有重量感 |
| 同步 | 容易滑步 | 脚步和位移天然同步 |
| 灵活度 | 高，想怎么动就怎么动 | 受动画限制 |
| 适合 | 简单移动、物理驱动 | 真实感强的角色动作 |

### 2.2 开启方式

```
步骤1  选中 FBX → Animation 选项卡 → 选中 Clip
       往下拉 Root Transform 部分：
       - Root Transform Position (XZ) → Bake Into Pose 取消勾选
         （不勾 = 这部分算 Root Motion）
       - Root Transform Rotation → 根据需要
       点 Apply

步骤2  选中角色的 Animator 组件
       勾选 Apply Root Motion

步骤3  Play 运行
       角色会被动画带着走
```

### 2.3 自定义 Root Motion：OnAnimatorMove

有时候你想 **混合**：一部分位移用动画，一部分用代码（比如加物理、加根运动缩放）。  
用 `OnAnimatorMove` 回调：

```csharp
using UnityEngine;

public class CustomRootMotion : MonoBehaviour
{
    Animator anim;
    public float speedScale = 1f;      // 根运动缩放

    void Awake()
    {
        anim = GetComponent<Animator>();
    }

    // 这个方法会在 Animator 每帧计算完 Root Motion 后调用
    void OnAnimatorMove()
    {
        // anim.deltaPosition：这一帧根运动算出来的位移
        transform.position += anim.deltaPosition * speedScale;

        // anim.deltaRotation：这一帧根运动算出来的旋转
        transform.rotation *= anim.deltaRotation;
    }
}
```

#### 逐词读懂

```csharp
void OnAnimatorMove()
{
    transform.position += anim.deltaPosition * speedScale;
}
```

| 部分 | 含义 |
|------|------|
| `OnAnimatorMove` | MonoBehaviour 回调：Animator 计算完 Root Motion 后自动调用 |
| `anim.deltaPosition` | 这一帧根运动产生的位移增量（Vector3） |
| `* speedScale` | 乘以缩放系数（调大 = 走得快，调小 = 走得慢） |
| `transform.position +=` | 加到物体位置上 |

**整行人话**：Animator 算出来这一帧该挪多少，我可以乘个系数再应用，相当于调速。

### 2.4 什么时候用 Root Motion？

| 场景 | 推荐方式 |
|------|---------|
| 真实感强的第三人称动作游戏 | ✅ Root Motion |
| MOBA / 俯视角，移动手感要灵敏 | ❌ 代码控制 |
| 攻击连招位移 | ✅ Root Motion（更自然） |
| 物理驱动的角色（Rigidbody） | ⚠️ 混合：用 MovePosition 应用 Root Motion |
| 简单小怪巡逻 | ❌ 代码够了 |

## 三、2D 动画设置

### 3.1 Sprite 序列帧动画（最常用）

**原理**：一组图片快速切换，形成动画（像翻书）。

**制作步骤**：

```
步骤1  导入 Sprite 序列帧图片
       图片的 Import Settings 设为 Sprite (2D and UI)
       多张是同个动画的 → Sprite Mode = Multiple → Sprite Editor 切割

步骤2  全选所有帧 → 拖到 Hierarchy 窗口
       Unity 会自动生成：
       - 一个 GameObject
       - 一个 Animation Clip
       - 一个 Animator Controller
       - 挂好 Sprite Renderer 和 Animator

步骤3  双击 Animation Clip 可以调速度、加事件
```

**调整帧率**：
- Animation 窗口 → 右键 Clip 名称 → Sample Rate
- 一般 12 fps、24 fps 常用

### 3.2 2D 骨骼动画（Skinned Sprite）

**原理**：给一张 2D 图加骨骼，骨骼驱动顶点变形（像 3D 蒙皮）。

**适用**：复杂角色动画、需要多角度、美术愿意做骨骼绑定。

**简要步骤**：

```
步骤1  安装 2D Animation 包（Package Manager）
步骤2  导入 PSB 文件（Photoshop 分层，或 Sprite 图）
步骤3  Sprite Editor → Skinning 模式 → 创建骨骼 → 自动/手动蒙皮
步骤4  生成 Animation Clip（和 3D 一样用 Animator）
```

> **小白建议**：先把序列帧玩熟练，骨骼动画等项目需要再学。

### 3.3 Sprite 渲染排序

| 设置 | 作用 |
|------|------|
| **Sorting Layer** | 排序层，先定义层，层与层之间有顺序 |
| **Order in Layer** | 同一层内的顺序，数字越大越靠前 |
| **Z 轴** | 也能影响（取决于相机设置），推荐用 Layer |

**设置 Sorting Layer**：
- 任意 Sprite Renderer → Sorting Layer 下拉 → Add Sorting Layer
- 拖顺序：上面的层在后面，下面的层在前面

## 四、UI 动画选型

### 4.1 三种常见方案对比

| 方案 | 学习成本 | 性能 | 适用场景 |
|------|---------|------|----------|
| **DOTween** | 低，代码写 | 高（轻量） | 按钮缩放、弹面板、飘字、进度条 |
| **Animator + Animation Clip** | 中，可视化 | 中 | 复杂 UI 状态机、多状态切换 |
| **Unity UI Layout + 代码** | 低 | 高 | 简单的显示/隐藏、颜色变化 |

### 4.2 选型建议

**90% 的 UI 动画 → DOTween**：
- 写代码 1 行搞定，不用建 Clip、建 Controller
- 性能好，GC 少
- 适合按钮点击反馈、面板进出场、数值滚动等

**状态很复杂的 UI → Animator**：
- 比如一个面板有 Normal / Hover / Pressed / Disabled / Selected 好几种状态
- 或者 UI 有复杂的状态流转
- 这时候状态机比一堆 if-else 清晰

### 4.3 DOTween 常用示例

```csharp
using UnityEngine;
using DG.Tweening;
using UnityEngine.UI;

public class UIAnimExample : MonoBehaviour
{
    public Button btn;
    public Image image;
    public RectTransform panel;

    void Start()
    {
        // 按钮点击缩放反馈
        btn.onClick.AddListener(() =>
        {
            btn.transform.DOScale(0.9f, 0.1f).SetLoops(2, LoopType.Yoyo);
        });

        // 图片渐显
        image.DOFade(1f, 0.5f).From(0f);

        // 面板从右边滑入
        panel.DOAnchorPosX(0, 0.5f).From(new Vector2(1000, 0)).SetEase(Ease.OutBack);
    }
}
```

> DOTween 更多用法参考 major3/05_DoTween.md

## 一、动画压缩

动画文件里存的是**曲线数据**（每根骨骼每帧的位置旋转），数据量很大。压缩可以大幅减小文件体积和内存占用。

### 1.1 三种压缩级别

FBX → Animation 选项卡 → 选中 Clip → **Compression**：

| 级别 | 说明 | 精度损失 | 文件大小 |
|------|------|---------|---------|
| **Off** | 不压缩 | 无 | 最大 |
| **Keyframe Reduction** | 删减冗余关键帧 | 小 | 中 |
| **Optimal** | Unity 自动优化（旋转用四元数压缩） | 极小 | 最小（推荐） |

**推荐设置**：**Optimal**（人眼几乎看不出区别，体积小很多）

### 1.2 精简动画曲线（Animation Curve）

有些骨骼/属性动画里用不到，但每帧都在采样，浪费性能。

**怎么精简**：

```
FBX → Animation 选项卡 → Curves 展开
    → 移除不需要的属性（比如手指不用就删）
    → 或用 Mask 只保留需要的身体部位
```

**或者运行时优化**：
- **Animator 组件 → Culling Mode**：
  - `Always Animate`：始终更新（默认）
  - `Cull Update Transforms`：渲染被剔除时，不更新骨骼 Transform（但状态机还跑）
  - `Cull Completely`：完全不更新

**同屏大量小怪** → 用 Cull Update Transforms，离屏的少算很多。

---

## 二、动画性能优化清单

| 优化项 | 做法 | 效果 |
|--------|------|------|
| 压缩 | Compression = Optimal | 体积减 50~80% |
| 精简曲线 | 移除不用的骨骼/属性 | 减少每帧计算量 |
| Culling Mode | 离屏角色设 Cull Update | 减少 Transform 更新 |
| 减少 Rebind | 对象池用 `animator.enabled=false`，不是 `gameObject.SetActive(false)` | 避免 Rebind 开销 |
| Generic 替代 Humanoid | 不需要重定向的角色用 Generic | CPU 降 15~20% |
| LOD | 远距离用低精度动画/少骨骼 | 远的少算 |
| 同屏数量 | 控制同屏 Animator 数量 | 每个 Animator 都有开销 |
| StringToHash | 参数名预计算哈希 | 减少字符串比较 GC |
| 状态单例化 | FSM 状态不复 new | 零 GC |
| 简易动画用 DOTween | UI、小道具别上 Animator | 省性能又省事 |

### Profiler 定位动画瓶颈

```
步骤1  Window → Analysis → Profiler（Ctrl+7）
步骤2  Play 运行
步骤3  看 CPU Usage 图表中的：
        - Animation：动画系统总耗时
        - Animator.Update：Animator 逻辑
        - Animation.Update 或 Culling：骨骼更新
步骤4  点具体帧看层级，找占比高的
```

**常见动画性能瓶颈**：
- 同屏角色太多（最常见）
- Humanoid + 大量手指/面部骨骼
- 动画曲线太密
- 频繁 Rebind（对象池实现不对）

## 三、人形动画重定向（Retargeting）

### 3.1 什么是重定向？

同一套 Humanoid Animation Clip，可以放到不同的人形模型上播放。  
因为 Humanoid 有统一的骨骼映射（Avatar），动画数据是「标准人形」的，套到任何映射好了的模型上都能用。

**好处**：买一套动画库，所有角色都能用，省钱省时。

### 3.2 操作步骤

```
前提：两个模型都是 Humanoid Rig，且都 Configure 成功（绿色勾勾）

步骤1  把 A 模型的动画 Clip 提取出来（或单独的动画 FBX）
步骤2  B 模型的 Controller 里直接用这个 Clip
步骤3  Play 运行，B 模型就会播 A 模型的动画
```

### 3.3 常见问题

| 问题 | 原因 / 解决 |
|------|------------|
| 手/脚位置不对 | 两个模型比例差太多，调 Avatar Muscle 或让美术修 |
| 手指动不了 | 检查 Avatar 的手指骨骼有没有映射上 |
| 动画变形严重 | 模型不是标准 T-Pose，或 Rig 配置有问题 |
| 高矮不一样脚插地 | 调 Scale Factor 或用 Root Motion Y 轴 |

## 四、项目规范与最佳实践

### 4.1 命名规范

| 资源类型 | 命名规范 | 示例 |
|----------|---------|------|
| 模型 FBX | `ch_角色名` / `pr_道具名` | `ch_knight.fbx` |
| 动画 Clip | `角色名_动作名` | `knight_idle.anim` |
| 循环动画后缀 | 加 `_loop` | `knight_walk_loop.anim` |
| Animator Controller | `AC_角色名` | `AC_Knight.controller` |
| Animator 参数 | 帕斯卡命名 | `IsGrounded`、`Speed`、`JumpTrigger` |
| Avatar Mask | `AM_部位` | `AM_UpperBody.mask` |
| 状态名 | 帕斯卡命名 | `Idle`、`Walk`、`Jump` |

### 4.2 文件夹组织

```
Assets/
├── Art/
│   ├── Characters/        # 角色模型
│   │   ├── Knight/
│   │   │   ├── ch_knight.fbx
│   │   │   ├── Textures/
│   │   │   └── Materials/
│   └── Props/             # 道具模型
│
├── Animations/
│   ├── Clips/             # 动画片段
│   │   ├── Shared/        # 通用/重定向动画
│   │   └── Knight/
│   ├── Controllers/       # Animator Controller
│   └── AvatarMasks/       # Avatar Mask
│
└── Scripts/
    ├── Animation/         # 动画相关脚本
    │   ├── PlayerAnim.cs
    │   └── AnimationEvents.cs
    └── FSM/               # 状态机
```

### 4.3 Controller 设计规范

- **分层清晰**：Base Layer（全身）、Upper Body（上半身）、Facial（面部）
- **默认状态**：一打开 Controller 就知道默认状态是什么
- **状态命名**：见名知意，不用缩写（写 `Idle` 别写 `idl`）
- **注释**：复杂的 Transition 加备注（右键 → Add Note）
- **参数少而精**：没用的参数删掉，别堆一堆

## 五、常见问题排查速查

### 5.1 动画不动 / 一直 T-Pose

- [ ] 物体上有没有 **Animator** 组件？
- [ ] Animator 的 **Controller** 字段拖了吗？
- [ ] Humanoid 的话 **Avatar** 拖了吗？Avatar Configure 成功了吗？
- [ ] Controller 里默认状态是不是有 **Motion**（空的就 T-Pose）？
- [ ] 物体是不是 Active 的？Animator 组件是不是 Enabled？
- [ ] `animator.speed` 是不是被设成 0 了？

### 5.2 动画滑步 / 脚飘

- [ ] 代码移动速度和动画速度匹配吗？（调 anim.speed）
- [ ] 该用 Root Motion 的用代码移了？（或反过来）
- [ ] Loop Pose 勾了吗？
- [ ] 动画本身脚是不是就滑？（让美术检查）

### 5.3 动画不循环

- [ ] Clip 的 **Loop Time** 勾了吗？
- [ ] Controller 里的 State 是不是循环动画？
- [ ] Transition 的 Has Exit Time 是不是勾了导致播完切走了？

### 5.4 Trigger 没反应 / 偶发失效

- [ ] 参数名拼写对吗？大小写对吗？
- [ ] Transition 的 **Has Exit Time** 是不是挡住了？
- [ ] 是不是 SetTrigger 太频繁？加 **ResetTrigger** 试试
- [ ] 是不是 Any State 自循环了？

### 5.5 动画卡顿 / 性能差

- [ ] Compression 设 Optimal 了吗？
- [ ] 同屏是不是有太多 Animator？
- [ ] 是不是 Humanoid 的小怪可以换成 Generic？
- [ ] Culling Mode 设了吗？
- [ ] 是不是频繁 SetActive 触发 Rebind？

### 5.6 动画事件不触发

- [ ] 函数名完全一致吗（大小写）？
- [ ] 脚本挂在 **同一个物体** 上了吗？
- [ ] 函数是 **public / private** 都可以，但不能是 static
- [ ] 参数类型匹配吗？
- [ ] 是不是在 Transition 过渡期间，事件帧被跳过了？
