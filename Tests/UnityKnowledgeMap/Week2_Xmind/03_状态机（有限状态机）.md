## 【全文总述】

**有限状态机（Finite State Machine，FSM）** 是一种行为建模模式：系统在 **有限个状态** 之间，根据 **条件** 切换，且 **同一时刻只处于一个状态**。  
在 Unity 游戏开发中无处不在——Animator Controller 就是可视化的 FSM，AI 行为、游戏流程、UI 状态都在用。  
本文从 if-else 入门，到枚举+Switch，再到经典 **State Pattern（接口/抽象类）**，最后讲 **层级状态机（HFSM）** 与 **栈式状态机（Pushdown FSM）**，全面覆盖。

# 第一阶段：零基础篇

## 【阶段总述】

零基础阶段建立 **FSM 三要素认知**，并从最朴素的 if-else 入手，理解「状态切换」到底在做什么。  
目标：能用自己的话解释「什么是状态机」，并写出最简单的 2~3 个状态切换。

---

## 一、定义 — 什么是有限状态机？

| 项目 | 说明 |
|------|------|
| **全称** | Finite State Machine（FSM），有限状态机 |
| **一句话** | 系统有有限个状态，根据条件在状态间切换，**同一时间只有一个状态** |
| **三要素** | **State（状态）**、**Transition（转移）**、**Condition（条件）** |
| **生活类比** | 电灯：关→开→关，同一时刻要么亮要么灭 |
| **游戏类比** | 角色：Idle / Walk / Jump / Attack，同一时刻只能做一个动作 |

### 三要素详解

| 要素 | 含义 | 举例（玩家） |
|------|------|-------------|
| **State 状态** | 系统当前所处的模式/行为 | Idle、Walk、Jump、Attack |
| **Transition 转移** | 从一个状态到另一个状态的路径 | Idle → Walk、Walk → Jump |
| **Condition 条件** | 触发转移的判断依据 | 按下方向键、按跳跃键、着地 |

**状态机的铁律**：
1. 状态数量是 **有限的**
2. 同一时刻 **只有一个** 当前状态
3. 状态切换是 **瞬时的**（要么在 A，要么在 B，没有中间态）

---

## 二、本质 — 为什么要用状态机？

### 2.1 没有状态机的代码长什么样？

```csharp
void Update()
{
    if (isDead)
    {
        // 死亡逻辑
    }
    else if (isAttacking)
    {
        // 攻击逻辑
        if (攻击结束) isAttacking = false;
    }
    else if (isJumping)
    {
        // 跳跃逻辑
    }
    else if (isMoving)
    {
        // 移动逻辑
    }
    // ... 十几种状态，if-else 堆成山
}
```

**问题**：
- 状态越多，if-else 越长，越难维护
- 状态之间的关系（谁能切到谁）散落在代码里，看不清
- 加新状态要改一大片，容易出 bug

### 2.2 有了状态机之后

```
每个状态是独立的「模块」：
    IdleState → Enter / Update / Exit
    WalkState → Enter / Update / Exit
    JumpState → Enter / Update / Exit

状态机只做两件事：
    1. 当前状态每帧调用 Update
    2. 条件满足时，切换状态（Exit 旧的 → Enter 新的）
```

**本质**：把「一大坨 if-else」拆成「每个状态一个模块」，用状态机统一调度。  
**好处**：加新状态只加新类，不改动原有代码（开闭原则）。

## 三、最朴素的 FSM — if-else 版

### 案例：台灯的三种状态

```
关 → 弱光 → 强光 → 关 ...
按一下切一个状态
```

```csharp
using UnityEngine;

public class SimpleFSM_Lamp : MonoBehaviour
{
    enum LampState { Off, Dim, Bright }    // 定义有哪些状态
    LampState currentState = LampState.Off; // 当前状态

    void Update()
    {
        if (Input.GetKeyDown(KeyCode.Space))
        {
            // 根据当前状态决定切到哪个状态
            if (currentState == LampState.Off)
                currentState = LampState.Dim;
            else if (currentState == LampState.Dim)
                currentState = LampState.Bright;
            else if (currentState == LampState.Bright)
                currentState = LampState.Off;
        }

        // 根据当前状态执行行为
        if (currentState == LampState.Off)
            RenderSettings.ambientIntensity = 0f;
        else if (currentState == LampState.Dim)
            RenderSettings.ambientIntensity = 0.3f;
        else if (currentState == LampState.Bright)
            RenderSettings.ambientIntensity = 1f;
    }
}
```

### 逐词读懂切换逻辑

```csharp
if (currentState == LampState.Off)
    currentState = LampState.Dim;
```

| 部分 | 含义 |
|------|------|
| `currentState == LampState.Off` | 判断「现在是不是关着的」 |
| `currentState = LampState.Dim` | 把当前状态改成「弱光」 |

**整行人话**：如果现在灯是关的，按一下就变成弱光。

### 优缺点

| 优点 | 缺点 |
|------|------|
| 简单直接，一看就懂 | 状态多了 if-else 爆炸 |
| 适合 2~3 个状态的超简单场景 | 状态行为复杂时代码又长又乱 |
| 零学习成本 | 加状态要改多处，容易漏 |

> **结论**：if-else 版是 FSM 的「Hello World」，用来建立概念。真实项目要用更好的实现。

|------|------|

| **三要素** | State 状态 / Transition 转移 / Condition 条件 |

## 一、实现 1：枚举 FSM（分方法版）

### 思路

- 用 `enum` 定义所有状态
- 用 `switch` 在 Update 里分发
- 每个状态的逻辑拆成独立方法（Enter_xxx / Update_xxx / Exit_xxx）
- 切换状态时手动调用 Exit 旧的、Enter 新的

### 完整代码

```csharp
using UnityEngine;

public class PlayerFSM_Enum : MonoBehaviour
{
    // ========== 状态枚举 ==========
    enum PlayerState { Idle, Walk, Jump }

    // ========== 字段 ==========
    PlayerState currentState;
    float moveSpeed = 5f;
    float jumpForce = 8f;
    bool isGrounded = true;
    Vector3 velocity;

    // ========== 生命周期 ==========
    void Start()
    {
        ChangeState(PlayerState.Idle);  // 初始状态
    }

    void Update()
    {
        // 根据当前状态调用对应的 Update 方法
        switch (currentState)
        {
            case PlayerState.Idle:   Update_Idle();   break;
            case PlayerState.Walk:   Update_Walk();   break;
            case PlayerState.Jump:   Update_Jump();   break;
        }
    }

    // ========== 切换状态 ==========
    void ChangeState(PlayerState next)
    {
        // 退出当前状态
        switch (currentState)
        {
            case PlayerState.Idle:   Exit_Idle();   break;
            case PlayerState.Walk:   Exit_Walk();   break;
            case PlayerState.Jump:   Exit_Jump();   break;
        }

        currentState = next;  // 切到新状态

        // 进入新状态
        switch (next)
        {
            case PlayerState.Idle:   Enter_Idle();   break;
            case PlayerState.Walk:   Enter_Walk();   break;
            case PlayerState.Jump:   Enter_Jump();   break;
        }
    }

    // ========== Idle 状态 ==========
    void Enter_Idle()  { /* 播放 Idle 动画等 */ }
    void Update_Idle()
    {
        float h = Input.GetAxisRaw("Horizontal");
        float v = Input.GetAxisRaw("Vertical");
        if (Mathf.Abs(h) + Mathf.Abs(v) > 0.1f)
            ChangeState(PlayerState.Walk);          // 有输入 → 走

        if (Input.GetKeyDown(KeyCode.Space) && isGrounded)
            ChangeState(PlayerState.Jump);          // 空格 → 跳
    }
    void Exit_Idle()   { }

    // ========== Walk 状态 ==========
    void Enter_Walk()  { /* 播放 Walk 动画 */ }
    void Update_Walk()
    {
        float h = Input.GetAxisRaw("Horizontal");
        float v = Input.GetAxisRaw("Vertical");
        Vector3 dir = new Vector3(h, 0, v).normalized;
        transform.position += dir * moveSpeed * Time.deltaTime;

        if (dir.magnitude < 0.1f)
            ChangeState(PlayerState.Idle);          // 没输入 → 待机

        if (Input.GetKeyDown(KeyCode.Space) && isGrounded)
            ChangeState(PlayerState.Jump);          // 空格 → 跳
    }
    void Exit_Walk()   { }

    // ========== Jump 状态 ==========
    void Enter_Jump()
    {
        velocity.y = jumpForce;
        isGrounded = false;
    }
    void Update_Jump()
    {
        velocity.y += Physics.gravity.y * Time.deltaTime;
        transform.position += velocity * Time.deltaTime;

        if (transform.position.y <= 0f)
        {
            transform.position = new Vector3(transform.position.x, 0, transform.position.z);
            velocity.y = 0;
            isGrounded = true;
            ChangeState(PlayerState.Idle);          // 落地 → 待机
        }
    }
    void Exit_Jump()   { }
}
```

### 语法拆解

##### `switch (currentState) { case ...: break; }` 是什么？

```csharp
switch (currentState)
{
    case PlayerState.Idle: Update_Idle(); break;
    case PlayerState.Walk: Update_Walk(); break;
    case PlayerState.Jump: Update_Jump(); break;
}
```

| 部分 | 含义 |
|------|------|
| `switch (变量)` | 根据变量的值走不同分支 |
| `case 值:` | 匹配这个值就执行后面的代码 |
| `break;` | 跳出 switch（必须写，不然会继续往下走） |

**整行人话**：根据当前状态是什么，调用对应的 Update_xxx 方法。

---

##### `ChangeState(PlayerState.Walk)` 切换流程

```
ChangeState 做了三件事：
    1. 调用 Exit_当前状态()   → 清理工作
    2. currentState = next    → 改状态变量
    3. 调用 Enter_新状态()    → 初始化工作
```

**为什么要分 Enter / Update / Exit？**
- **Enter**：进入时做一次的事（播放动画、设初速度）
- **Update**：每帧持续做的事（移动、检测输入）
- **Exit**：离开时做一次的事（重置变量、停特效）

---

### 优缺点

| 优点 | 缺点 |
|------|------|
| 比纯 if-else 清晰，每个状态逻辑独立 | 状态多了 switch 还是很长 |
| 不用接口/抽象类，简单直接 | 加新状态要改 3 处 switch（Enter/Update/Exit） |
| 适合 3~8 个状态 | 状态类不能复用（每个角色一套代码） |

## 二、实现 2：State Pattern（状态模式）— 标准 OOP 实现

### 思路

- 定义 `IState` 接口，所有状态都实现它（Enter / Update / Exit）
- 每个状态是一个 **独立的类**
- `StateMachine` 类负责管理当前状态和切换
- 状态类持有 Context（上下文）引用，通过 Context 访问共享数据

### 2.1 核心接口与状态机

```csharp
// ===== 状态接口 =====
public interface IState
{
    void Enter();       // 进入状态
    void Update();      // 每帧更新
    void Exit();        // 退出状态
}

// ===== 状态机 =====
public class StateMachine
{
    IState currentState;

    // 切换状态
    public void ChangeState(IState newState)
    {
        currentState?.Exit();       // 退出旧状态（?. = 空就不调用）
        currentState = newState;    // 改当前状态
        currentState.Enter();       // 进入新状态
    }

    // 每帧更新
    public void Update()
    {
        currentState?.Update();
    }
}
```

### 逐词读懂 ChangeState

```csharp
public void ChangeState(IState newState)
{
    currentState?.Exit();
    currentState = newState;
    currentState.Enter();
}
```

| 部分 | 含义 |
|------|------|
| `IState newState` | 要切到的新状态（任何实现了 IState 的类的实例） |
| `currentState?.Exit()` | 如果当前状态不为 null，调用它的 Exit |
| `currentState = newState` | 把当前状态换成新的 |
| `currentState.Enter()` | 调用新状态的 Enter |

**整行人话**：跟旧状态说再见，换上新状态，跟新状态说你好。

---

### 2.2 Context（上下文）

状态之间共享的数据（比如角色引用、速度、是否着地）放在 **Context** 里，状态类通过 Context 访问。

```csharp
// ===== 上下文：状态之间共享的数据和引用 =====
public class PlayerContext
{
    public Transform transform;
    public float moveSpeed = 5f;
    public float jumpForce = 8f;
    public bool isGrounded = true;
    public Vector3 velocity;

    // 可以在这里持有状态机，方便状态内部切状态
    public StateMachine stateMachine;
}
```

> **为什么需要 Context？**  
> 状态类是独立的，不知道角色有什么字段。通过 Context 这个「中间人」访问数据，状态类就可以复用。

---

### 2.3 具体状态类

```csharp
// ===== Idle 状态 =====
public class IdleState : IState
{
    PlayerContext ctx;

    public IdleState(PlayerContext context)
    {
        ctx = context;  // 构造时传入上下文
    }

    public void Enter()
    {
        // 进入 Idle：播放 Idle 动画等
    }

    public void Update()
    {
        float h = Input.GetAxisRaw("Horizontal");
        float v = Input.GetAxisRaw("Vertical");

        if (Mathf.Abs(h) + Mathf.Abs(v) > 0.1f)
            ctx.stateMachine.ChangeState(new WalkState(ctx));   // 切到走

        if (Input.GetKeyDown(KeyCode.Space) && ctx.isGrounded)
            ctx.stateMachine.ChangeState(new JumpState(ctx));   // 切到跳
    }

    public void Exit() { }
}

// ===== Walk 状态 =====
public class WalkState : IState
{
    PlayerContext ctx;

    public WalkState(PlayerContext context) { ctx = context; }

    public void Enter() { }

    public void Update()
    {
        float h = Input.GetAxisRaw("Horizontal");
        float v = Input.GetAxisRaw("Vertical");
        Vector3 dir = new Vector3(h, 0, v).normalized;
        ctx.transform.position += dir * ctx.moveSpeed * Time.deltaTime;

        if (dir.magnitude < 0.1f)
            ctx.stateMachine.ChangeState(new IdleState(ctx));

        if (Input.GetKeyDown(KeyCode.Space) && ctx.isGrounded)
            ctx.stateMachine.ChangeState(new JumpState(ctx));
    }

    public void Exit() { }
}

// ===== Jump 状态 =====
public class JumpState : IState
{
    PlayerContext ctx;

    public JumpState(PlayerContext context) { ctx = context; }

    public void Enter()
    {
        ctx.velocity.y = ctx.jumpForce;
        ctx.isGrounded = false;
    }

    public void Update()
    {
        ctx.velocity.y += Physics.gravity.y * Time.deltaTime;
        ctx.transform.position += ctx.velocity * Time.deltaTime;

        if (ctx.transform.position.y <= 0f)
        {
            ctx.transform.position = new Vector3(ctx.transform.position.x, 0, ctx.transform.position.z);
            ctx.velocity.y = 0;
            ctx.isGrounded = true;
            ctx.stateMachine.ChangeState(new IdleState(ctx));
        }
    }

    public void Exit() { }
}
```

### 语法拆解

##### `currentState?.Exit()` 空条件运算符

```csharp
currentState?.Exit();
```

| 部分 | 含义 |
|------|------|
| `?.` | 空条件运算符：左边为 null 就什么都不做，也不报错 |
| 等价于 | `if (currentState != null) currentState.Exit();` |

**整行人话**：有当前状态就让它退出，没有就算了（第一次进入时还没有旧状态）。

##### `new WalkState(ctx)` — 状态类实例化

```csharp
ctx.stateMachine.ChangeState(new WalkState(ctx));
```

| 部分 | 含义 |
|------|------|
| `new WalkState(ctx)` | 创建一个 WalkState 的实例，把上下文传进去 |
| 传给 ChangeState | 状态机把它设为当前状态 |

### 案例 2：AI 巡逻状态机（枚举版）

**功能**：敌人有 Patrol（巡逻） / Chase（追击） / Attack（攻击）三个状态。

```csharp
using UnityEngine;

public class AIFSM_Enum : MonoBehaviour
{
    enum AIState { Patrol, Chase, Attack }
    AIState currentState;

    public Transform player;           // 玩家引用
    public float patrolSpeed = 2f;
    public float chaseSpeed = 4f;
    public float chaseRange = 8f;      // 追击范围
    public float attackRange = 2f;     // 攻击范围
    public float patrolWaitTime = 2f;  // 巡逻等待时间

    Vector3 patrolTarget;
    float waitTimer;
    float patrolRange = 10f;

    void Start()
    {
        ChangeState(AIState.Patrol);
    }

    void Update()
    {
        switch (currentState)
        {
            case AIState.Patrol: Update_Patrol(); break;
            case AIState.Chase:  Update_Chase();  break;
            case AIState.Attack: Update_Attack(); break;
        }
    }

    void ChangeState(AIState next)
    {
        // 简单版：没有 Enter/Exit，直接切
        currentState = next;
        Debug.Log("AI 进入状态：" + next);
    }

    // ===== Patrol 巡逻 =====
    void Update_Patrol()
    {
        float dist = Vector3.Distance(transform.position, player.position);
        if (dist < chaseRange)
        {
            ChangeState(AIState.Chase);        // 玩家靠近 → 追击
            return;
        }

        // 巡逻：走向随机点，到了等一下再找新点
        if (Vector3.Distance(transform.position, patrolTarget) < 0.5f)
        {
            waitTimer += Time.deltaTime;
            if (waitTimer >= patrolWaitTime)
            {
                patrolTarget = Random.insideUnitSphere * patrolRange;
                patrolTarget.y = 0;
                waitTimer = 0;
            }
        }
        else
        {
            Vector3 dir = (patrolTarget - transform.position).normalized;
            transform.position += dir * patrolSpeed * Time.deltaTime;
            transform.LookAt(patrolTarget);
        }
    }

    // ===== Chase 追击 =====
    void Update_Chase()
    {
        float dist = Vector3.Distance(transform.position, player.position);

        if (dist < attackRange)
        {
            ChangeState(AIState.Attack);       // 够近了 → 攻击
            return;
        }
        if (dist > chaseRange * 1.5f)
        {
            ChangeState(AIState.Patrol);       // 太远了 → 回去巡逻
            return;
        }

        Vector3 dir = (player.position - transform.position).normalized;
        transform.position += dir * chaseSpeed * Time.deltaTime;
        transform.LookAt(player);
    }

    // ===== Attack 攻击 =====
    void Update_Attack()
    {
        float dist = Vector3.Distance(transform.position, player.position);

        if (dist > attackRange)
        {
            ChangeState(AIState.Chase);        // 拉开了 → 继续追
            return;
        }

        // 面向玩家 + 攻击冷却计时等
        transform.LookAt(player);
        Debug.Log("攻击玩家！");
    }
}
```

#### 语法拆解

##### `Random.insideUnitSphere * patrolRange` 随机巡逻点

```csharp
patrolTarget = Random.insideUnitSphere * patrolRange;
patrolTarget.y = 0;
```

| 部分 | 含义 |
|------|------|
| `Random.insideUnitSphere` | 单位球内的随机点（半径 1） |
| `* patrolRange` | 放大到巡逻半径 |
| `.y = 0` | 限定在平面上（不要上下飞） |

**整行人话**：在以原点为中心、巡逻半径为半径的圆内，随机找一个点作为巡逻目标。

## 一、状态单例化优化

### 问题

每次 `ChangeState(new WalkState(ctx))` 都会 new 一个状态对象，产生 GC 分配。

### 解决方案

状态类本身 **不存状态数据**（数据都在 Context 里），所以状态实例可以 **复用**，不用每次 new。

```csharp
// ===== 优化后的 Context：持有所有状态实例 =====
public class PlayerContext
{
    public Transform transform;
    public float moveSpeed = 5f;
    // ... 其他数据

    // 所有状态只创建一次
    public readonly IState IdleState;
    public readonly IState WalkState;
    public readonly IState JumpState;

    public StateMachine stateMachine;

    public PlayerContext()
    {
        IdleState = new IdleState(this);
        WalkState = new WalkState(this);
        JumpState = new JumpState(this);
    }
}

// 切换时直接传已有实例：
ctx.stateMachine.ChangeState(ctx.IdleState);
```

**好处**：零 GC 分配，状态切换完全不产生垃圾。

## 二、泛型状态机封装

把 State Machine 做成泛型，让不同类型的 Context 都能用：

```csharp
// ===== 泛型状态接口 =====
public interface IState<T> where T : class
{
    void Enter(T context);
    void Update(T context);
    void Exit(T context);
}

// ===== 泛型状态机 =====
public class StateMachine<T> where T : class
{
    IState<T> currentState;
    T context;

    public StateMachine(T ctx) { context = ctx; }

    public void ChangeState(IState<T> newState)
    {
        currentState?.Exit(context);
        currentState = newState;
        currentState.Enter(context);
    }

    public void Update()
    {
        currentState?.Update(context);
    }
}
```

**用法**：

```csharp
// 玩家状态类
public class IdleState : IState<PlayerContext>
{
    public void Enter(PlayerContext ctx) { }
    public void Update(PlayerContext ctx) { }
    public void Exit(PlayerContext ctx) { }
}

// 初始化
var fsm = new StateMachine<PlayerContext>(playerContext);
fsm.ChangeState(playerContext.IdleState);
```

**好处**：一套状态机代码，玩家、敌人、UI、游戏流程都能复用。

## 三、层级状态机 HFSM

### 3.1 问题：状态爆炸

如果角色有很多动作，状态会越来越多：
```
地上：Idle / Walk / Run / Jump / Attack1 / Attack2 / Attack3 / Hit / ...
空中：Fall / JumpUp / AirAttack / ...
```
每个状态都要连一堆 Transition，乱。

### 3.2 思路：分组 + 嵌套

把相关的状态分组，放到一个 **父状态** 里，父状态内部又是一个小状态机。

```
Grounded（地上，父状态）
    ├── Idle（子状态）
    ├── Walk（子状态）
    └── Run（子状态）

Air（空中，父状态）
    ├── JumpUp（子状态）
    ├── Fall（子状态）
    └── AirAttack（子状态）
```

- 父状态有 Enter/Update/Exit，Update 时驱动**子状态机**
- 从 Grounded 切到 Air，只需要一条 Transition（着地判定）
- Grounded 内部 Idle↔Walk↔Run 的切换，不影响上层

### 3.3 实现思路

```csharp
// ===== 层级状态基类：自己就是一个状态机 =====
public class HierarchicalState<T> : IState<T>
{
    protected StateMachine<T> subFSM;   // 子状态机
    protected IState<T> defaultSubState;

    public virtual void Enter(T context)
    {
        subFSM.ChangeState(defaultSubState);   // 进入父状态时，子状态从默认开始
    }

    public virtual void Update(T context)
    {
        subFSM.Update();   // 驱动子状态机
    }

    public virtual void Exit(T context)
    {
        // 退出父状态时，也要退出当前子状态
    }
}

// ===== 具体父状态 =====
public class GroundedState : HierarchicalState<PlayerContext>
{
    public GroundedState(PlayerContext ctx)
    {
        subFSM = new StateMachine<PlayerContext>(ctx);
        // 子状态：Idle / Walk / Run
        // defaultSubState = ctx.IdleState;
    }
}
```

**适用场景**：角色动作很多、有明显的地上/空中分组；AI 有 Global / Combat / NonCombat 大分类。

## 四、栈式状态机（Pushdown FSM）

### 4.1 问题：暂停后回来

普通 FSM 切到新状态，旧状态就 Exit 没了。  
但有些场景是「临时切出去，办完事儿回来接着刚才的」：
- 游戏中打开暂停菜单 → 关了菜单回到游戏
- 角色对话中被攻击 → 打完回来继续对话
- 开宝箱时被敌人打断 → 打完回去继续开

### 4.2 思路：用栈保存历史

```
栈底 → 栈顶
[Gameplay]              ← 正常玩
[Gameplay, Pause]       ← 按 Esc，Push 暂停，游戏暂停更新
[Gameplay]              ← 再按 Esc，Pop 暂停，游戏继续（还是之前的状态）
```

- **Push**：把当前状态压入栈顶，新状态 Enter
- **Pop**：弹出栈顶，回到上一个状态（不需要重新 Enter 吗？要看设计）
- 栈顶的状态是当前活跃状态

### 4.3 实现

```csharp
public class PushdownFSM<T> where T : class
{
    Stack<IState<T>> stateStack = new Stack<IState<T>>();
    T context;

    public PushdownFSM(T ctx) { context = ctx; }

    public void Push(IState<T> newState)
    {
        // 当前状态不 Exit，只是暂停
        stateStack.Push(newState);
        newState.Enter(context);
    }

    public void Pop()
    {
        if (stateStack.Count <= 1) return;

        var top = stateStack.Pop();
        top.Exit(context);

        // 回到上一个状态（可以调个 Resume 方法）
        var prev = stateStack.Peek();
        // prev.Resume(context);
    }

    public void Update()
    {
        if (stateStack.Count > 0)
            stateStack.Peek().Update(context);
    }
}
```

**典型用途**：UI 导航（返回上一个面板）、游戏暂停、对话打断。

## 五、Unity Animator 就是 FSM

把 Animator Controller 和 FSM 概念对应起来：

| FSM 概念 | Animator 对应 |
|----------|---------------|
| State 状态 | Animator State（橙色方块） |
| Transition 转移 | 箭头 + Conditions |
| Condition 条件 | Parameters 参数（Float/Bool/Int/Trigger） |
| Enter | State Machine Behaviour 的 OnStateEnter |
| Update | OnStateUpdate |
| Exit | OnStateExit |
| HFSM | Sub-State Machine（子状态机） |
| Layer | 多层状态机叠加 |

**一句话**：Animator Controller 是 **可视化的、专门用于动画的 FSM**。

> **什么时候用代码 FSM，什么时候用 Animator？**  
> - 动画逻辑 → 用 Animator（可视化好调，美术也能改）  
> - 游戏逻辑 / AI / UI / 流程 → 用代码 FSM（灵活可控）  
> - 两者配合：代码 FSM 管角色行为，代码里 Set 参数驱动 Animator

## 六、选型对比

| 方案 | 复杂度 | 适用场景 | 代表工具/实现 |
|------|--------|----------|---------------|
| **if-else / switch** | ⭐ | 2~3 个状态的超简单逻辑 | 直接写 |
| **枚举 FSM** | ⭐⭐ | 3~8 个状态，中小项目 | enum + switch |
| **State Pattern** | ⭐⭐⭐ | 状态多、需要扩展、需要复用 | 接口/抽象类 + 每个状态一个类 |
| **HFSM 层级状态机** | ⭐⭐⭐⭐ | 状态很多、可分组（地上/空中） | 嵌套状态机 |
| **Pushdown FSM** | ⭐⭐⭐ | 需要「暂停-恢复」语义 | 栈 + 状态 |
| **行为树 BT** | ⭐⭐⭐⭐⭐ | 复杂 AI 决策、层级 + 优先级 | Behaviour Designer / 自己写 |
| **决策树** | ⭐⭐⭐ | 条件判断为主的决策 | 树结构 |

**选择原则**：
- 状态少 → 怎么简单怎么来
- 状态 5~15 个 → State Pattern
- 状态 > 15 个且能分组 → HFSM
- AI 行为很复杂 → 行为树

## 七、常见坑与最佳实践

| 坑 | 正确做法 |
|----|----------|
| 状态类里存数据 | 数据放 Context，状态类尽量无状态 |
| 状态之间直接 new 对方 | 通过 Context + 单例状态引用切换 |
| 切换状态没调 Exit | ChangeState 里一定先 Exit 再 Enter |
| Update 里写一堆转移判断 | 把转移条件封装成方法或 Transition 表 |
| 状态太多类爆炸 | 考虑 HFSM 分组，或行为树 |
| 同一帧切多次状态 | 加标志位 / 队列化切换 |
| 忘记处理初始状态 | Start 里一定要 ChangeState 初始状态 |
| GC 太多 | 状态单例化，不要每次切换都 new |
