## 【全文总述】

**怪物 AI** 是游戏里让敌人「看起来像活的」的核心技术：怪物会巡逻、会发现玩家、会追过来、会攻击、死了会倒下。背后通常是 **NavMesh 导航（自动寻路）** + **状态机（行为切换）** + **感知系统（发现玩家）** 三件套。  
NavMesh 是 Unity 内置的导航系统：你在场景里烘焙一张「可行走区域网格」，给怪物加个 NavMeshAgent 组件，告诉它「去那个位置」，它就会自动找路绕开障碍物走过去。  
本文从 NavMesh 烘焙到巡逻、追击、攻击，再到仇恨系统和感知系统，逐层展开。

## 一、定义 — NavMesh 是什么？

| 项目 | 说明 |
|------|------|
| **全称** | Navigation Mesh（导航网格） |
| **类型** | Unity 官方内置的 AI 导航系统 |
| **组成** | NavMesh（可行走网格）+ NavMeshAgent（寻路组件）+ NavMeshObstacle（动态障碍） |
| **一句话** | 告诉怪物「去那里」，它自己找路绕着墙走过去 |

```csharp
// 怪物追玩家，一句话就够了
agent.SetDestination(player.position);
```

---

## 二、本质 — 怪物怎么知道走哪条路？

### 2.1 烘焙（Bake）— 先画好「能走的地方」

```
编辑器里：
  把地板设为 Walkable（能走）
  把墙、柱子设为 Not Walkable（不能走）
  点 Bake（烘焙）
  → 生成一张蓝色的网格图 = 哪些地方能走
```

**烘焙 = 提前算好可行走区域**，运行时怪物就在这张网格上找路。

### 2.2 运行时寻路 — A* 算法

```
怪物在 A 点，玩家在 B 点
  │
  ▼
Unity 用 A* 算法在 NavMesh 上找一条最短路径
  │
  ▼
怪物沿着路径走过去，路上有动态障碍物自动绕开
```

**本质理解**：NavMesh = 「地图」，NavMeshAgent = 「拿着地图找路的人」，SetDestination = 「告诉他目的地」。

### 2.3 与 CharacterController 的关系

| 对比 | CharacterController | NavMeshAgent |
|------|---------------------|--------------|
| **谁来控制方向** | 你写脚本传方向 | 它自己算路径 |
| **碰撞检测** | 胶囊体扫描碰撞 | NavMesh 网格 + RVO 避让 |
| **适用** | 玩家精确控制、平台跳跃 | 怪物/NPC 自动寻路 |
| **能配合用** | ✅ 可以同时有，但注意谁管移动 | — |

> 常见组合：怪物用 NavMeshAgent 自动寻路 + 控制动画；玩家用 CharacterController 手动控制。

---

## 三、怪物 AI 三件套

| 组件 | 作用 | 类比 |
|------|------|------|
| **导航（NavMesh）** | 「怎么去」— 找路、走路 | 腿脚 |
| **决策（状态机）** | 「做什么」— 巡逻/追击/攻击 | 大脑 |
| **感知（检测）** | 「知道什么」— 看没看到玩家 | 眼睛耳朵 |

```
眼睛（感知） → 发现玩家 → 告诉大脑（状态机）
                        ↓
                  大脑说「追！」
                        ↓
                  腿脚（NavMesh） → 跑过去
                        ↓
                  到了攻击距离 → 大脑说「打！」
```

---

## 四、核心一课：如何读懂 SetDestination

```csharp
agent.SetDestination(player.position);
```

| 部分 | 含义 |
|------|------|
| `agent` | 变量名：NavMeshAgent 组件引用 |
| `.SetDestination` | 方法名：设置目的地，开始寻路 |
| `player.position` | 参数：目标点坐标（Vector3） |
| 返回值 | `bool`：成功设置目标返回 true，目标不在 NavMesh 上返回 false |

**整行人话**：告诉导航组件「去玩家那个位置」，它就开始自己找路走过去。

### 注意：目标必须在 NavMesh 上

```csharp
// 目标点在墙上 → 找不到路 → 返回 false
bool success = agent.SetDestination(target);
if (!success)
{
    Debug.Log("目标不在可行走区域！");
}
```

## 一、核心组件与方法详解（逐词读懂）

### 1.1 NavMeshAgent 组件

| 属性 | 类型 | 含义 | 常用值 |
|------|------|------|--------|
| `speed` | float | 移动速度（米/秒） | 3~5 |
| `angularSpeed` | float | 转身速度（度/秒） | 120 |
| `acceleration` | float | 加速度 | 8 |
| `stoppingDistance` | float | 离目标多远就停下 | 1.5（攻击距离） |
| `remainingDistance` | float | 到目标还剩多少距离（只读） | — |
| `hasPath` | bool | 当前有没有路径（只读） | — |
| `isStopped` | bool | 是否停下（true 就不动了） | — |
| `destination` | Vector3 | 当前目标点 | — |
| `radius` | float | Agent 半径（决定能不能挤进窄缝） | 0.5 |
| `height` | float | Agent 高度 | 2.0 |

---

### 1.2 SetDestination — 设置目的地

```csharp
agent.SetDestination(targetPosition);
```

| 部分 | 含义 |
|------|------|
| 参数 | `Vector3 target`：目标世界坐标 |
| 返回值 | `bool`：true=成功设置；false=目标不在可行走区域 |
| 作用 | 告诉 Agent「去这里」，自动计算路径并开始移动 |

**整行人话**：下达命令「去那个位置」，之后 Agent 就自己找路走了。

---

### 1.3 Stop / Resume — 停下和继续

```csharp
agent.isStopped = true;   // 停下（攻击时用）
agent.isStopped = false;  // 继续走
```

> 也可以用 `agent.Stop()` 和 `agent.Resume()`，效果一样。`isStopped` 属性更常用。

---

### 1.4 remainingDistance — 还剩多远

```csharp
if (agent.remainingDistance <= agent.stoppingDistance)
{
    // 到地方了，可以攻击了
}
```

| 部分 | 含义 |
|------|------|
| `remainingDistance` | 到目的地还剩多少距离（只读） |
| `stoppingDistance` | 离目标多远就自动停下 |

**整行人话**：判断怪物是不是已经走到攻击范围内了。

---

### 1.5 Warp — 瞬移

```csharp
agent.Warp(spawnPoint);
```

| 部分 | 含义 |
|------|------|
| `Warp(position)` | 把 Agent 直接瞬移到某个位置（目标位置必须在 NavMesh 上） |

**整行人话**：怪物复活/传送时用，直接把 Agent 放到新位置，重新挂到 NavMesh 上。

## 二、NavMesh 烘焙步骤

```
1. 打开 Navigation 窗口：Window → AI → Navigation
2. 选中场景里的地板、地形（能走的静态物体）
   → Inspector 右上角 Static 打勾 → Navigation Static
3. 选中墙、柱子、房子（障碍物）
   → Static 打勾 → Navigation Static
   → Navigation 窗口 Object 标签 → Navigation Area 选 Not Walkable
4. 切到 Bake 标签：
   - Agent Radius：怪物半径（能通过多窄的门）
   - Agent Height：怪物高度（能从多矮的地方过）
   - Max Slope：最大爬坡角度
   - Step Height：能迈多高的台阶
5. 点 Bake 按钮
6. 场景里出现蓝色/浅蓝色网格 = 烘焙好了
```

> **注意**：烘焙的是**静态物体**。运行时移动的障碍物要用 `NavMeshObstacle` 组件。

### 案例 1：巡逻 + 追击 + 攻击 三状态怪物

**功能**：怪物平时在几个巡逻点之间走；玩家靠近就追；追到攻击距离就停下来攻击；玩家跑远了就回去巡逻。

```csharp
using UnityEngine;
using UnityEngine.AI;                                          // 必须引用 AI 命名空间

// 怪物 AI：巡逻 → 追击 → 攻击
public class EnemyAI : MonoBehaviour
{
    // --- 状态枚举 ---
    enum State { Patrol, Chase, Attack }

    // --- 参数 ---
    [Header("巡逻")]
    [SerializeField] private Transform[] _patrolPoints;        // 巡逻点数组
    [SerializeField] private float _patrolSpeed = 2f;
    [SerializeField] private float _waitTime = 2f;              // 到巡逻点等多久

    [Header("追击")]
    [SerializeField] private float _chaseSpeed = 4f;
    [SerializeField] private float _detectRange = 10f;          // 发现玩家的距离

    [Header("攻击")]
    [SerializeField] private float _attackRange = 2f;           // 攻击距离
    [SerializeField] private float _attackCooldown = 1.5f;      // 攻击冷却
    [SerializeField] private int _damage = 10;

    // --- 引用 ---
    [Header("引用")]
    [SerializeField] private Transform _player;                 // 玩家引用
    private NavMeshAgent _agent;
    private Animator _anim;

    // --- 状态 ---
    private State _currentState = State.Patrol;
    private int _currentPatrolIndex;
    private float _waitTimer;
    private float _attackTimer;

    void Awake()
    {
        _agent = GetComponent<NavMeshAgent>();
        _anim = GetComponent<Animator>();

        // 自动找玩家（简单版，正式项目用 GameManager.Player）
        if (_player == null)
            _player = GameObject.FindGameObjectWithTag("Player").transform;
    }

    void Start()
    {
        GoToNextPatrolPoint();                                   // 开局先去第一个巡逻点
    }

    void Update()
    {
        float distToPlayer = Vector3.Distance(transform.position, _player.position);

        // ===== 状态切换 =====
        switch (_currentState)
        {
            case State.Patrol:
                UpdatePatrol(distToPlayer);
                break;
            case State.Chase:
                UpdateChase(distToPlayer);
                break;
            case State.Attack:
                UpdateAttack(distToPlayer);
                break;
        }

        // 同步动画参数
        _anim.SetFloat("Speed", _agent.velocity.magnitude);
    }

    // ===== 巡逻状态 =====
    void UpdatePatrol(float distToPlayer)
    {
        // 条件：玩家进入发现范围 → 切追击
        if (distToPlayer < _detectRange)
        {
            ChangeState(State.Chase);
            return;
        }

        // 到了巡逻点 → 等待
        if (_agent.remainingDistance <= _agent.stoppingDistance)
        {
            _waitTimer += Time.deltaTime;
            if (_waitTimer >= _waitTime)
            {
                _waitTimer = 0;
                GoToNextPatrolPoint();
            }
        }
    }

    void GoToNextPatrolPoint()
    {
        if (_patrolPoints.Length == 0) return;

        _agent.speed = _patrolSpeed;
        _agent.SetDestination(_patrolPoints[_currentPatrolIndex].position);

        // 下一个巡逻点（循环）
        _currentPatrolIndex = (_currentPatrolIndex + 1) % _patrolPoints.Length;
    }

    // ===== 追击状态 =====
    void UpdateChase(float distToPlayer)
    {
        // 条件 1：玩家跑远了 → 回巡逻
        if (distToPlayer > _detectRange * 1.5f)                  // 脱战距离比发现距离远一点（迟滞）
        {
            ChangeState(State.Patrol);
            return;
        }

        // 条件 2：到攻击距离了 → 切攻击
        if (distToPlayer <= _attackRange)
        {
            ChangeState(State.Attack);
            return;
        }

        // 持续追击：更新目标点为玩家位置
        _agent.speed = _chaseSpeed;
        _agent.SetDestination(_player.position);
    }

    // ===== 攻击状态 =====
    void UpdateAttack(float distToPlayer)
    {
        // 攻击时面对玩家
        Vector3 lookDir = _player.position - transform.position;
        lookDir.y = 0;
        transform.rotation = Quaternion.Lerp(
            transform.rotation,
            Quaternion.LookRotation(lookDir),
            Time.deltaTime * 5f);

        // 条件：玩家跑出攻击范围 → 切追击
        if (distToPlayer > _attackRange * 1.2f)                   // 迟滞，避免边打边切
        {
            ChangeState(State.Chase);
            return;
        }

        // 停下
        _agent.isStopped = true;

        // 攻击冷却
        _attackTimer -= Time.deltaTime;
        if (_attackTimer <= 0)
        {
            Attack();
            _attackTimer = _attackCooldown;
        }
    }

    void Attack()
    {
        _anim.SetTrigger("Attack");
        // 实际伤害可以在动画事件里打
        Debug.Log("怪物攻击了玩家，造成 " + _damage + " 点伤害");
    }

    // ===== 状态切换 =====
    void ChangeState(State newState)
    {
        // 退出旧状态
        switch (_currentState)
        {
            case State.Attack:
                _agent.isStopped = false;                          // 离开攻击状态时恢复移动
                break;
        }

        _currentState = newState;
        Debug.Log("怪物切到状态：" + newState);

        // 进入新状态
        switch (newState)
        {
            case State.Patrol:
                GoToNextPatrolPoint();
                break;
            case State.Chase:
                _agent.speed = _chaseSpeed;
                break;
        }
    }

    // Scene 视图里画检测范围
    void OnDrawGizmosSelected()
    {
        Gizmos.color = Color.yellow;
        Gizmos.DrawWireSphere(transform.position, _detectRange);  // 发现范围

        Gizmos.color = Color.red;
        Gizmos.DrawWireSphere(transform.position, _attackRange);  // 攻击范围
    }
}
```

##### `using UnityEngine.AI;` 是什么？

##### `enum State { Patrol, Chase, Attack }` 是什么？

```csharp
enum State { Patrol, Chase, Attack }
```

| 部分 | 含义 |
|------|------|
| `enum` | 枚举：一组有名字的整数常量 |
| `State` | 枚举类型名 |
| 三个值 | Patrol=0, Chase=1, Attack=2 |

**整行人话**：给三种状态起名字，比直接用 0、1、2 更清楚。

---

##### `(_currentPatrolIndex + 1) % _patrolPoints.Length` 是什么？

```csharp
_currentPatrolIndex = (_currentPatrolIndex + 1) % _patrolPoints.Length;
```

| 部分 | 含义 |
|------|------|
| `_currentPatrolIndex + 1` | 取下一个 |
| `% _patrolPoints.Length` | 对巡逻点总数取余 → 到最后一个就回到第一个 |
| 例：3 个点，索引 2 + 1 = 3，3 % 3 = 0 | → 回到第 0 个，循环巡逻 |

**整行人话**：巡逻点循环，走到最后一个就回到第一个。

---

##### 什么是「迟滞」（Hysteresis）？

```csharp
// 发现距离 10m，脱战距离 15m
if (dist < _detectRange) → 切追击
if (dist > _detectRange * 1.5f) → 回巡逻
```

**为什么要迟滞**：如果发现和脱战都是同一个距离，玩家刚好站在 10m 位置时，怪物会「追一下、停一下、追一下、停一下」来回切——俗称「抽风」。  
脱战距离设远一点，就不会在边界来回抖。

### 案例 2：NavMeshObstacle 动态障碍物

```csharp
using UnityEngine;
using UnityEngine.AI;

// 可破坏的障碍物：HP 归零就消失
public class DestructibleObstacle : MonoBehaviour
{
    [SerializeField] private int _hp = 50;
    private NavMeshObstacle _obstacle;

    void Awake()
    {
        _obstacle = GetComponent<NavMeshObstacle>();
    }

        _hp -= damage;
        if (_hp <= 0)
        {

        }
    }
}
```

| 与静态障碍物区别 | 静态烘焙时算进去；动态的运行时才挡 |

## 一、感知系统 — 视野检测（扇形视野）

距离检测太简单——怪物「背后也能看到玩家」不真实。更真实的是**扇形视野**：玩家在视野角度 + 距离内才发现。

```csharp
// 扇形视野检测
bool CanSeePlayer(Transform player, float viewDistance, float viewAngle)
{
    Vector3 dirToPlayer = player.position - transform.position;
    float dist = dirToPlayer.magnitude;

    // 1. 距离检测：太远看不到
    if (dist > viewDistance) return false;

    // 2. 角度检测：不在视野角度内看不到
    float angle = Vector3.Angle(transform.forward, dirToPlayer);
    if (angle > viewAngle / 2) return false;

    // 3. 射线检测：中间有墙挡着也看不到
    if (Physics.Raycast(transform.position + Vector3.up, dirToPlayer.normalized, dist))
    {
        // 如果打中了别的东西（不是玩家）→ 被挡住了
        // 简化版：可以用 LayerMask 精确过滤
        return false;
    }

    return true;
}
```

| 检测层级 | 作用 | 性能开销 |
|----------|------|----------|
| 距离检测 | 粗筛，距离内才往下查 | 最低（算个距离平方） |
| 角度检测 | 中筛，在扇形内才往下 | 低（算个角度） |
| 射线检测 | 精筛，没被挡住才算看到 | 中（物理射线） |

> **优化技巧**：先距离、再角度、最后射线——从便宜到贵，越早淘汰越省性能。

## 二、仇恨系统（Aggro）

简单 AI 是「谁近打谁」，但 RPG / MMO 里怪物有**仇恨列表**：打它伤害越高、越嘲讽它，仇恨越高，它就追谁。

```csharp
// 仇恨条目
public class AggroEntry
{
    public GameObject Target;
    public float Hate;  // 仇恨值
}

// 仇恨系统
List<AggroEntry> _aggroList = new List<AggroEntry>();

// 增加仇恨
public void AddHate(GameObject target, float amount)
{
    // 找有没有这个目标
    var entry = _aggroList.Find(e => e.Target == target);
    if (entry != null)
    {
        entry.Hate += amount;
    }
    else
    {
        _aggroList.Add(new AggroEntry { Target = target, Hate = amount });
    }
}

// 获取仇恨最高的目标
public GameObject GetTopHateTarget()
{
    if (_aggroList.Count == 0) return null;

    // 按仇恨值从高到低排序
    _aggroList.Sort((a, b) => b.Hate.CompareTo(a.Hate));
    return _aggroList[0].Target;
}
```

**常见仇恨规则**：
- 伤害 = 1 倍仇恨
- 治疗 = 0.5 倍仇恨
- 嘲讽技能 = 大量仇恨 + 强制攻击几秒
- 脱战 = 仇恨慢慢衰减

## 三、大量怪物的性能优化

| 优化点 | 做法 |
|--------|------|
| **组更新** | 不是每个怪物自己 Update，而是一个管理器统一更新 20 个，下一帧更新下 20 个 |
| **禁用远的** | 离玩家太远的怪物，减少 Update 频率甚至停掉 |
| **共享路径** | 大量怪物去同一个地方，只算一次路径大家共享 |
| **LOD** | 远的怪物简化动画、减少逻辑 |
| **对象池** | 怪物死亡不 Destroy，回收到池里复用 |
| **避障质量** | 普通怪物避障质量调低，精英怪调高 |

```csharp
// 组更新示例：每 N 帧更新一次
int _updateFrameInterval = 3;   // 每 3 帧更新一次
int _frameOffset;               // 错开，别所有怪物同一帧更新

void Update()
{
    if ((Time.frameCount + _frameOffset) % _updateFrameInterval != 0) return;
    // ... AI 逻辑 ...
}
```

## 四、NavMesh + 动画集成

| NavMeshAgent 属性 | 传给 Animator 的参数 | 用途 |
|-------------------|---------------------|------|
| `agent.velocity.magnitude` | `Speed` (float) | 走/跑/待机混合树 |
| `agent.desiredVelocity`（归一化） | `MoveX` / `MoveZ` | 定向混合树（前后左右走） |
| 是否在攻击 | `IsAttacking` (bool) | 攻击状态 |
| 剩余距离 < stoppingDistance | `IsGrounded` 等 | — |

```csharp
// 在 Update 里同步
Vector3 localVel = transform.InverseTransformDirection(_agent.velocity);
_anim.SetFloat("Forward", localVel.z);
_anim.SetFloat("Turn", localVel.x);
```

> 用 `InverseTransformDirection` 把世界速度转成「角色自身坐标系」的前后左右——这样 Animator 才知道是往前走还是往左走。

### Root Motion 配合

如果用 Root Motion（动画驱动位移），可以让 NavMeshAgent 只负责算路径和方向，位移交给动画：

```csharp
// Agent 只算路径，不实际移动位置
agent.updatePosition = false;
agent.updateRotation = false;

// 在 OnAnimatorMove 里用 root motion 的位置
void OnAnimatorMove()
{
    // 让 agent 在动画根运动的位置上
    agent.nextPosition = transform.position;
}
```

## 五、常见误区与最佳实践

| 误区 | 正确做法 |
|------|----------|
| 没烘焙 NavMesh 就用 Agent | 先烘焙，确保蓝色网格覆盖了行走区域 |
| 目标点不在 NavMesh 上 | 用 NavMesh.SamplePosition 找最近的可行走点 |
| 攻击后忘了 isStopped=false | 在 ChangeState 退出攻击时恢复 |
| 每帧都 SetDestination | 目标没变就不用反复设，或有距离阈值再更 |
| 巡逻点设到墙里 | 确保所有目标点都在蓝色网格内 |
| 大量怪物每帧都 SetDestination | 节流：每 0.2 秒更新一次目标就够了 |
| 怪物卡住不动 | 检查 remainingDistance + hasPath + pathStatus，加 stuck 检测 |
| 门开了怪物还绕路 | 用 NavMeshObstacle + carve（挖洞）或动态烘焙 |
| 检测用 Vector3.Distance | 用 sqrMagnitude 比较（省开方），或先粗后精 |
