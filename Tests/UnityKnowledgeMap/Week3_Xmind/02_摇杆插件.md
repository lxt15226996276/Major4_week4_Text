## 【全文总述】

**虚拟摇杆（Joystick）** 是手游里最常见的输入方式：屏幕上一个圆盘，手指按住拖动，角色就朝那个方向走。本质是一个 UI 控件 + 拖拽事件处理：算出手指相对圆心的偏移，输出一个 Vector2 方向值给角色移动脚本。  
和键盘 Input.GetAxis 不同：摇杆是「触摸/鼠标拖拽」驱动，每帧输出一个 -1~1 的二维向量，完全可以替代 WASD。  
本文从 UI 结构到拖拽事件，再到死区、灵敏度、动态摇杆等进阶玩法，逐层展开。

# 第一阶段：零基础篇

## 【阶段总述】

零基础阶段建立**正确认知**：虚拟摇杆是什么、由什么 UI 组成、为什么能控制角色移动。  
同时学会**读懂** `OnDrag` 里的核心代码——坐标转换与方向计算。

---

## 一、定义 — 虚拟摇杆是什么？

| 项目 | 说明 |
|------|------|
| **类型** | UI 控件 + C# 脚本组合（不是 Unity 官方组件，需自己实现） |
| **组成** | 一个大圆盘（Background）+ 一个小圆点（Handle） |
| **作用** | 手指拖动小圆点 → 输出一个 Vector2 方向 → 控制角色移动 |
| **一句话** | 屏幕上的「虚拟方向盘」，替代键盘 WASD |

```csharp
// 摇杆对外提供的核心数据
Vector2 moveDir = Joystick.Instance.Direction;
// x = 左右（-1 左 ~ 1 右）
// y = 前后（-1 后 ~ 1 前）
```

---

## 二、本质 — 摇杆怎么知道往哪拖？

### 2.1 工作流程

```
手指按下屏幕 ─→ 记录位置
    │
手指拖动中 ─→ 算「手指相对中心偏移了多少」
    │         → 偏移比例就是方向（-1 ~ 1）
    │         → 让小圆点（Handle）跟着动
    │
手指抬起 ─→ 方向归零，小圆点回中心
```

### 2.2 与键盘输入的对应关系

| 键盘 | 摇杆 Direction |
|------|----------------|
| 按 D（右） | `(1, 0)` |
| 按 W（前） | `(0, 1)` |
| 按 A + S | `(-1, -1)` |
| 都不按 | `(0, 0)` |

**本质理解**：摇杆输出的 Vector2 和 `Input.GetAxis("Horizontal")` + `Input.GetAxis("Vertical")` 格式完全一样，所以角色移动脚本几乎不用改，把输入源换一下就行。

---

## 三、UI 结构（零基础必认）

```
Canvas
└── Joystick_BG（大圆盘，Image）
    └── Handle（小圆点，Image，在圆盘中心）
```

| 物体 | 作用 | 组件 |
|------|------|------|
| **Canvas** | UI 的根，所有 UI 都在下面 | Canvas + Canvas Scaler + Graphic Raycaster |
| **EventSystem** | 处理 UI 事件（点击、拖拽），场景必须有一个 | EventSystem + Standalone Input Module |
| **Background** | 摇杆背景圆盘，限制活动范围 | Image + 摇杆脚本 |
| **Handle** | 摇杆手柄小圆点，手指拖动的视觉反馈 | Image |

> **小白注意**：场景里必须有 **EventSystem**，否则 UI 不会响应点击和拖拽。创建 UI 时 Unity 会自动建，但是别删。

---

## 四、核心一课：如何读懂 OnDrag 里的坐标转换

```csharp
RectTransformUtility.ScreenPointToLocalPointInRectangle(
    _background,
    eventData.position,
    eventData.pressEventCamera,
    out Vector2 localPos);
```

| 部分 | 含义 |
|------|------|
| `RectTransformUtility` | Unity 工具类：专门做 RectTransform 坐标转换 |
| `ScreenPointToLocalPointInRectangle` | 方法名：把「屏幕像素点」转成「某个 UI 矩形里的局部坐标」 |
| `_background` | 参数1：参照物——摇杆背景的 RectTransform |
| `eventData.position` | 参数2：手指的屏幕位置（像素坐标） |
| `eventData.pressEventCamera` | 参数3：UI 相机（Screen Space - Overlay 时是 null） |
| `out Vector2 localPos` | 输出参数：转换后的局部坐标（相对背景中心的偏移） |

**整行人话**：问「手指这个点，在摇杆圆盘的局部坐标系里，相对中心偏移了多少？」——算出来的 localPos 就是手柄相对于圆盘中心的位置。

### 为什么要转坐标？

```
手指位置是「屏幕像素坐标」（如 1920×1080 屏幕上的某个点）
我们需要的是「相对摇杆中心的偏移」（以圆盘中心为原点）
所以要用 ScreenPointToLocalPointInRectangle 做转换
```

## 一、核心接口详解（逐词读懂）

### 1.1 IPointerDownHandler — 按下时

```csharp
public void OnPointerDown(PointerEventData eventData)
{
    // 手指按下那一瞬间触发一次
}
```

| 部分 | 含义 |
|------|------|
| `IPointerDownHandler` | 接口：实现它就代表「我能响应按下事件」 |
| `OnPointerDown` | 接口方法：按下时 Unity 自动调用 |
| `PointerEventData eventData` | 参数：包含按下位置、按下的是哪个手指等信息 |

**整行人话**：告诉 Unity「这个 UI 能被点」，手指按上去的瞬间触发一次。

---

### 1.2 IDragHandler — 拖拽中

```csharp
public void OnDrag(PointerEventData eventData)
{
    // 按住拖动时，每帧触发一次
}
```

| 部分 | 含义 |
|------|------|
| `IDragHandler` | 接口：实现它就代表「我能被拖拽」 |
| `OnDrag` | 接口方法：拖拽中每帧自动调用 |
| `eventData.position` | 当前拖拽位置（屏幕像素） |

**整行人话**：手指按住拖动的过程中，每帧都调用一次——摇杆的核心计算就在这里面。

---

### 1.3 IPointerUpHandler — 抬起时

```csharp
public void OnPointerUp(PointerEventData eventData)
{
    // 手指抬起那一瞬间触发一次
}
```

| 部分 | 含义 |
|------|------|
| `IPointerUpHandler` | 接口：抬起事件 |
| `OnPointerUp` | 抬起时自动调用 |

**整行人话**：手指松开的瞬间，把方向归零，手柄回到中心。

---

### 1.4 三个接口的执行顺序

```
OnPointerDown（按下）
    ↓
OnDrag（拖拽中，每帧多次）
    ↓
OnPointerUp（抬起）
```

> **注意**：要让拖拽工作正常，**三个接口通常一起实现**——只写 IDragHandler 不写 IPointerDownHandler，有些情况下拖拽可能不生效。

---

## 二、方向计算完整步骤

```
步骤 1：屏幕点 → UI 局部坐标（相对背景中心）
        ScreenPointToLocalPointInRectangle
            ↓
步骤 2：除以背景的半宽半高 → 得到比例（-1 ~ 1）
        position.x / background.sizeDelta.x * 2
            ↓
步骤 3：判断死区，小于死区算 0
        if (dir.magnitude < deadZone) dir = Vector2.zero
            ↓
步骤 4：超过 1 就归一化（防止斜向速度更快）
        if (dir.magnitude > 1) dir = dir.normalized
            ↓
步骤 5：Handle 位置同步
        handle.anchoredPosition = dir * background.sizeDelta / 2
```

### 为什么要归一化？

```
向右拖到底 → (1, 0) → 长度 1
向右上 45° 拖到底 → (1, 1) → 长度 √2 ≈ 1.414
```

如果不归一化，斜着拖会比正方向「值更大」，导致角色斜着走更快。归一化后最大长度永远是 1。

### 案例 1：固定摇杆（最常用）

**功能**：左下角一个固定位置的摇杆，手指拖动输出 Direction，角色读取 Direction 移动。

```csharp
using UnityEngine;
using UnityEngine.EventSystems;                                  // UI 事件命名空间，必须引用

// 固定虚拟摇杆：挂在 Background 上
public class Joystick : MonoBehaviour, IPointerDownHandler, IDragHandler, IPointerUpHandler
{
    // --- UI 引用 ---
    [Header("UI 引用")]
    [SerializeField] private RectTransform _background;         // 背景圆盘
    [SerializeField] private RectTransform _handle;             // 手柄小圆点

    // --- 参数 ---
    [Header("参数")]
    [SerializeField] [Range(0, 1)] private float _deadZone = 0.1f;  // 死区（0~1）
    [SerializeField] [Range(0.5f, 2f)] private float _handleRange = 1f; // 手柄移动范围

    // --- 对外输出 ---
    public Vector2 Direction { get; private set; }              // 方向向量，x/y 都是 -1 ~ 1

    // --- 单例（方便其他脚本访问）---
    public static Joystick Instance { get; private set; }

    void Awake()
    {
        Instance = this;

        // 如果没在 Inspector 拖，自动从子物体找
        if (_background == null) _background = GetComponent<RectTransform>();
        if (_handle == null && transform.childCount > 0)
            _handle = transform.GetChild(0).GetComponent<RectTransform>();
    }

    // 按下：触发一次
    public void OnPointerDown(PointerEventData eventData)
    {
        OnDrag(eventData);   // 按下就算一次拖拽，立刻响应
    }

    // 拖拽中：每帧触发
    public void OnDrag(PointerEventData eventData)
    {
        // 1. 屏幕坐标 → 背景局部坐标（相对背景中心）
        RectTransformUtility.ScreenPointToLocalPointInRectangle(
            _background,
            eventData.position,
            eventData.pressEventCamera,
            out Vector2 localPos);

        // 2. 转成比例（除以背景半宽半高 → -1 ~ 1）
        Vector2 dir = new Vector2(
            localPos.x / (_background.sizeDelta.x / 2),
            localPos.y / (_background.sizeDelta.y / 2));

        // 3. 死区处理：偏移太小算 0（防误触）
        if (dir.magnitude < _deadZone)
        {
            Direction = Vector2.zero;
        }
        else
        {
            // 4. 超出范围就归一化（最大长度 = 1）
            if (dir.magnitude > 1f)
                dir = dir.normalized;

            Direction = dir;
        }

        // 5. 更新 Handle 位置（跟着手指走）
        _handle.anchoredPosition = Direction * (_background.sizeDelta / 2) * _handleRange;
    }

    // 抬起：触发一次
    public void OnPointerUp(PointerEventData eventData)
    {
        Direction = Vector2.zero;                          // 方向归零
        _handle.anchoredPosition = Vector2.zero;            // 手柄回到中心
    }
}
```

#### 语法拆解

##### `: MonoBehaviour, IPointerDownHandler, IDragHandler, IPointerUpHandler` 是什么？

```csharp
public class Joystick : MonoBehaviour, IPointerDownHandler, IDragHandler, IPointerUpHandler
```

| 部分 | 含义 |
|------|------|
| `: MonoBehaviour` | 继承 MonoBehaviour，能挂到 GameObject 上 |
| `IPointerDownHandler` 等 | 实现接口：告诉 Unity 这个脚本可以响应这些 UI 事件 |
| 逗号分隔 | 一个类可以继承一个基类 + 实现多个接口 |

**整行人话**：这个脚本既能挂到物体上，又能响应「按下/拖拽/抬起」三种 UI 事件。

---

##### `eventData.pressEventCamera` 是什么？

```csharp
eventData.pressEventCamera
```

| 部分 | 含义 |
|------|------|
| `pressEventCamera` | 触发这次按下事件的相机 |
| Canvas 是 Screen Space - Overlay 时 | 这个值是 null（因为 UI 直接叠在屏幕上，不用相机） |
| Canvas 是 World Space / Screen Space - Camera 时 | 是对应的 UI 相机 |

**整行人话**：直接传进去就行，不用管它是什么——RectTransformUtility 内部会处理。

---

##### `Vector2.magnitude` 是什么？

```csharp
dir.magnitude
```

| 部分 | 含义 |
|------|------|
| `magnitude` | 向量的「长度」（从原点到这个点的距离） |
| Vector2(1, 0).magnitude | = 1 |
| Vector2(1, 1).magnitude | = √2 ≈ 1.414 |

**整行人话**：向量有多长——用来判断有没有超出圆盘范围。

### 案例 2：角色移动对接摇杆

**功能**：把原来用 Input.GetAxis 的角色控制器，改成读取摇杆 Direction。

```csharp
using UnityEngine;

// 角色移动：用摇杆 Direction 替代键盘输入
public class PlayerJoystickMove : MonoBehaviour
{
    [SerializeField] private float _speed = 5f;
    [SerializeField] private float _jumpForce = 8f;
    [SerializeField] private float _gravity = 20f;

    private CharacterController _controller;
    private Vector3 _velocity;

    void Awake()
    {
        _controller = GetComponent<CharacterController>();
    }

    void Update()
    {
        // ===== 输入：从摇杆读，替代 Input.GetAxis =====
        Vector2 joyDir = Joystick.Instance != null ? Joystick.Instance.Direction : Vector2.zero;

        float h = joyDir.x;    // 原来：Input.GetAxis("Horizontal")
        float v = joyDir.y;    // 原来：Input.GetAxis("Vertical")

        Vector3 moveDir = transform.right * h + transform.forward * v;

        // 重力 + 跳跃
        if (_controller.isGrounded)
        {
            if (_velocity.y < 0) _velocity.y = -2f;

            // 跳跃用按钮（手机上可以单独做个跳跃按钮）
            if (Input.GetKeyDown(KeyCode.Space))
                _velocity.y = _jumpForce;
        }
        else
        {
            _velocity.y -= _gravity * Time.deltaTime;
        }

        Vector3 finalMove = moveDir.normalized * _speed + Vector3.up * _velocity.y;
        _controller.Move(finalMove * Time.deltaTime);
    }
}
```

#### 关键点

- **替换输入源**：`Input.GetAxis("Horizontal")` → `Joystick.Instance.Direction.x`
- **完全兼容**：因为都是 -1~1 的值，其他逻辑完全不用改
- **空值保护**：`Joystick.Instance != null ? ... : Vector2.zero`，防止摇杆没创建时报错

### 案例 3：带灵敏度的摇杆

**功能**：增加灵敏度参数，手指拖动更少距离就能达到满值——适合小屏幕。

```csharp
[SerializeField] [Range(0.5f, 3f)] private float _sensitivity = 1f;

public void OnDrag(PointerEventData eventData)
{
    // ... 坐标转换 ...

    Vector2 dir = new Vector2(
        localPos.x / (_background.sizeDelta.x / 2) * _sensitivity,   // 乘以灵敏度
        localPos.y / (_background.sizeDelta.y / 2) * _sensitivity);

    if (dir.magnitude > 1f) dir = dir.normalized;
    Direction = dir;

    // Handle 还是按背景范围移动（不受灵敏度影响视觉）
    _handle.anchoredPosition = Direction * (_background.sizeDelta / 2);
}
```

| 灵敏度 | 效果 |
|--------|------|
| 0.5 | 迟钝，要拖很远才满值 |
| 1.0 | 正常，拖到边缘满值 |
| 2.0 | 灵敏，拖一半就满值 |

## 一、动态摇杆（Dynamic Joystick）

固定摇杆永远在左下角；**动态摇杆**是手指按哪，摇杆中心就在哪——更自由，手游 FPS 常用。

```csharp
public class DynamicJoystick : MonoBehaviour, IPointerDownHandler, IDragHandler, IPointerUpHandler
{
    [SerializeField] private RectTransform _bgRect;
    [SerializeField] private RectTransform _handle;
    [SerializeField] private float _radius = 100f;   // 摇杆活动半径（像素）

    public Vector2 Direction { get; private set; }

    Canvas _canvas;

    void Awake()
    {
        _canvas = GetComponentInParent<Canvas>();
        _bgRect.gameObject.SetActive(false);   // 默认隐藏，按下才显示
    }

    // 按下：摇杆出现在按下位置
    public void OnPointerDown(PointerEventData eventData)
    {
        _bgRect.gameObject.SetActive(true);

        // 把背景移到按下位置
        RectTransformUtility.ScreenPointToLocalPointInRectangle(
            GetComponent<RectTransform>(),
            eventData.position,
            eventData.pressEventCamera,
            out Vector2 localPos);

        _bgRect.anchoredPosition = localPos;
        OnDrag(eventData);
    }

    public void OnDrag(PointerEventData eventData)
    {
        RectTransformUtility.ScreenPointToLocalPointInRectangle(
            _bgRect,
            eventData.position,
            eventData.pressEventCamera,
            out Vector2 localPos);

        Vector2 dir = localPos / _radius;
        if (dir.magnitude > 1f) dir = dir.normalized;
        Direction = dir;

        _handle.anchoredPosition = dir * _radius;
    }

    public void OnPointerUp(PointerEventData eventData)
    {
        Direction = Vector2.zero;
        _handle.anchoredPosition = Vector2.zero;
        _bgRect.gameObject.SetActive(false);  // 抬起隐藏
    }
}
```

**用法**：动态脚本挂在一个全屏的透明 Image 上（作为触摸区域），按下时 Background 出现在手指位置。

---

## 二、事件驱动模式（比单例更灵活）

单例是「其他脚本主动来读 Direction」；**事件驱动**是「摇杆值变了就通知所有订阅者」——更解耦。

```csharp
// 用 UnityEvent 或 C# 事件
public event System.Action<Vector2> OnValueChanged;

Direction = dir;
OnValueChanged?.Invoke(Direction);   // 值变了就触发事件
```

角色脚本订阅：

```csharp
void Start()
{
    Joystick.Instance.OnValueChanged += HandleJoystick;
}

void OnDestroy()
{
    Joystick.Instance.OnValueChanged -= HandleJoystick;  // 取消订阅，防内存泄漏
}

void HandleJoystick(Vector2 dir)
{
    _moveInput = dir;
}
```

| 模式 | 优点 | 缺点 |
|------|------|------|
| **单例读取** | 简单，每帧读就行 | 紧密耦合，摇杆换了实现就要改 |
| **事件驱动** | 解耦，多个订阅者都能收到 | 要记得取消订阅，初学者容易忘 |

## 三、技能方向摇杆 + 双摇杆

### 技能摇杆

右边一个摇杆，拖向哪个方向就朝哪个方向放技能——MOBA 游戏常见。

```csharp
public class SkillJoystick : MonoBehaviour, IPointerDownHandler, IDragHandler, IPointerUpHandler
{
    public event System.Action<Vector2> OnSkillCast;    // 释放技能事件
    [SerializeField] private float _castThreshold = 0.8f; // 拖到多少程度算释放

    public void OnPointerUp(PointerEventData eventData)
    {
        if (Direction.magnitude >= _castThreshold)
        {
            OnSkillCast?.Invoke(Direction.normalized);   // 抬起时如果拖得够远，触发技能
        }
        // ... 归零 ...
    }
}
```

### 双摇杆

左摇杆移动 + 右摇杆瞄准/视角——FPS 手游标配。

```
左摇杆（移动）  →  角色 CharacterController.Move
右摇杆（视角）  →  相机 transform.Rotate
```

## 四、与 Input System 集成

Unity 新 Input System 支持「虚拟摇杆作为输入设备」，可以和手柄摇杆统一处理：

1. 把摇杆脚本的 Direction 写到 InputAction 里
2. 角色只从 InputAction 读值
3. 键盘/手柄/摇杆都用同一套逻辑

```csharp
// 简化版：手动写值到 InputAction
public void OnDrag(PointerEventData eventData)
{
    // ... 计算 Direction ...
    _moveAction.WriteValue(Direction);
}
```

> **选型建议**：中小型项目自己写摇杆 + 直接用 Direction 就够了；多平台、多输入设备的大项目推荐 Input System。

## 五、常见误区与最佳实践

| 误区 | 正确做法 |
|------|----------|
| 忘了加 EventSystem | 场景必须有 EventSystem，UI 事件才能响应 |
| 死区设为 0 | 留 0.1 左右死区，防手指轻微抖动 |
| Handle 范围 = 背景大小 | 视觉上可以小一点，体验更好 |
| 用 transform.position 算 | 用 anchoredPosition，UI 走 RectTransform |
| Canvas 缩放不对 | 用 ScreenPointToLocalPointInRectangle，自动处理缩放 |
| 动态摇杆触摸区太小 | 做一个全屏透明 UI 作为触摸区 |
| 多点触控冲突 | 每个摇杆用不同的 pointerId 区分手指 |
| 忘了 OnPointerUp 归零 | 抬起必须归零，否则角色一直走 |

| 事件驱动 | OnValueChanged 事件，解耦更灵活 |
