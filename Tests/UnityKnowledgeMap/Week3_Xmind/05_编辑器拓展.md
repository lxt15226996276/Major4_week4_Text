## 【全文总述】

**编辑器扩展**就是给 Unity 编辑器本身加功能：加个菜单项一键生成 Prefab、自定义个 Inspector 面板方便策划填数据、做个关卡编辑窗口可视化摆东西……  
这些功能**只在 Editor 里有用，打包后会被去掉**——所以它们要放在 `Editor` 文件夹里，用 `#if UNITY_EDITOR` 包起来，或者继承 `EditorWindow` / `Editor` 这些只有 Editor 才有的类。  
本文从最常用的 `MenuItem` 菜单，到 `CustomEditor` 自定义 Inspector，再到 `EditorWindow` 自定义窗口、`PropertyDrawer` 自定义属性、Gizmos 场景绘制，逐层展开。

## 一、定义 — 编辑器扩展是什么？

| 项目 | 说明 |
|------|------|
| **是什么** | 给 Unity 编辑器本身加功能的脚本 |
| **在哪用** | **只在 Unity Editor 里有用**，打包后会被剥离 |
| **放哪** | `Editor` 文件夹里（或用 `#if UNITY_EDITOR` 包裹） |
| **一句话** | 自己做 Unity 的「插件」，让开发更顺手 |

```csharp
// 例子：在 Unity 顶部菜单加一个按钮，一键创建空物体
[MenuItem("Tools/Create Empty")]
static void CreateEmpty()
{
    new GameObject("New Empty");
}
```

---

## 二、本质 — 为什么能改编辑器？

### 2.1 Unity 编辑器本身也是用 Unity 做的

Unity 编辑器的很大一部分是用 Unity 自己的 API 写的。官方开放了 `UnityEditor` 命名空间，让你也能调用这些 API 来改编辑器界面和行为。

### 2.2 Editor 文件夹的特殊性

```
Assets/
├── Scripts/           ← 运行时代码（打包进游戏）
└── Editor/            ← 编辑器代码（不进包，只在 Editor 用）
    ├── MyTool.cs
    └── CustomInspector.cs
```

| 放在哪 | 打包后 | 能调用 UnityEditor 吗 |
|--------|--------|----------------------|
| **Editor 文件夹** | ❌ 被剥离 | ✅ 可以 |
| 普通文件夹 | ✅ 进包 | ❌ 打包会报错 |

> **小白必记**：只要你写的代码用了 `using UnityEditor;`，就必须放到 `Editor` 文件夹里，否则打包时报错。

## 三、常用编辑器扩展技术（入门必认）

| 技术 | 用途 | 难度 |
|------|------|------|
| **MenuItem** | 在顶部菜单加个按钮，点一下执行代码 | ⭐ 最简单 |
| **CustomEditor** | 自定义某个 MonoBehaviour 的 Inspector 面板 | ⭐⭐ 常用 |
| **EditorWindow** | 做一个独立的编辑器窗口（比如关卡编辑器） | ⭐⭐⭐ 中等 |
| **PropertyDrawer** | 自定义某个类/字段在 Inspector 里的显示方式 | ⭐⭐⭐ 中等 |
| **Gizmos** | 在 Scene 视图里画线、画圆、画图标 | ⭐⭐ 实用 |

## 四、核心一课：如何读懂 MenuItem

```csharp
[MenuItem("Tools/批量操作/批量重命名")]
static void BatchRename()
{
    // 执行逻辑...
}
```

| 部分 | 含义 |
|------|------|
| `[MenuItem(...)]` | **特性（Attribute）**：写在方法上面，告诉 Unity「这个方法是一个菜单项」 |
| `"Tools/批量操作/批量重命名"` | 菜单路径：在顶部菜单 Tools → 批量操作 → 批量重命名 |
| `static void` | 菜单项方法**必须是静态的** |
| 方法名 | 随便起，不显示在菜单上 |

**整行人话**：在 Unity 顶部菜单栏加一个按钮，用户点它就执行这个方法里的代码。

### 为什么必须是 static？

因为菜单项是「全局」的——不需要实例化某个对象，点菜单就直接调用。

# 第二阶段：入门篇

## 【阶段总述】

入门阶段掌握 **MenuItem 菜单项、CustomEditor 自定义 Inspector、EditorWindow 自定义窗口** 三个最常用的编辑器扩展技术。

## 一、核心技术详解（逐词读懂）

### 1.1 MenuItem — 菜单项

```csharp
[MenuItem("Tools/My Tool %t")]
static void MyTool()
{
    // 点击菜单执行的代码
}
```

| 部分 | 含义 |
|------|------|
| `"Tools/My Tool"` | 菜单路径，用 `/` 分子菜单 |
| `%t` | 快捷键：%=Ctrl / #=Shift / &=Alt / _=普通键；%t = Ctrl+T |
| 方法要求 | 必须 `static` |

**整行人话**：在 Tools 菜单下加个叫 My Tool 的项，按 Ctrl+T 也能触发。

### 常用快捷键符号

| 符号 | 键 | 示例 |
|------|----|------|
| `%` | Ctrl (Win) / Cmd (Mac) | `%t` = Ctrl+T |
| `#` | Shift | `#t` = Shift+T |
| `&` | Alt | `&t` = Alt+T |
| `_` | 普通字符键 | `_t` = T 键 |
| `LEFT` / `RIGHT` / `UP` / `DOWN` | 方向键 | `_LEFT` = 左箭头 |

---

### 1.2 EditorGUILayout.LabelField 等 IMGUI 控件

| 控件 | 作用 |
|------|------|
| `EditorGUILayout.LabelField("标题")` | 显示文字标签 |
| `EditorGUILayout.IntField("数量", ref value)` | 整数输入框 |
| `EditorGUILayout.FloatField("速度", ref value)` | 浮点数输入框 |
| `EditorGUILayout.TextField("名字", ref value)` | 文字输入框 |
| `EditorGUILayout.Toggle("开关", ref value)` | 勾选框 |
| `EditorGUILayout.Slider(ref value, 0, 100)` | 滑条 |
| `EditorGUILayout.ObjectField("物体", obj, typeof(GameObject), true)` | 物体引用拖拽 |
| `EditorGUILayout.Space()` | 空行 |
| `if (GUILayout.Button("按钮")) { }` | 按钮（点了执行 if 里的代码） |
| `EditorGUILayout.BeginHorizontal() / EndHorizontal()` | 水平排列 |
| `EditorGUILayout.BeginVertical() / EndVertical()` | 垂直排列 |

> `EditorGUILayout` = 自动布局（省心，不用算坐标）  
> `EditorGUI` = 手动布局（需要指定 Rect 坐标，更灵活但麻烦）

---

### 1.3 SerializedObject / SerializedProperty — 安全改数据

```csharp
// 在自定义 Editor 里
SerializedObject _so;
SerializedProperty _speedProp;

void OnEnable()
{
    _so = new SerializedObject(target);
    _speedProp = _so.FindProperty("m_Speed");   // 找字段
}

public override void OnInspectorGUI()
{
    _so.Update();                                // 更新序列化对象

    EditorGUILayout.PropertyField(_speedProp);   // 画属性（自动处理 Undo）

    _so.ApplyModifiedProperties();               // 应用修改
}
```

| 部分 | 含义 |
|------|------|
| `SerializedObject` | 序列化对象包装器，安全地操作目标对象的字段 |
| `FindProperty("字段名")` | 找到指定字段的 SerializedProperty |
| `Update()` | 从目标对象同步最新值 |
| `ApplyModifiedProperties()` | 把修改写回目标对象（自动支持 Undo） |
| `PropertyField(prop)` | 自动画这个属性的 Inspector UI |

**整行人话**：不要直接 `myScript.speed = 10;`，要用 SerializedProperty——这样 Unity 的撤销（Ctrl+Z）、多对象编辑、Prefab 覆盖才会正常工作。

### 案例 1：MenuItem 批量重命名工具

**功能**：选中多个物体，点菜单一键按规则批量重命名（如「Enemy_01、Enemy_02…」）。

```csharp
using UnityEngine;
using UnityEditor;                                   // 必须引用

// 编辑器工具：批量重命名选中物体
// 放到 Editor 文件夹里
public static class BatchRenameTool
{
    [MenuItem("Tools/批量重命名选中物体 %#r")]      // 菜单路径 + 快捷键 Ctrl+Shift+R
    static void RenameSelected()
    {
        // 1. 拿到当前选中的物体
        GameObject[] selected = Selection.gameObjects;

        if (selected.Length == 0)
        {
            EditorUtility.DisplayDialog("提示", "请先选中要重命名的物体", "确定");
            return;
        }

        // 2. 弹出输入框，让用户输入前缀
        string prefix = EditorInputDialog.Show("批量重命名", "前缀：", "Item_");
        if (string.IsNullOrEmpty(prefix)) return;

        // 3. 记录 Undo（支持 Ctrl+Z 撤销）
        Undo.RecordObjects(selected as Object[], "批量重命名");

        // 4. 按 Hierarchy 顺序排序（可选）
        System.Array.Sort(selected, (a, b) => a.transform.GetSiblingIndex().CompareTo(b.transform.GetSiblingIndex()));

        // 5. 逐个重命名
        for (int i = 0; i < selected.Length; i++)
        {
            selected[i].name = prefix + (i + 1).ToString("D2");  // D2 = 两位数，01、02...
        }

        Debug.Log($"重命名了 {selected.Length} 个物体");
    }

    // 菜单是否可用（没选中就灰掉）
    [MenuItem("Tools/批量重命名选中物体 %#r", true)]
    static bool RenameSelectedValidate()
    {
        return Selection.gameObjects.Length > 0;
    }
}

// 简单的输入对话框（EditorUtility 没有输入框，自己做一个）
public class EditorInputDialog : EditorWindow
{
    string _input;
    string _message;
    System.Action<string> _onOk;

    public static string Show(string title, string message, string defaultText)
    {
        var window = GetWindow<EditorInputDialog>(title);
        window._message = message;
        window._input = defaultText;
        window._onOk = null;
        window.ShowModal();
        return window._input;
    }

    void OnGUI()
    {
```

### 案例 2：CustomEditor 自定义 Inspector

**功能**：给 PlayerController 脚本做一个自定义 Inspector 面板，比默认的更好看、更方便。

```csharp
using UnityEngine;
using UnityEditor;

// 自定义 PlayerController 的 Inspector
// 放到 Editor 文件夹里
[CustomEditor(typeof(PlayerController))]
public class PlayerControllerEditor : Editor
{
    SerializedProperty _speedProp;
    SerializedProperty _jumpForceProp;
    SerializedProperty _gravityProp;

    // 启用时找字段
    void OnEnable()
    {
        _speedProp = serializedObject.FindProperty("_speed");
        _jumpForceProp = serializedObject.FindProperty("_jumpForce");
        _gravityProp = serializedObject.FindProperty("_gravity");
    }

    // 绘制 Inspector
    public override void OnInspectorGUI()
    {
        serializedObject.Update();                            // 先同步

        // === 标题 ===
        GUILayout.Label("角色移动参数", EditorStyles.boldLabel);
        EditorGUILayout.Space();

        // === 速度滑条 ===
        EditorGUILayout.Slider(_speedProp, 1f, 20f, new GUIContent("移动速度"));

        // === 跳跃力滑条 ===
        EditorGUILayout.Slider(_jumpForceProp, 1f, 30f, new GUIContent("跳跃力"));

        // === 重力滑条 ===
        EditorGUILayout.Slider(_gravityProp, 5f, 50f, new GUIContent("重力"));

        EditorGUILayout.Space();

        // === 提示框 ===
        EditorGUILayout.HelpBox(
            "提示：\n速度 5 左右是普通人走速\n跳跃力 8 + 重力 20 手感适中",
            MessageType.Info);

        // === 按钮：跳到玩家 ===
        if (GUILayout.Button("选中玩家物体"))
        {
            Selection.activeGameObject = (target as PlayerController).gameObject;
            SceneView.FrameLastActiveSceneView();
        }

        serializedObject.ApplyModifiedProperties();             // 最后应用修改
    }
}

// 这是示例目标脚本（Runtime）
// public class PlayerController : MonoBehaviour
// {
//     [SerializeField] private float _speed = 5f;
//     [SerializeField] private float _jumpForce = 8f;
//     [SerializeField] private float _gravity = 20f;
// }
```

#### 关键点

- **`[CustomEditor(typeof(MyScript))]`**：告诉 Unity 这个 Editor 是给哪个脚本画 Inspector 的
- **`serializedObject`**：基类 `Editor` 自带的，就是 target 的 SerializedObject 包装
- **`target`**：当前选中的物体组件（就是你加 CustomEditor 的那个类型）
- **Update → 画 → ApplyModifiedProperties**：三部曲，少了就会出问题

### 案例 3：EditorWindow 自定义窗口

**功能**：做一个独立的「关卡编辑器」窗口，可以快速创建不同类型的敌人。

```csharp
using UnityEngine;
using UnityEditor;

// 自定义编辑器窗口：关卡工具
// 放到 Editor 文件夹里
public class LevelEditorWindow : EditorWindow
{
    string _enemyName = "Enemy";
    int _enemyCount = 1;
    float _spawnRadius = 5f;
    GameObject _enemyPrefab;
    Transform _parent;

    // 打开窗口的菜单项
    [MenuItem("Tools/关卡编辑器")]
    static void OpenWindow()
    {
        var window = GetWindow<LevelEditorWindow>("关卡编辑器");
        window.Show();
    }

    void OnGUI()
    {
        GUILayout.Label("敌人生成工具", EditorStyles.boldLabel);
        EditorGUILayout.Space();

        // 输入字段
        _enemyName = EditorGUILayout.TextField("敌人名字前缀", _enemyName);
        _enemyCount = EditorGUILayout.IntSlider("数量", _enemyCount, 1, 50);
        _spawnRadius = EditorGUILayout.FloatField("生成半径", _spawnRadius);
        _enemyPrefab = EditorGUILayout.ObjectField(
            "敌人 Prefab", _enemyPrefab, typeof(GameObject), false) as GameObject;
        _parent = EditorGUILayout.ObjectField(
            "父物体", _parent, typeof(Transform), true) as Transform;

        EditorGUILayout.Space();
        EditorGUILayout.HelpBox("会在场景原点周围随机生成敌人", MessageType.Info);
        EditorGUILayout.Space();

        // 生成按钮
        using (new EditorGUI.DisabledScope(_enemyPrefab == null))   // 没选 Prefab 就灰掉
        {
            if (GUILayout.Button("生成敌人", GUILayout.Height(40)))
            {
                SpawnEnemies();
            }
        }

        // 清空按钮
        if (GUILayout.Button("清空父物体下所有敌人"))
        {
            if (EditorUtility.DisplayDialog("确认", "确定删除父物体下所有子物体？", "删", "取消"))
            {
                ClearChildren();
            }
        }
    }

    void SpawnEnemies()
    {
        // 记录 Undo 组
        Undo.IncrementCurrentGroup();
        int undoGroup = Undo.GetCurrentGroup();

        for (int i = 0; i < _enemyCount; i++)
        {
            Vector3 pos = Random.insideUnitCircle * _spawnRadius;
            pos.z = pos.y;
            pos.y = 0;

            GameObject go = PrefabUtility.InstantiatePrefab(_enemyPrefab) as GameObject;
            go.name = $"{_enemyName}_{i + 1:D2}";
            go.transform.position = pos;

            if (_parent != null)
            {
                Undo.SetTransformParent(go.transform, _parent, "设父物体");
            }

            Undo.RegisterCreatedObjectUndo(go, "生成敌人");
        }

        Undo.CollapseUndoOperations(undoGroup);
    }

    void ClearChildren()
    {
        if (_parent == null) return;

        var children = new Transform[_parent.childCount];
        for (int i = 0; i < _parent.childCount; i++)
            children[i] = _parent.GetChild(i);

        foreach (var child in children)
        {
            Undo.DestroyObjectImmediate(child.gameObject);
        }
    }
}
```

## 一、PropertyDrawer — 自定义属性绘制

CustomEditor 是改**整个组件**的 Inspector；**PropertyDrawer** 是改**某一个类型/字段**的显示方式。  
比如你有个 `RangeFloat` 类（min + max），默认 Inspector 是折叠的，用 PropertyDrawer 可以把它画成一行两个滑条。

```csharp
// Runtime 类（普通脚本里）
[System.Serializable]
public class MinMaxFloat
{
    public float min;
    public float max;
}

// Editor 里的 PropertyDrawer（放 Editor 文件夹）
[CustomPropertyDrawer(typeof(MinMaxFloat))]
public class MinMaxFloatDrawer : PropertyDrawer
{
    public override void OnGUI(Rect position, SerializedProperty property, GUIContent label)
    {
        EditorGUI.BeginProperty(position, label, property);

        // 画标签
        position = EditorGUI.PrefixLabel(position, GUIUtility.GetControlID(FocusType.Passive), label);

        // 找 min 和 max 属性
        SerializedProperty minProp = property.FindPropertyRelative("min");
        SerializedProperty maxProp = property.FindPropertyRelative("max");

        // 一行分三部分：min输入框 + "~" 文字 + max输入框
        float w = (position.width - 20) / 2f;
        Rect minRect = new Rect(position.x, position.y, w, position.height);
        Rect labelRect = new Rect(position.x + w, position.y, 20, position.height);
        Rect maxRect = new Rect(position.x + w + 20, position.y, w, position.height);

        EditorGUI.PropertyField(minRect, minProp, GUIContent.none);
        EditorGUI.LabelField(labelRect, "~");
        EditorGUI.PropertyField(maxRect, maxProp, GUIContent.none);

        EditorGUI.EndProperty();
    }
}
```

**适用场景**：自定义 struct / class 的显示、大范围复用的属性、让数据类在 Inspector 里更紧凑。

## 二、Gizmos / Handles — Scene 视图绘制

在 Scene 视图里画辅助线、画范围、画图标——比 Log 更直观。

### Gizmos（简单绘制）

```csharp
// 运行时脚本里，不是 Editor 脚本也能用 Gizmos
public class EnemyAI : MonoBehaviour
{
    public float detectRange = 10f;
    public float attackRange = 2f;

    // 只在 Scene 视图显示 Gizmos 时画
    void OnDrawGizmosSelected()
    {
        // 发现范围 - 黄色
        Gizmos.color = Color.yellow;
        Gizmos.DrawWireSphere(transform.position, detectRange);

        // 攻击范围 - 红色
        Gizmos.color = Color.red;
        Gizmos.DrawWireSphere(transform.position, attackRange);

        // 面朝方向
        Gizmos.color = Color.blue;
        Gizmos.DrawRay(transform.position, transform.forward * 5f);
    }
}
```

| Gizmos 方法 | 作用 |
|-------------|------|
| `Gizmos.DrawWireSphere(center, radius)` | 画线框球 |
| `Gizmos.DrawSphere(center, radius)` | 画实心球 |
| `Gizmos.DrawLine(a, b)` | 画线 |
| `Gizmos.DrawRay(origin, direction)` | 画射线 |
| `Gizmos.DrawCube(center, size)` | 画立方体 |
| `Gizmos.color` | 设置颜色 |
| `Gizmos.matrix` | 设置变换矩阵（平移/旋转/缩放） |

| 方法名 | 什么时候画 |
|--------|------------|
| `OnDrawGizmos` | 总是画（只要 Gizmos 开着） |
| `OnDrawGizmosSelected` | 只有选中这个物体时才画 |

### Handles（Editor 用，更强大）

Handles 只能在 Editor 脚本里用，可以画可交互的手柄——比如在 Scene 里拖一个点来调整巡逻路径。

```csharp
// 自定义 Editor 里的 OnSceneGUI
[CustomEditor(typeof(PatrolPath))]
public class PatrolPathEditor : Editor
{
    void OnSceneGUI()
    {
        PatrolPath path = target as PatrolPath;
        if (path.Points == null) return;

        // 画路径线
        Handles.color = Color.cyan;
        for (int i = 0; i < path.Points.Length; i++)
        {
            int next = (i + 1) % path.Points.Length;
            Handles.DrawLine(path.Points[i], path.Points[next]);
        }

        // 画可拖动的点（用户可以在 Scene 里直接拖）
        Handles.color = Color.yellow;
        for (int i = 0; i < path.Points.Length; i++)
        {
            EditorGUI.BeginChangeCheck();
            Vector3 newPos = Handles.PositionHandle(path.Points[i], Quaternion.identity);
            if (EditorGUI.EndChangeCheck())
            {
                Undo.RecordObject(path, "移动巡逻点");
                path.Points[i] = newPos;
                EditorUtility.SetDirty(path);
            }
        }
    }
}
```

## 三、ScriptableObject + Editor 数据配置器

大量配置数据（武器表、道具表、关卡表）通常用 ScriptableObject 存，再加个自定义 Editor 让策划填表更方便——这是编辑器扩展最常用的组合之一。

```csharp
// Runtime：武器数据
[CreateAssetMenu(fileName = "NewWeapon", menuName = "配置/武器")]
public class WeaponData : ScriptableObject
{
    public string weaponName;
    public int damage;
    public float fireRate;
    public Sprite icon;
    public GameObject bulletPrefab;
}

// Editor：自定义 Editor
[CustomEditor(typeof(WeaponData))]
public class WeaponDataEditor : Editor
{
    public override void OnInspectorGUI()
    {
        WeaponData weapon = target as WeaponData;

        base.OnInspectorGUI();   // 先画默认的

        EditorGUILayout.Space();
        GUILayout.Label("预览", EditorStyles.boldLabel);

        // 预览图标
        if (weapon.icon != null)
        {
            Texture2D tex = AssetPreview.GetAssetPreview(weapon.icon);
            if (tex != null)
                GUILayout.Label(tex, GUILayout.Width(100), GUILayout.Height(100));
        }

        // 一键复制
        if (GUILayout.Button("复制这个武器配置"))
        {
            WeaponData copy = ScriptableObject.CreateInstance<WeaponData>();
            EditorUtility.CopySerialized(weapon, copy);
            string path = AssetDatabase.GetAssetPath(weapon);
            string newPath = AssetDatabase.GenerateUniqueAssetPath(path.Replace(".asset", "_copy.asset"));
            AssetDatabase.CreateAsset(copy, newPath);
            AssetDatabase.SaveAssets();
            Selection.activeObject = copy;
        }
    }
}
```

## 四、IMGUI vs UI Toolkit 选型

| 对比 | IMGUI（传统） | UI Toolkit（新） |
|------|---------------|-----------------|
| **写法** | 代码驱动，OnGUI 里每帧画 | UXML + USS，类似 HTML/CSS |
| **上手难度** | 低（代码直接写） | 中（要学新格式） |
| **适用** | 简单工具、快速原型、自定义 Inspector | 复杂工具、列表/树视图、可复用 UI |
| **官方定位** | 仍支持，长期维护 | 未来方向，新功能优先更 |
| **性能** | 一般 | 更好 |

**新手建议**：先学 IMGUI，足够应付 90% 的编辑器工具需求。等 IMGUI 用熟了、做复杂工具时再学 UI Toolkit。

## 五、常见误区与最佳实践

| 误区 | 正确做法 |
|------|----------|
| 不放 Editor 文件夹 | 用了 UnityEditor 必须放 Editor 文件夹或用 `#if UNITY_EDITOR` 包 |
| 直接改 target 字段，不用 SerializedProperty | 用 SerializedProperty，支持 Undo 和多对象编辑 |
| 忘了加 Undo | 所有修改操作都要支持 Ctrl+Z |
| 每帧 OnGUI 做复杂计算 | OnGUI 每帧调很多次，重逻辑缓存起来 |
| Prefab 用 Instantiate | Editor 里用 PrefabUtility.InstantiatePrefab |
| 用 Resources 找资源 | 编辑器里用 AssetDatabase.FindAssets / LoadAssetAtPath |
| 不处理多对象编辑 | 用 SerializedObject 和 PropertyField 天然支持 |
| Editor 代码引用 Runtime 没问题 | Runtime 不能引用 Editor，反过来可以 |
| 直接改 scene 里的东西忘了 SetDirty | 修改后 EditorUtility.SetDirty，确保保存 |
