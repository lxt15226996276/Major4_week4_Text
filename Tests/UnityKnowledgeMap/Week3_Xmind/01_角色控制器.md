## 【全文总述】

**CharacterController（角色控制器）** 是 Unity 专门为「人形角色移动」设计的组件：既有碰撞体、又不受物理引擎力的影响，专门用来做「角色走路、跑步、跳跃」这种可控运动。  
和 Rigidbody 不同：CharacterController 不会被外力推走、不会自动倒下，你给它一个移动向量，它就沿那个方向移动，同时自动处理墙壁、台阶、斜坡的碰撞。  
本文从组件属性到 Move / SimpleMove 方法，再到跳跃、重力、动画集成，逐层展开。

# 第一阶段：零基础篇

## 【阶段总述】

零基础阶段建立**正确认知**：CharacterController 是什么、为什么不用 Rigidbody、它和普通碰撞体有什么区别。  
同时学会**读懂** `controller.Move(moveDir * Time.deltaTime)` 每个部分的含义，并认识 Inspector 核心参数。

---

## 一、定义 — CharacterController 是什么？

| 项目 | 说明 |
|------|------|
| **类型** | `UnityEngine` 命名空间下的 **组件（Component）** |
| **挂载** | 添加在 GameObject 上（角色物体） |
| **官方定义** | A CharacterController allows you to easily do movement constrained by collisions |
| **一句话** | 专门给「人形角色」用的「移动 + 碰撞二合一组件」 |

```csharp
// 获取角色控制器组件
CharacterController controller = GetComponent<CharacterController>();

// 让角色移动
controller.Move(Vector3.forward * Time.deltaTime);
```

---

## 二、本质 — 为什么有了 Rigidbody 还要 CharacterController？

### 2.1 与 Rigidbody 的本质区别

| 对比项 | Rigidbody（刚体） | CharacterController（角色控制器） |
|--------|-------------------|-----------------------------------|
| **驱动力** | 物理引擎（力、速度、碰撞反弹） | 你写的脚本（直接传位移） |
| **受外力** | 会被推飞、会倒下、会弹跳 | 完全不受力，你让它怎么动就怎么动 |
| **碰撞体形状** | 任意（Box/Sphere/Capsule/Mesh） | 只能是胶囊体 |
| **适用** | 汽车、箱子、弹球、布娃娃 | 玩家角色、NPC、人形怪物 |
| **控制难度** | 较难（物理惯性不好调） | 简单（精确到帧） |

### 2.2 本质理解

```
Rigidbody 模式：
  你给一个力 → 物理引擎算半天 → 物体动起来（可能滑出去停不下来）

CharacterController 模式：
  你给一个位移向量 → 直接移动 → 碰到墙自动停下（精确可控）
```

**一句话本质**：CharacterController = 胶囊碰撞体 + 一个叫 Move 的「瞬移函数」，移动时自动检测碰撞。

---

## 三、官方定位 — 设计来干什么？

Unity 官方把 CharacterController 定位为 **「人形角色移动的专用方案」**。

| 官方意图 | 说明 |
|----------|------|
| ✅ 设计目的 | 玩家角色、人形怪物、FPS/TPS 角色移动 |
| ❌ 非设计目的 | 汽车、弹球、布娃娃、需要真实物理的物体 |
| 官方建议 | 主角移动优先 CharacterController；物理物体用 Rigidbody |

---

## 四、Inspector 核心参数（零基础必认）

| 参数 | 含义 | 常用值 |
|------|------|--------|
| **Radius** | 胶囊体半径（角色胖瘦） | 0.5m 左右 |
| **Height** | 胶囊体总高度 | 2.0m 左右（人高） |
| **Center** | 胶囊中心偏移（上下左右调中心点） | Y=1 时中心在 1 米高度 |
| **Slope Limit** | 能走上的最大斜坡角度（度） | 45° 左右 |
| **Step Offset** | 能迈上的台阶高度（米） | 0.3m 左右（一级台阶高） |
| **Skin Width** | 皮肤厚度（防止卡墙） | 0.08（约 radius 的 10%） |
| **Min Move Distance** | 小于这个距离就不移动（防抖） | 0.001 |

> **类比理解**：  
> Radius = 角色有多「胖」  
> Height = 角色有多「高」  
> Slope Limit = 能爬多陡的坡（45° 以内的坡都能上）  
> Step Offset = 能迈多高的坎（30cm 以下的台阶自动上去）

---

## 五、核心一课：如何读懂一行代码

```csharp
controller.Move(moveDir * Time.deltaTime);
```

```
controller . Move ( moveDir * Time.deltaTime ) ;
    ①       ②       ③             ④           ⑤
```

| 序号 | 部分 | 含义 |
|:----:|------|------|
| ① | `controller` | 变量名：CharacterController 组件引用 |
| ② | `.Move` | 方法名：移动函数，传入「位移向量」 |
| ③ | `moveDir` | 移动方向 + 速度（Vector3，每秒走的距离） |
| ④ | `* Time.deltaTime` | 乘以每帧时间 → 从「每秒速度」变「本帧位移」 |
| ⑤ | `;` | C# 语句结束 |

**整行人话**：让角色按 moveDir 的方向和速度，移动本帧应该走的那一小段距离，碰到墙就停。

### 传位移，不是传速度！

```csharp
// ❌ 错误：Move 接收的是「本帧位移」，不是「每秒速度」
controller.Move(moveDir);   // 不乘 deltaTime → 帧率越高跑得越快

// ✅ 正确：速度 × 时间 = 本帧位移
controller.Move(moveDir * Time.deltaTime);
```

> **小白必记**：  
> `moveDir` 单位是「米 / 秒」（速度）  
> `Time.deltaTime` 单位是「秒 / 帧」（一帧多久）  
> 相乘之后单位变成「米 / 帧」（本帧走多少米）  
> 这样不管帧率多少，每秒走的距离都一样。

## 【阶段总述】

## 一、核心方法详解（逐词读懂）

### 1.1 Move 方法 — 最常用、最灵活

```csharp
controller.Move(Vector3 motion);
```

| 部分 | 含义 |
|------|------|
| `motion` | **Vector3**：本帧要移动的「位移向量」（不是速度！） |
| 返回值 | `CollisionFlags` 枚举：上一次移动都碰了哪（Sides/Above/Below/None） |

**整行人话**：让角色朝 motion 的方向移动这么长一段距离，路上撞到东西就停下。

```csharp
// 示例：向前移动 5 米/秒
Vector3 moveDir = transform.forward * 5f;           // 5 米/秒（速度）
controller.Move(moveDir * Time.deltaTime);          // 本帧位移
```

---

### 1.2 SimpleMove 方法 — 简化版（带重力）

```csharp
controller.SimpleMove(Vector3 speed);
```

| 部分 | 含义 |
|------|------|
| `speed` | **Vector3**：水平速度（米/秒），y 分量会被忽略 |
| 返回值 | `bool`：是否在地面上（isGrounded） |

**整行人话**：SimpleMove 是 Unity 帮你封装好的「简化版移动」，自动加了重力，你只需要传水平速度就行，不用自己算重力。

```csharp
// SimpleMove 示例：传水平速度即可，自动下落
Vector3 moveSpeed = transform.forward * 5f;
controller.SimpleMove(moveSpeed);   // 不用乘 deltaTime，内部处理了
```

| 对比 | Move | SimpleMove |
|------|------|------------|
| 参数 | 位移（位移向量，需乘 deltaTime） | 速度（水平速度，不用乘 deltaTime） |
| 重力 | 自己处理 | 自动处理 |
| 灵活性 | 高（可自定义跳跃、飞行） | 低（只能地面行走） |
| 推荐 | **大多数情况** | 快速原型、简单行走 |

---

### 1.3 isGrounded 属性 — 是否在地面

```csharp
bool onGround = controller.isGrounded;
```

| 部分 | 含义 |
|------|------|
| `isGrounded` | **bool，只读**：上一次调用 Move 时，角色底部是否碰到了东西 |

**整行人话**：告诉我们角色现在是不是站在地上——跳跃前要检查、着地动画要用到。

> **重要**：`isGrounded` 是「上一次 Move 的结果」，所以必须在 **Move 调用之后** 读取才准确。Move 之前读的是上一帧的状态。

---

### 1.4 velocity 属性 — 当前实际速度

```csharp
Vector3 currentSpeed = controller.velocity;
```

| 部分 | 含义 |
|------|------|
| `velocity` | **Vector3，只读**：角色当前实际移动速度（米/秒） |

**整行人话**：角色现在真实在怎么动——用来传给 Animator 做走跑动画很方便。

---

### 1.5 collisionFlags — 碰撞标记

```csharp
CollisionFlags flags = controller.collisionFlags;
```

| 枚举值 | 含义 |
|--------|------|
| `CollisionFlags.None` | 没碰到任何东西 |
| `CollisionFlags.Sides` | 侧面碰到墙 |
| `CollisionFlags.Above` | 头顶碰到东西 |
| `CollisionFlags.Below` | 脚下碰到地面（和 isGrounded=true 等价） |

**整行人话**：上一次 Move 都撞了哪——可以用 `if ((flags & CollisionFlags.Sides) != 0)` 判断是否撞墙。

## 二、标准移动四步 + 重力跳跃

### 2.1 完整移动逻辑流程

```
每帧：
  1. 读输入：Input.GetAxis("Horizontal"/"Vertical") → 水平方向
  2. 转世界方向：transform.right * x + transform.forward * z
  3. 处理重力：
     - 在地面上 → y 速度设为小的负数（保持贴地）
     - 不在地面上 → y 速度 += 重力 * deltaTime（加速下落）
  4. 处理跳跃：按跳跃键 且 在地面 → y 速度 = 跳跃力
  5. 组合最终速度：水平方向 * speed + Vector3.up * yVelocity
  6. 调用 Move(最终速度 * deltaTime)
```

### 2.2 为什么重力要自己写？

CharacterController **不是物理引擎**，它不会自动把你往下拉。如果你不加重力，角色跳起来就会飘在天上。

```csharp
// 最常见的新手 bug：跳起来下不来
if (Input.GetButtonDown("Jump"))
{
    _velocity.y = _jumpForce;   // 跳是加上去了
}
// 但是没有 _velocity.y += _gravity * Time.deltaTime; → 飘在空中
```

**正确做法**：每帧都给 y 方向速度加重力，模拟自由落体。

### 案例 1：基础第三人称移动（带重力跳跃）

**功能**：WASD 控制角色前后左右移动，空格跳跃，带重力下落，碰到墙自动停下。

```csharp
using UnityEngine;                                                  // 引入 Unity 核心库

// 角色控制器基础移动：WASD 走 + 空格跳 + 重力
public class PlayerController : MonoBehaviour
{
    // --- 移动参数 ---
    [SerializeField] private float _speed = 5f;                     // 移动速度（米/秒）
    [SerializeField] private float _jumpForce = 8f;                 // 跳跃初速度
    [SerializeField] private float _gravity = 20f;                  // 重力加速度（下落越来越快）

    // --- 组件引用 ---
    private CharacterController _controller;                        // 角色控制器组件引用
    private Vector3 _velocity;                                      // 当前速度（含 y 方向重力）

    void Awake()
    {
        _controller = GetComponent<CharacterController>();         // 获取本物体上的 CC 组件
    }

    void Update()
    {
        // 1. 读取水平输入（WASD 或方向键）
        float h = Input.GetAxis("Horizontal");                     // A/D 或 ←/→，-1 ~ 1
        float v = Input.GetAxis("Vertical");                       // W/S 或 ↑/↓，-1 ~ 1

        // 2. 把输入转成「角色自身坐标系」的方向
        Vector3 moveDir = transform.right * h + transform.forward * v;

        // 3. 处理重力
        if (_controller.isGrounded)                                 // 站在地上
        {
            if (_velocity.y < 0) _velocity.y = -2f;                // 保持一个小的向下速度，贴紧地面

            // 4. 跳跃（只有在地面上才能跳）
            if (Input.GetButtonDown("Jump"))
            {
                _velocity.y = _jumpForce;                          // 给一个向上的初速度
            }
        }
        else                                                        // 在空中
        {
            _velocity.y -= _gravity * Time.deltaTime;              // 重力加速下落
        }

        // 5. 组合最终移动向量：水平方向 + 垂直重力/跳跃
        Vector3 finalMove = moveDir.normalized * _speed + Vector3.up * _velocity.y;

        // 6. 执行移动（速度 × 时间 = 位移）
        _controller.Move(finalMove * Time.deltaTime);
    }
}
```

##### `[SerializeField] private float _speed = 5f;` 是什么？


|------|------|

##### `transform.right * h + transform.forward * v` 是什么？


| 部分 | 含义 |
|------|------|
| `transform.right` | 角色自身的「右方向」（世界坐标下的 Vector3） |
| `transform.forward` | 角色自身的「前方向」 |

##### `_velocity.y -= _gravity * Time.deltaTime;` 是什么？


| 部分 | 含义 |
|------|------|
| `_velocity.y` | 垂直方向的速度分量 |
| `-=` | 减法赋值：`_velocity.y = _velocity.y - ...` |
| `_gravity * Time.deltaTime` | 重力加速度 × 本帧时间 = 本帧速度增量 |

**整行人话**：每帧把下落速度再加一点，模拟真实的重力加速——越落越快。

### 案例 2：第一人称视角移动 + 鼠标转视角

**功能**：WASD 前后左右走，鼠标控制视角左右转（Yaw）和上下看（Pitch），第一人称常用组合。

```csharp
using UnityEngine;

// 第一人称控制器：WASD 移动 + 鼠标转视角
public class FirstPersonController : MonoBehaviour
{
    // --- 移动参数 ---
    [Header("移动")]
    [SerializeField] private float _walkSpeed = 6f;
    [SerializeField] private float _jumpForce = 7f;
    [SerializeField] private float _gravity = 20f;

    // --- 视角参数 ---
    [Header("视角")]
    [SerializeField] private float _mouseSensitivity = 100f;       // 鼠标灵敏度
    [SerializeField] private Transform _playerCamera;              // 子物体：相机（用来上下看）
    [SerializeField] private float _minPitch = -80f;               // 往下看的最大角度
    [SerializeField] private float _maxPitch = 80f;                // 往上看的最大角度

    // --- 组件与状态 ---
    private CharacterController _controller;
    private Vector3 _velocity;
    private float _yaw;                                             // 左右旋转角度
    private float _pitch;                                           // 上下俯仰角度

    void Awake()
    {
        _controller = GetComponent<CharacterController>();
        Cursor.lockState = CursorLockMode.Locked;                   // 锁定鼠标到屏幕中心（FPS 标配）
    }

    void Update()
    {
        // ===== 视角旋转 =====
        float mouseX = Input.GetAxis("Mouse X") * _mouseSensitivity * Time.deltaTime;
        float mouseY = Input.GetAxis("Mouse Y") * _mouseSensitivity * Time.deltaTime;

        _yaw += mouseX;                                              // 左右：角色整体转
        _pitch -= mouseY;                                             // 上下：相机单独转（注意是减）
        _pitch = Mathf.Clamp(_pitch, _minPitch, _maxPitch);         // 限制上下角度，防止翻过头

        transform.rotation = Quaternion.Euler(0f, _yaw, 0f);        // 角色身体只转 Y 轴（左右）
        _playerCamera.localRotation = Quaternion.Euler(_pitch, 0f, 0f); // 相机转 X 轴（上下）

        // ===== 移动 =====
        float h = Input.GetAxis("Horizontal");
        float v = Input.GetAxis("Vertical");

        Vector3 moveDir = transform.right * h + transform.forward * v;

        // 重力与跳跃
        if (_controller.isGrounded)
        {
            if (_velocity.y < 0) _velocity.y = -2f;
            if (Input.GetButtonDown("Jump")) _velocity.y = _jumpForce;
        }
        else
        {
            _velocity.y -= _gravity * Time.deltaTime;
        }

        Vector3 finalMove = moveDir.normalized * _walkSpeed + Vector3.up * _velocity.y;
        _controller.Move(finalMove * Time.deltaTime);
    }
}
```

### 案例 3：平台跳跃（双跳 + 土狼时间）

**功能**：在基础移动上增加「双跳」和「土狼时间」，让平台跳跃手感更好。  
土狼时间 = 掉下平台后短暂时间内仍能起跳（Coyote Time，经典平台跳跃手感优化）。

```csharp
using UnityEngine;

// 平台跳跃角色控制器：双跳 + 土狼时间
public class PlatformerController : MonoBehaviour
{
    [Header("移动")]
    [SerializeField] private float _moveSpeed = 7f;
    [SerializeField] private float _jumpForce = 12f;

    [Header("重力")]
    [SerializeField] private float _gravity = 25f;
    [SerializeField] private float _fallMultiplier = 1.5f;         // 下落加重力（更紧凑的跳跃手感）

    [Header("跳跃优化")]
    [SerializeField] private int _maxJumps = 2;                     // 最大跳跃次数（2 = 双跳）
    [SerializeField] private float _coyoteTime = 0.1f;              // 土狼时间：掉下去后多久还能跳
    [SerializeField] private float _jumpBuffer = 0.1f;              // 跳跃缓冲：提前按空格也能触发

    private CharacterController _controller;
    private Vector3 _velocity;
    private int _jumpsRemaining;                                    // 剩余跳跃次数
    private float _coyoteTimer;                                     // 土狼时间计时
    private float _jumpBufferTimer;                                 // 跳跃缓冲计时

    void Awake()
    {
        _controller = GetComponent<CharacterController>();
    }

    void Update()
    {
        // ===== 输入 =====
        float h = Input.GetAxis("Horizontal");

        // ===== 跳跃缓冲：提前按空格 =====
        if (Input.GetButtonDown("Jump"))
        {
            _jumpBufferTimer = _jumpBuffer;                          // 记录按键时间
        }
        else
        {
            _jumpBufferTimer -= Time.deltaTime;                     // 倒计时
        }

        // ===== 土狼时间 + 跳跃次数重置 =====
        if (_controller.isGrounded)
        {
            _jumpsRemaining = _maxJumps;                            // 着地：重置跳跃次数
            _coyoteTimer = _coyoteTime;                              // 着地：重置土狼时间
            if (_velocity.y < 0) _velocity.y = -2f;
        }
        else
        {
            _coyoteTimer -= Time.deltaTime;                         // 空中：土狼时间倒计时
        }

        // ===== 是否可跳（在地面 或 土狼时间内 或 还有跳跃次数） =====
        bool canJump = (_coyoteTimer > 0f || _jumpsRemaining > 0) && _jumpBufferTimer > 0f;

        if (canJump)
        {
            _velocity.y = _jumpForce;
            _jumpBufferTimer = 0f;                                   // 消耗缓冲

            if (_coyoteTimer <= 0f) _jumpsRemaining--;               // 不是土狼跳，消耗一次跳跃次数
            _coyoteTimer = 0f;                                       // 土狼时间清零
        }

        // ===== 重力（下落时加重，让跳跃更紧凑） =====
        if (_velocity.y < 0)
        {
            _velocity.y -= _gravity * _fallMultiplier * Time.deltaTime;
        }
        else
        {
            _velocity.y -= _gravity * Time.deltaTime;
        }

        // ===== 移动 =====
        Vector3 move = transform.right * h * _moveSpeed + Vector3.up * _velocity.y;
        _controller.Move(move * Time.deltaTime);
    }
}
```

#### 语法拆解

##### `_fallMultiplier` 下落加重是什么？

```csharp
if (_velocity.y < 0)
    _velocity.y -= _gravity * _fallMultiplier * Time.deltaTime;
```

| 部分 | 含义 |
|------|------|
| `_velocity.y < 0` | 正在下落（y 速度为负） |
| `_fallMultiplier` | 下落时重力乘这个系数（如 1.5 = 1.5 倍重力） |

**整行人话**：下落时比上升时「更重」，跳起来感觉轻盈，掉下来干脆——这是马里奥等平台游戏的标准手感优化。

---

##### 土狼时间 Coyote Time 是什么？

```csharp
// 掉下去 0.1 秒内还能跳
if (_controller.isGrounded)
    _coyoteTimer = _coyoteTime;
else
    _coyoteTimer -= Time.deltaTime;

// 土狼时间内算可跳
bool canJump = _coyoteTimer > 0f || ...
```

**整行人话**：角色刚掉下平台那一小段时间内，按空格还能起跳——玩家感觉「更跟手」，不会因为差 1 帧就没跳起来而摔死。

平台跳跃场景：创建几个高低不同的 Platform，角色放第一个平台上，挂此脚本即可测试双跳手感。

## 一、与 Animator 动画集成

角色控制器负责「动」，Animator 负责「演」——把移动状态映射成动画参数。

```csharp
// 在移动脚本里同步参数给 Animator
[SerializeField] private Animator _anim;

void Update()
{
    // ... 移动逻辑 ...

    // 水平速度大小 → 走/跑/Idle 混合树
    float speed = new Vector3(_velocity.x, 0, _velocity.z).magnitude;
    _anim.SetFloat("Speed", speed);

    // 是否在地面 → 着地/下落状态机
    _anim.SetBool("IsGrounded", _controller.isGrounded);

    // 跳跃触发 → 跳转动画
    if (Input.GetButtonDown("Jump") && _controller.isGrounded)
    {
        _anim.SetTrigger("Jump");
    }
}
```

| Animator 参数 | 来源 | 用途 |
|---------------|------|------|
| `Speed` (float) | 水平速度大小 | 混合树：Idle ↔ Walk ↔ Run |
| `IsGrounded` (bool) | controller.isGrounded | 着地/空中状态切换 |
| `Jump` (trigger) | 跳跃输入时 | 触发跳跃动画 |
| `Fall` (float) | _velocity.y（为负时） | 下落动画速度 |

## 三、CharacterController vs Rigidbody 选型决策

```
需要角色移动？
  ├─ 人形角色、精确控制、不受干扰 → CharacterController ✅
  ├─ 完全物理化、会被击飞、布娃娃 → Rigidbody ✅
  └─ 部分物理部分脚本 → 混合（角色用 CC，被击飞时切 Rigidbody）
```

| 对比项 | CharacterController | Rigidbody |
|--------|---------------------|-----------|
| 移动控制 | 精确到帧，想怎么动就怎么动 | 受物理影响，有惯性、摩擦、弹跳 |
| 适用 | 玩家主角、NPC、怪物 | 汽车、箱子、弹球、物理道具 |
| 碰撞体 | 只能胶囊 | 任意形状 |
| 受力 | 不受力（脚本完全控制） | 受力、碰撞、爆炸 |
| 调试难度 | 低 | 高（参数多） |
| 性能 | 低（扫描碰撞） | 较高（物理模拟） |

## 四、工程化封装

```csharp
public class PlayerController : MonoBehaviour
{
    public static PlayerController Instance { get; private set; }

    [SerializeField] private float _walkSpeed = 5f;
    [SerializeField] private float _runSpeed = 8f;
    [SerializeField] private float _jumpForce = 8f;
    [SerializeField] private float _gravity = 20f;

    private CharacterController _controller;
    private Vector3 _velocity;

    public bool IsGrounded => _controller.isGrounded;
    public float HorizontalSpeed => new Vector3(_velocity.x, 0, _velocity.z).magnitude;

    void Awake()
    {
        Instance = this;
        _controller = GetComponent<CharacterController>();
    }

    public void Move(Vector2 input, bool isRunning)
    {
        Vector3 moveDir = transform.right * input.x + transform.forward * input.y;
        float speed = isRunning ? _runSpeed : _walkSpeed;

        ApplyGravity();
        Vector3 finalMove = moveDir.normalized * speed + Vector3.up * _velocity.y;
        _controller.Move(finalMove * Time.deltaTime);
    }

    public bool TryJump()
    {
        if (_controller.isGrounded)
        {
            _velocity.y = _jumpForce;
            return true;
        }
        return false;
    }

    void ApplyGravity()
    {
        if (_controller.isGrounded && _velocity.y < 0)
            _velocity.y = -2f;
        else
            _velocity.y -= _gravity * Time.deltaTime;
    }
}
```

## 五、常见误区与最佳实践

| 误区 | 正确做法 |
|------|----------|
| Move 不乘 deltaTime | 速度 × deltaTime = 位移 |
| 忘写重力 → 飘在空中 | 每帧 `_velocity.y -= gravity * dt` |
| 在 Move 之前读 isGrounded | Move 之后读，是刚移动完的状态 |
| 同时加 Rigidbody + CharacterController | 两个不要混用，选一个 |
| 斜着走比正方向快 | 方向向量 `.normalized` 归一化 |
| 胶囊体和模型不匹配 | 仔细调 Radius/Height/Center |
| 每帧 GetComponent | Awake/Start 里缓存引用 |
| 重力用 Physics.gravity | 自己定义的 _gravity 数值更好调 |

# 【全文总结】

## 最重要的一行代码

```csharp
_controller.Move((moveDir * _speed + Vector3.up * _velocity.y) * Time.deltaTime);
```

| 部分 | 含义 |
|------|------|
| `_controller.Move(...)` | 角色控制器移动方法 |
| `moveDir * _speed` | 水平速度 |
| `Vector3.up * _velocity.y` | 垂直速度（重力 + 跳跃） |
| `* Time.deltaTime` | 速度 → 本帧位移 |

## 三阶段对照

| 阶段 | 代表案例 |
|------|----------|
| 零基础 | 认识组件、Inspector 参数、读懂 Move 一行 |
| 入门 | WASD + 跳跃 + 重力、第一人称视角、平台跳跃 |
| 进阶 | 动画集成、选型决策、封装、踩坑清单 |

## API 速查

| 代码 | 作用 |
|------|------|
| `controller.Move(motion)` | 移动（传位移，自己处理重力） |
| `controller.SimpleMove(speed)` | 简化移动（传水平速度，自动重力） |
| `controller.isGrounded` | 是否在地面（只读） |
| `controller.velocity` | 当前实际速度（只读） |
| `controller.radius / height` | 胶囊体大小 |
| `controller.slopeLimit / stepOffset` | 爬坡/迈台能力 |
| `OnControllerColliderHit` | 碰撞回调 |
