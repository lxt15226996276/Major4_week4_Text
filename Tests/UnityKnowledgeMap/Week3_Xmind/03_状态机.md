## 【全文总述】

**有限状态机（Finite State Machine，简称 FSM）** 是游戏里最常用的设计模式之一：把「角色的行为」拆成一个个独立的「状态」（比如 Idle 待机、Move 移动、Attack 攻击、Jump 跳跃），同一时间只能处在一个状态里，满足条件就在状态之间切换。  
没有状态机时，你可能写几十个 if-else 堆在 Update 里——「如果在地上按空格就跳、如果看到敌人就攻击、如果没血就逃跑」——代码堆成山，加新状态还要改老代码。有了状态机，每个状态只管自己的事，切换逻辑清晰，加新状态也很方便。  
本文从概念到接口设计，再到完整的角色状态机实现，逐层展开。

# 第一阶段：零基础篇

## 【阶段总述】

零基础阶段建立**正确认知**：什么是状态机、为什么不用 if-else 堆着写、状态机的三要素是什么。  
同时学会**读懂** `IState` 接口的三个方法分别干什么。

---

## 一、定义 — 有限状态机是什么？

| 项目 | 说明 |
|------|------|
| **全称** | Finite State Machine，简称 FSM |
| **三要素** | 状态（State）、转换（Transition）、状态机（StateMachine） |
| **核心规则** | 同一时间**只有一个**状态是激活的；满足条件时从一个状态切到另一个状态 |
| **一句话** | 把「一堆 if-else」拆成「一个个独立的状态」，让代码更清晰 |

```csharp
// 不用状态机：Update 里堆 if-else
void Update()
{
    if (isGrounded && moveInput.magnitude > 0.1f)
    {
        // 移动逻辑...
    }
    else if (isGrounded && moveInput.magnitude < 0.1f)
    {
        // 待机逻辑...
    }
    else if (!isGrounded)
    {
        // 跳跃逻辑...
    }
    // ... 加一个攻击状态，这里又多一堆 if
}

// 用状态机：每个状态一个类
void Update()
{
    _stateMachine.CurrentState.Update();   // 一句话，当前状态自己管自己
}
```

---

## 二、本质 — 为什么要有状态机？

### 2.1 没有状态机的痛苦（if-else 地狱）

```
加一个攻击状态 → 改 Update，加两三个 if
加一个死亡状态 → 又加四五个判断
加一个受击状态 → 每个状态里都要判断「被打了怎么办」
三个月后 → 没人敢动这段代码，怕改崩
```

### 2.2 用了状态机的清爽

```
每个状态是一个独立的文件：
  IdleState.cs    → 待机时做什么、什么条件下切到 Move / Jump
  MoveState.cs    → 移动时做什么、什么条件下切到 Idle / Attack
  AttackState.cs  → 攻击时做什么、攻击完切回哪

加新状态 → 新建一个文件，在转换表里加几条规则
老代码一行都不用动
```

**本质理解**：状态机 = 「行为模块化」，把大堆 if-else 拆成一个个独立小模块，每个模块管自己的事。

### 2.3 生活中的类比

| 生活中的状态机 | 状态 | 转换条件 |
|----------------|------|----------|
| **交通灯** | 红灯 / 黄灯 / 绿灯 | 倒计时到了就切 |
| **电梯** | 开门 / 关门 / 上行 / 下行 / 待机 | 按按钮、到楼层 |
| **手机** | 亮屏 / 锁屏 / 关机 / 通话 | 按电源键、来电 |

游戏里的角色也是一样：待机、走路、跳跃、攻击……都是一个个状态。

---

## 三、状态机三要素（零基础必记）

| 要素 | 作用 | 类比 |
|------|------|------|
| **State（状态）** | 每个状态管「在这个状态下做什么」 | 红灯：亮红灯 + 倒计时 |
| **Transition（转换）** | 「满足什么条件 → 切到哪个状态」 | 红灯倒计时 0 → 切绿灯 |
| **StateMachine（状态机）** | 管理当前状态、负责切换 | 交通灯控制器 |

### 状态的四件套

每个状态都有这三个关键方法 + 转换逻辑：

```
Enter  →  进入状态时调用一次（初始化、播动画）
Update →  状态中每帧调用（做事、检测切换条件）
Exit   →  退出状态时调用一次（清理、重置）
```

> **记忆口诀**：  
> Enter 入场，Update 干活，Exit 退场。

---

## 四、核心一课：如何读懂 IState 接口

```csharp
public interface IState
{
    void Enter();
    void Update();
    void Exit();
}
```

| 部分 | 含义 |
|------|------|
| `interface IState` | 接口：定义「所有状态都必须有这三个方法」 |
| `void Enter()` | 进入：切到这个状态时，只调用一次 |
| `void Update()` | 更新：在这个状态中，每帧调用 |
| `void Exit()` | 退出：离开这个状态时，只调用一次 |

**整行人话**：所有状态都得有「进来、每帧更新、离开」这三件事——接口就是给所有状态立规矩，保证每个状态都按同一套模板来。

### 为什么用接口不用继承？

C# 只能继承一个基类，但可以实现多个接口。状态接口只是规定「要有这三个方法」，具体实现完全由各个状态自己决定。

## 【零基础阶段小结】

| 维度 | 要点 |
|------|------|
| **定义** | 有限个状态，同一时间只有一个激活 |
| **本质** | 把 if-else 拆成独立模块，行为模块化 |
| **三要素** | 状态 / 转换 / 状态机 |
| **状态三件套** | Enter（入场）→ Update（干活）→ Exit（退场） |

**阶段检验**：能说出状态机三要素；能解释为什么不用 if-else 堆；能说出 IState 三个方法分别什么时候调用。

# 第二阶段：入门篇

## 【阶段总述】

入门阶段完整实现一个 **角色状态机**：`IState` 接口 + `StateMachine` 管理器 + `PlayerContext` 上下文 + `Idle / Move / Jump` 三个具体状态。  
重点：**上下文（Context）**——所有状态共享的数据和组件引用都放在这里，避免每个状态都去找组件。

---

## 一、核心组成详解（逐词读懂）

### 1.1 StateMachine — 状态机管理器

```csharp
public class StateMachine
{
    public IState CurrentState { get; private set; }

    public void Initialize(IState startState) { ... }
    public void ChangeState(IState newState) { ... }
}
```

| 部分 | 含义 |
|------|------|
| `CurrentState` | 当前激活的状态（只读，外部不能直接改） |
| `Initialize(startState)` | 初始化：设置第一个状态，调用它的 Enter |
| `ChangeState(newState)` | 切换状态：旧状态 Exit → 新状态 Enter |

**整行人话**：状态机就是个「管家」——它知道当前是谁在干活，你让它换人，它就叫旧的下班、新的上班。

---

### 1.2 Context — 上下文是什么？

```csharp
public class PlayerContext
{
    public CharacterController Controller;
    public Animator Animator;
    public Vector2 MoveInput;
    public float Speed;
    public float JumpForce;
    // ... 所有状态都需要用到的东西都放这里
}
```

| 部分 | 含义 |
|------|------|
| `Context` | 上下文 / 环境：所有状态共享的「数据和组件仓库」 |
| 放什么 | 角色组件引用（Controller、Animator）、数据（速度、血量）、输入 |
| 为什么要 | 每个状态构造时拿到同一个 Context，不用每个状态都 GetComponent |

**整行人话**：上下文就是「共享工具箱」——所有状态都用同一套工具和数据，不用各自去找。

---

### 1.3 状态切换流程

```
ChangeState(newState) 被调用
        │
        ▼
  CurrentState.Exit()   ← 旧状态退场，清理东西
        │
        ▼
  CurrentState = newState
        │
        ▼
  newState.Enter()      ← 新状态入场，初始化
        │
        ▼
  之后每帧调用
  CurrentState.Update()
```

> **注意**：切换状态时，Exit 和 Enter 在同一帧内完成。不要在 Exit 里等几帧再切。

---

## 二、标准实现步骤

```
步骤 1：定义 IState 接口（Enter/Update/Exit）
步骤 2：定义 StateMachine 状态机管理器
步骤 3：定义 PlayerContext 上下文（共享数据+组件）
步骤 4：写具体状态类（IdleState / MoveState / JumpState）
         ├─ 构造函数接收 context
         ├─ Enter：播动画、初始化
         ├─ Update：执行行为 + 检测切换条件 + 调用 _stateMachine.ChangeState
         └─ Exit：清理、重置
步骤 5：在 PlayerController 里创建状态机、创建所有状态、Initialize
步骤 6：Update 里 _stateMachine.CurrentState.Update()
```

### 案例 1：完整角色状态机（Idle / Move / Jump）

**功能**：角色有三个状态——待机、移动、跳跃；根据输入和地面检测自动切换。

```csharp
// ============================================
// 第一步：IState 接口
// ============================================
public interface IState
{
    void Enter();       // 进入状态（一次）
    void Update();      // 状态中更新（每帧）
    void Exit();        // 退出状态（一次）
}

// ============================================
// 第二步：StateMachine 状态机
// ============================================
public class StateMachine
{
    public IState CurrentState { get; private set; }

    // 初始化：设置起始状态
    public void Initialize(IState startState)
    {
        CurrentState = startState;
        CurrentState.Enter();
    }

    // 切换状态
    public void ChangeState(IState newState)
    {
        CurrentState.Exit();              // 旧状态退场
        CurrentState = newState;          // 换状态
        CurrentState.Enter();             // 新状态入场
    }
}

// ============================================
// 第三步：PlayerContext 上下文（共享数据）
// ============================================
public class PlayerContext
{
    public readonly StateMachine StateMachine;   // 状态机引用
    public readonly CharacterController Controller; // 角色控制器
    public readonly Animator Animator;           // 动画器
    public readonly Transform Transform;         // Transform

    public Vector2 MoveInput;                    // 移动输入
    public float MoveSpeed;                      // 移动速度
    public float JumpForce;                      // 跳跃力
    public float Gravity;                        // 重力
    public Vector3 Velocity;                     // 当前速度

    public PlayerContext(StateMachine sm, CharacterController cc, Animator anim, Transform tr)
    {
        StateMachine = sm;
        Controller = cc;
        Animator = anim;
        Transform = tr;
    }
}

// ============================================
// 第四步：具体状态类 —— Idle 待机
// ============================================
public class IdleState : IState
{
    private readonly PlayerContext _context;     // 上下文引用

    public IdleState(PlayerContext context)
    {
        _context = context;
    }

    public void Enter()
    {
        // 进入待机：播待机动画
        _context.Animator.SetBool("IsMoving", false);
    }

    public void Update()
    {
        // 检测切换条件
        // 条件 1：有输入 → 切到 Move
        if (_context.MoveInput.magnitude > 0.1f)
        {
            _context.StateMachine.ChangeState(new MoveState(_context));
            return;
        }

        // 条件 2：按跳跃键 → 切到 Jump
        if (Input.GetButtonDown("Jump") && _context.Controller.isGrounded)
        {
            _context.Velocity.y = _context.JumpForce;
            _context.StateMachine.ChangeState(new JumpState(_context));
            return;
        }

        // 待机要做的事：重力下落（保持贴地）
        ApplyGravity();
        Move();
    }

    public void Exit()
    {
        // 退出待机：可以做点清理，这里暂时空着
    }

    // --- 私有方法 ---
    void ApplyGravity()
    {
        if (_context.Controller.isGrounded && _context.Velocity.y < 0)
            _context.Velocity.y = -2f;
        else
            _context.Velocity.y -= _context.Gravity * Time.deltaTime;
    }

    void Move()
    {
        Vector3 dir = _context.Transform.right * _context.MoveInput.x
                    + _context.Transform.forward * _context.MoveInput.y;
        Vector3 final = dir.normalized * _context.MoveSpeed + Vector3.up * _context.Velocity.y;
        _context.Controller.Move(final * Time.deltaTime);
    }
}

// ============================================
// 第四步续：MoveState 移动状态
// ============================================
public class MoveState : IState
{
    private readonly PlayerContext _context;

    public MoveState(PlayerContext context) { _context = context; }

    public void Enter()
    {
        _context.Animator.SetBool("IsMoving", true);   // 播移动动画
    }

    public void Update()
    {
        // 条件 1：没输入了 → 切回 Idle
        if (_context.MoveInput.magnitude < 0.1f)
        {
            _context.StateMachine.ChangeState(new IdleState(_context));
            return;
        }

        // 条件 2：按跳跃 → 切 Jump
        if (Input.GetButtonDown("Jump") && _context.Controller.isGrounded)
        {
            _context.Velocity.y = _context.JumpForce;
            _context.StateMachine.ChangeState(new JumpState(_context));
            return;
        }

        ApplyGravity();
        Move();
    }

    public void Exit() { }

    void ApplyGravity()
    {
        if (_context.Controller.isGrounded && _context.Velocity.y < 0)
            _context.Velocity.y = -2f;
        else
            _context.Velocity.y -= _context.Gravity * Time.deltaTime;
    }

    void Move()
    {
        Vector3 dir = _context.Transform.right * _context.MoveInput.x
                    + _context.Transform.forward * _context.MoveInput.y;
        Vector3 final = dir.normalized * _context.MoveSpeed + Vector3.up * _context.Velocity.y;
        _context.Controller.Move(final * Time.deltaTime);
    }
}

// ============================================
// 第四步续：JumpState 跳跃状态
// ============================================
public class JumpState : IState
{
    private readonly PlayerContext _context;

    public JumpState(PlayerContext context) { _context = context; }

    public void Enter()
    {
        _context.Animator.SetTrigger("Jump");    // 触发跳跃动画
    }

    public void Update()
    {
        ApplyGravity();
        Move();

        // 条件：落地了 → 切回 Idle 或 Move
        if (_context.Controller.isGrounded && _context.Velocity.y <= 0)
        {
            if (_context.MoveInput.magnitude > 0.1f)
                _context.StateMachine.ChangeState(new MoveState(_context));
            else
                _context.StateMachine.ChangeState(new IdleState(_context));
        }
    }

    public void Exit() { }

    void ApplyGravity()
    {
        _context.Velocity.y -= _context.Gravity * Time.deltaTime;
    }

    void Move()
    {
        Vector3 dir = _context.Transform.right * _context.MoveInput.x
                    + _context.Transform.forward * _context.MoveInput.y;
        Vector3 final = dir.normalized * _context.MoveSpeed + Vector3.up * _context.Velocity.y;
        _context.Controller.Move(final * Time.deltaTime);
    }
}
```

#### 语法拆解

##### `interface IState` 是什么？

```csharp
public interface IState
{
    void Enter();
    void Update();
    void Exit();
}
```

| 部分 | 含义 |
|------|------|
| `interface` | 接口关键字：只定义「要有什么方法」，不写具体实现 |
| `IState` | 接口名，惯例以 I 开头 |
| 三个方法 | 只有方法签名，没有 `{ }` 体——具体实现交给实现接口的类 |

**整行人话**：接口就是「规矩」——所有状态都必须有 Enter、Update、Exit 这三个方法，具体怎么干各状态自己写。

---

##### `readonly PlayerContext _context` 是什么？

```csharp
private readonly PlayerContext _context;
```

| 部分 | 含义 |
|------|------|
| `readonly` | 只读：构造函数里赋值之后就不能改了 |
| `_context` | 上下文引用，所有状态共享同一套数据 |

**整行人话**：每个状态都拿着同一个「工具箱」的引用，只读不能换，防止不小心把工具箱弄丢了。

---

##### `new MoveState(_context)` 每次 new 会浪费吗？

入门阶段每次切换都 new 一个新状态是没问题的——状态对象很小（只有引用和几个变量），GC 压力可以忽略。  
进阶阶段可以用「状态字典」预创建所有状态，切换时直接取（见进阶篇）。

#### 操作提示

1. 在项目里新建以上 5 个脚本（IState、StateMachine、PlayerContext、IdleState、MoveState、JumpState）  
2. 在 PlayerController 里创建并初始化：

```csharp
public class PlayerController : MonoBehaviour
{
    [SerializeField] private float _moveSpeed = 5f;
    [SerializeField] private float _jumpForce = 8f;
    [SerializeField] private float _gravity = 20f;

    private StateMachine _stateMachine;
    private PlayerContext _context;

    void Start()
    {
        var cc = GetComponent<CharacterController>();
        var anim = GetComponent<Animator>();

        _stateMachine = new StateMachine();
        _context = new PlayerContext(_stateMachine, cc, anim, transform);
        _context.MoveSpeed = _moveSpeed;
        _context.JumpForce = _jumpForce;
        _context.Gravity = _gravity;

        // 初始状态：待机
        _stateMachine.Initialize(new IdleState(_context));
    }

    void Update()
    {
        // 读输入
        _context.MoveInput = new Vector2(
            Input.GetAxis("Horizontal"),
            Input.GetAxis("Vertical"));

        // 当前状态更新
        _stateMachine.CurrentState.Update();
    }
}
```

### 案例 2：游戏流程状态机（Menu / Playing / Pause / GameOver）

状态机不只管角色——游戏流程也能用状态机管：

```csharp
// MainMenuState：主菜单（显示菜单 UI，点击开始按钮切到 Playing）
// PlayingState：游戏中（游戏逻辑跑、按 ESC 切到 Pause）
// PauseState：暂停（显示暂停 UI、Time.timeScale=0）
// GameOverState：游戏结束（显示结算 UI）

public class GameManager : MonoBehaviour
{

    void Start()
    {
        _gameSM = new StateMachine();
        _gameSM.Initialize(new MainMenuState(_gameSM, this));
    }

    void Update()
    {
        _gameSM.CurrentState.Update();
    }
}
```

> **举一反三**：UI 流程、关卡流程、对话系统……只要是「分阶段、按条件切换」的东西，都适合用状态机。

## 一、状态字典 + 预创建状态

每次切换都 `new` 一个状态虽然没问题，但更规范的做法是：**一开始就把所有状态创建好，放到字典里，切换时直接取**。

```csharp
public class PlayerStateMachine : StateMachine
{
    public readonly IdleState Idle;
    public readonly MoveState Move;
    public readonly JumpState Jump;
    public readonly AttackState Attack;

    public PlayerStateMachine(PlayerContext context)
    {
        Idle = new IdleState(context);
        Move = new MoveState(context);
        Jump = new JumpState(context);
        Attack = new AttackState(context);
    }
}

// 使用：
_sm = new PlayerStateMachine(_context);
_sm.Initialize(_sm.Idle);

// 切换时直接取，不用 new
_sm.ChangeState(_sm.Move);
```

**好处**：
1. 没有运行时 new，无 GC 分配
2. 所有状态实例一目了然
3. 可以在 Inspector 或外部获取某个状态做调试

## 二、泛型状态机（更通用）

可以做一个泛型版本的状态机，不同类型的上下文都能用：

```csharp
public class StateMachine<TContext>
{
    public IState<TContext> CurrentState { get; private set; }
    public TContext Context { get; }

    public StateMachine(TContext context)
    {
        Context = context;
    }

    public void Initialize(IState<TContext> startState)
    {
        CurrentState = startState;
        CurrentState.Enter(Context);
    }

    public void ChangeState(IState<TContext> newState)
    {
        CurrentState.Exit(Context);
        CurrentState = newState;
        CurrentState.Enter(Context);
    }
}

public interface IState<TContext>
{
    void Enter(TContext ctx);
    void Update(TContext ctx);
    void Exit(TContext ctx);
}
```

**用法**：玩家状态机用 `StateMachine<PlayerContext>`，敌人状态机用 `StateMachine<EnemyContext>`，同一套模板。

## 三、与 Animator 深度集成

| 集成方式 | 说明 |
|----------|------|
| **状态 Enter 设置参数** | 进入 Idle 就 `SetBool("IsMoving", false)` |
| **状态 Update 同步 float** | Move 状态里 `SetFloat("Speed", speed)` |
| **动画事件回调驱动切换** | 攻击动画最后一帧发「AttackEnd」事件 → 切回 Idle |
| **状态机行为（StateMachineBehaviour）** | 给 Animator 里的状态加脚本，OnStateEnter/Update/Exit 时通知逻辑状态机 |

```csharp
// 示例：攻击状态 + 动画事件
public class AttackState : IState
{
    bool _isAttackFinished;

    public void Enter()
    {
        _context.Animator.SetTrigger("Attack");
        _isAttackFinished = false;
        // 订阅动画事件（需要自定义动画事件系统）
        _context.AnimatorEvents.OnAttackEnd += HandleAttackEnd;
    }

    public void Update()
    {
        if (_isAttackFinished)
        {
            _context.StateMachine.ChangeState(_context.StateMachine.Idle);
        }
    }

    public void Exit()
    {
        _context.AnimatorEvents.OnAttackEnd -= HandleAttackEnd;
    }

    void HandleAttackEnd() => _isAttackFinished = true;
}
```

## 四、FSM vs 行为树 vs GOAP 选型

| 方案 | 适合场景 | 复杂度 | 学习成本 |
|------|----------|--------|----------|
| **有限状态机 FSM** | 几个 ~ 十几个状态，角色行为、游戏流程 | 低 | 低（本文内容） |
| **行为树 Behavior Tree** | 复杂 AI、几十个行为、需要组合 | 中 | 中 |
| **GOAP（目标导向）** | 策略 AI、自动规划行动 | 高 | 高 |

```
状态数量？
  ├─ < 10 → FSM（最常用，90% 的情况够用）
  ├─ 10 ~ 50 → 分层 FSM 或 行为树
  └─ > 50 且复杂 → 行为树 / GOAP
```

> **新手建议**：先把 FSM 用熟。大多数游戏的角色和怪物，FSM 完全够用。行为树等真正遇到复杂 AI 再学。

## 五、常见误区与最佳实践

| 误区 | 正确做法 |
|------|----------|
| 忘记 Exit 里清理订阅 | Exit 里 `-=` 事件，避免内存泄漏和野回调 |
| Update 里每帧都 ChangeState | 加 return 或状态门闩，切了就直接返回，不再继续执行 |
| 状态间互相 new | 用状态字典预创建，切换时直接取 |
| 状态里放太多逻辑 | 一个状态只做一件事，复杂了就拆子状态 |
| 上下文变成上帝类 | 合理拆分，数据按职责归类 |
| 循环切换（A→B→A→B…） | 加冷却或检查条件，避免同一帧来回切 |
| 在 Enter 里直接 ChangeState | 进入立刻又切出去容易出问题，至少等一帧 |
| 所有状态都复制同样的代码 | 抽取基类 StateBase，共用逻辑写在基类里 |
