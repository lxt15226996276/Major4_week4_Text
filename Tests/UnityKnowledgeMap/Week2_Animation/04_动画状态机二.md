## 思维导图总览

```
动画状态机（二）
│
├── 【第一梯队】人型动画重定向 ★★
│   ├── Humanoid Rig + Avatar Configure
│   ├── 标准骨骼映射 → 多模型共享 Clip
│   └── Controller 重用（同一 .controller 多角色）
│
├── 【第二梯队】BlendTree 混合树 ★★★
│   ├── 单 State 内多 Motion 按参数混合
│   ├── 1D Simple Directional（Speed 走跑）
│   ├── 2D Simple / Freeform（八方向 MoveX/MoveZ）
│   ├── 3D（了解）
│   └── Threshold 与 Parameter 值域对齐
│
├── 【第三梯队】Animator IK ★★
│   ├── FK（Clip 默认）vs IK（运行时修正）
│   ├── Layer 勾 IK Pass
│   ├── OnAnimatorIK(int layerIndex)
│   └── SetIKPosition/Rotation + SetLookAt
│
├── 【第四梯队】MatchTarget 目标匹配 ★
│   ├── Clip 特定 normalizedTime 窗口内对齐骨骼
│   ├── AvatarTarget（RightFoot / LeftHand…）
│   └── 攀爬 / 开门 / 踩点
│
└── 【注意要点】
    ├── SetFloat 驱动 BlendTree
    ├── IK 需 Humanoid + IK Pass
    └── MatchTarget 需在正确 State 播放中调用
```

## 一、四大模块关系（必背）

```
Retarget（谁播同一套动画）
    ↓
BlendTree（怎么走跑混合得更平滑）
    ↓
IK（脚怎么贴地、手怎么瞄）
    ↓
MatchTarget（怎么精确对齐到世界坐标）
```

**[重难点]** 四者解决不同问题，可组合使用：Retarget 复用资源 → BlendTree 平滑 Locomotion → IK 适应地形 → MatchTarget 精确交互。

## 二、人型动画重定向（⭐⭐）

### 2.1 原理

| 概念 | 说明 |
|------|------|
| **原理** | 不同人形模型骨架相似 → Unity 映射到标准 Humanoid → 同一 Clip 可用于不同模型 |
| **Avatar** | 人形骨骼映射表 |
| **官方** | [Manual: Retargeting](https://docs.unity3d.com/Manual/Retargeting.html) |

### 2.2 配置与重用

| 步骤 | 操作 | 频率 |
|:---:|------|------|
| 1 | FBX → Rig → **Humanoid** | ⭐⭐⭐ |
| 2 | **Configure** → 全绿 → Apply | ⭐⭐⭐ |
| 3 | 多角色绑定**同一 Controller** | ⭐⭐ |
| 4 | 脚本 Set 参数不变 | ⭐⭐⭐ |

```csharp
// 角色 A、B 共用 Controller，仅 Avatar 不同
animA.SetFloat("Speed", speedA);
animB.SetFloat("Speed", speedB);
```

**[易错点]** Configure 报红 → Retarget 失败，动画扭曲或 T-Pose。

**[教材精华·陈俊宇/王磊/Dean]** 共用 Controller + 各自 Avatar → Parameters 接口不变，模型差异由 Retarget 消化。

## 三、BlendTree 混合树（⭐⭐⭐）

### 3.1 概念

| 项目 | 内容 |
|------|------|
| **是什么** | 在一个 **State** 内，按参数权重**混合多个 Motion（Clip）** |
| **官方** | [Manual: Blend Trees](https://docs.unity3d.com/Manual/class-BlendTree.html) |
| **优势** | 走↔跑平滑过渡，避免离散 State 切换卡顿 |
| **创建** | Create State → **From New Blend Tree** → 双击进入编辑 |

### 3.2 混合类型

| 类型 | 参数 | 典型应用 | 频率 |
|------|------|---------|------|
| **1D Simple Directional** | 1 Float（Speed 0~1） | Idle / Walk / Run | ⭐⭐⭐ |
| **2D Simple Directional** | MoveX + MoveZ | 四/八方向 | ⭐⭐⭐ |
| **2D Freeform Cartesian** | MoveX + MoveZ | 自由平面分布 | ⭐⭐ |
| **2D Freeform Directional** | MoveX + MoveZ | 方向性移动 | ⭐⭐ |
| **Direct** | 多个 Float 直接控权重 | 精细控制 | ⭐ |
| **3D** | 3 Float | 特殊需求 | ⭐ |

### 3.3 属性设置

| 属性 | 说明 | 频率 |
|------|------|------|
| **Blend Type** | 1D / 2D / Direct 等 | ⭐⭐⭐ |
| **Parameter** | 与脚本 Set 的参数名**完全一致** | ⭐⭐⭐ |
| **Threshold** | 各 Motion 对应的参数值 | ⭐⭐⭐ |
| **Motion 列表** | Threshold 对应的 Clip | ⭐⭐⭐ |

```csharp
// 1D 走跑
anim.SetFloat("Speed", currentSpeed);  // 0=Idle, 0.5=Walk, 1=Run

// 2D 八方向
anim.SetFloat("MoveX", input.x);
anim.SetFloat("MoveZ", input.z);
```

**[重难点] Threshold 与参数值域**

```
BlendTree 1D 示例：
  Threshold 0.0 → Idle Clip
  Threshold 0.5 → Walk Clip
  Threshold 1.0 → Run Clip

脚本 SetFloat("Speed", 0.3) → 在 Idle 与 Walk 之间混合
```

**[易错点]**
- 参数名拼写不一致 → BlendTree **不动**
- SetFloat 值永远 0 → 永远只播 Threshold 0 的 Motion
- 用离散 Walk/Run State 代替 BlendTree → 切换生硬（能混则用 BlendTree）

## 四、Animator IK 反向动力学（⭐⭐）

### 4.1 FK vs IK

| 类型 | 全称 | 驱动方式 | 来源 |
|------|------|---------|------|
| **FK** | Forward Kinematics | 父骨骼带动子骨骼 | Clip 默认 |
| **IK** | Inverse Kinematics | 子节点目标位置反推全身 | 运行时 OnAnimatorIK |

| 应用 | 说明 |
|------|------|
| 脚踩 uneven 地形 | LeftFoot / RightFoot IK |
| 手扶栏杆 / 瞄目标 | Hand IK |
| 头部注视 | SetLookAtPosition |

### 4.2 实现条件

| 条件 | 说明 | 频率 |
|------|------|------|
| **Humanoid** | Generic 不支持标准 AvatarIKGoal | ⭐⭐⭐ |
| **Layer IK Pass** | Animator Layer 勾选 **IK Pass** | ⭐⭐⭐ |
| **OnAnimatorIK** | 脚本实现 `void OnAnimatorIK(int layerIndex)` | ⭐⭐⭐ |

### 4.3 常用 API

| API | 说明 | 频率 |
|-----|------|------|
| `SetIKPositionWeight(goal, w)` | 位置 IK 权重 0~1 | ⭐⭐⭐ |
| `SetIKPosition(goal, pos)` | 世界坐标目标 | ⭐⭐⭐ |
| `SetIKRotationWeight(goal, w)` | 旋转 IK 权重 | ⭐⭐ |
| `SetIKRotation(goal, rot)` | 目标旋转 | ⭐⭐ |
| `SetLookAtWeight(w)` | 头部注视权重 | ⭐⭐ |
| `SetLookAtPosition(pos)` | 注视点 | ⭐⭐ |

**AvatarIKGoal**：LeftFoot / RightFoot / LeftHand / RightHand 等。

```csharp
public class HandIK : MonoBehaviour
{
    public Transform handTarget;
    public Transform lookTarget;
    Animator anim;

    void Awake() => anim = GetComponent<Animator>();

    void OnAnimatorIK(int layerIndex)
    {
        if (handTarget == null) return;

        anim.SetIKPositionWeight(AvatarIKGoal.RightHand, 1f);
        anim.SetIKRotationWeight(AvatarIKGoal.RightHand, 1f);
        anim.SetIKPosition(AvatarIKGoal.RightHand, handTarget.position);
        anim.SetIKRotation(AvatarIKGoal.RightHand, handTarget.rotation);

        if (lookTarget != null)
        {
            anim.SetLookAtWeight(1f);
            anim.SetLookAtPosition(lookTarget.position);
        }
    }
}
```

**[易错点]**
- 未勾 **IK Pass** → `OnAnimatorIK` **从不调用**
- Generic 角色用 AvatarIKGoal → 不支持或行为异常
- Weight 直接设 1 无渐变 → 姿态硬切，应 0→1 插值

## 五、MatchTarget 目标匹配（⭐）

### 5.1 概念

| 项目 | 说明 |
|------|------|
| **是什么** | 在 Clip **特定 normalizedTime 区间**内，将某骨骼对齐到世界坐标/旋转 |
| **官方** | [API: Animator.MatchTarget](https://docs.unity3d.com/ScriptReference/Animator.MatchTarget.html) |
| **典型** | 攀爬抓沿、开门贴把手、跳跃脚踩点 |

### 5.2 API 参数

| 参数 | 说明 |
|------|------|
| `matchPosition` | 目标世界坐标 |
| `matchRotation` | 目标世界旋转 |
| `targetBodyPart` | AvatarTarget（Root/LeftFoot/RightHand…） |
| `weightMask` | MatchTargetWeightMask，位置/旋转权重 |
| `startNormalizedTime` | 开始匹配 0~1 |
| `targetNormalizedTime` | 完成匹配 0~1 |
| `completeMatch` | 是否在 target 时刻完全对齐 |

```csharp
void OnAnimatorIK(int layerIndex)
{
    if (!matching) return;
    var info = anim.GetCurrentAnimatorStateInfo(0);
    if (!info.IsName("Climb")) return;

    anim.MatchTarget(
        ledge.position,
        ledge.rotation,
        AvatarTarget.RightHand,
        new MatchTargetWeightMask(AvatarTarget.RightHand, 1f),
        0.2f, 0.5f,
        true);
}
```

**[易错点]**
- normalizedTime 窗口与 Clip 实际动作不对齐 → 无效或跳变
- 在 **Transition 过程中**调用 → 无效
- 每帧无条件 MatchTarget → 应在正确 State + 时间窗口内调用

## 六、注意要点与最佳实践

| 易错点 | 说明 |
|--------|------|
| BlendTree 不动 | 参数名/Threshold/类型错误 |
| IK 不执行 | 未勾 IK Pass / 非 Humanoid |
| MatchTarget 无效 | 时间窗口错 / 不在正确 State |
| Retarget 失败 | Avatar Configure 未全绿 |

### 最佳实践

- Locomotion 优先 1D/2D BlendTree，少建离散 Walk/Run State
- IK Weight 用插值渐变
- MatchTarget 先在 Animation 窗口预览 Clip 时间轴再设窗口
- Controller 设计为可复用 + Retarget 多角色
