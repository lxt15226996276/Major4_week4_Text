```

├── 【共同流程】三步 ★★★
│   ├── ① 模型/资源导入（FBX / Sprite / 2D 骨骼）
│   ├── ② Animator Controller（State / Transition / Parameters / Layer）

│
├── 【射击类】TPS/FPS ★★★
│   ├── Humanoid FBX + Clip
│   ├── Layer0 Locomotion + Layer1 上半身 Mask
│   ├── Parameters：Speed, IsAiming, Fire, Reload
│   └── SetLayerWeight 控制瞄准层
│
├── 【RPG 类】ARPG/MMO ★★★
│   ├── Humanoid + Avatar Configure
│   ├── Locomotion：1D BlendTree（Speed）
│   ├── Combat：Sub-State Machine
│   └── ComboIndex + normalizedTime 连击
│
├── 【2D 类】平台/横版 ★★
│   ├── Sprite 序列 / 2D 骨骼 Clip
│   ├── 离散 State：Idle / Run / Jump / Attack
│   ├── flipX 控制朝向
│   └── Animation Event 帧回调
│
├── 【选型总结】★★★
│   ├── Animator → 角色多状态
│   ├── Animation Legacy → 环境/Prop/镜头
│   ├── Humanoid → Retarget / IK
│   └── Generic → 怪物/载具/专属骨骼
│
└── 【融合与分层】
    ├── BlendTree / Transition → 平滑混合
    └── Layer + Mask + SetLayerWeight → 上下半身
```

## 一、共同工作流（⭐⭐⭐）

```
① 导入资源
   FBX（3D）/ Sprite 切图 / 2D Animation 包
        ↓
② 创建 Animator Controller
   State / Transition / Parameters / Layer（按需）
        ↓
③ 脚本驱动
   SetFloat / SetBool / SetTrigger — 逻辑与表现分离
```

**[重难点]** 脚本**只 Set 参数**；State、Transition、Layer 在 Controller **编辑器**配置。这是三类游戏的共同规范。

## 二、射击类游戏（⭐⭐⭐）

### 2.1 模型导入

| 步骤 | 操作 | 频率 |
|:---:|------|------|
| 1 | 导入带动画 FBX（Idle/Walk/Aim/Fire/Reload） | ⭐⭐⭐ |
| 2 | Rig → Humanoid 或 Generic | ⭐⭐⭐ |
| 3 | Animation 页签确认 Clip 与 Loop 设置 | ⭐⭐ |

### 2.2 状态机结构

| 模块 | 内容 | 频率 |
|------|------|------|
| **Layer 0** | Idle ↔ Walk，Speed 参数 | ⭐⭐⭐ |
| **Layer 1** | Aim / Fire / Reload，**Avatar Mask** 上半身 | ⭐⭐⭐ |
| **Parameters** | Speed, IsAiming, Fire, Reload | ⭐⭐⭐ |

```
Layer 0: Idle ──Speed──► Walk
Layer 1: IdleUpper ──IsAiming──► Aim ──Fire──► Fire
                              └──Reload──► Reload
```

### 2.3 脚本示例

```csharp
public class ShooterAnim : MonoBehaviour
{
    Animator anim;

    void Awake() => anim = GetComponent<Animator>();

    void Update()
    {
        anim.SetFloat("Speed", moveSpeed);
        anim.SetBool("IsAiming", isAiming);
        anim.SetLayerWeight(1, isAiming ? 1f : 0f);  // ⭐⭐⭐

        if (isAiming && Input.GetButtonDown("Fire1"))
            anim.SetTrigger("Fire");
        if (Input.GetKeyDown(KeyCode.R))
            anim.SetTrigger("Reload");
    }
}
```

**[教材精华·吴亚峰/王磊/Cookbook]** Layer0 移动 + Layer1 上半身 Mask + `SetLayerWeight` — 射击/TPS 标准结构。

## 三、RPG 类（⭐⭐⭐）

### 3.1 状态机结构

| 模块 | 结构 | 频率 |
|------|------|------|
| **Locomotion** | 1D BlendTree：Idle/Walk/Run，Speed 驱动 | ⭐⭐⭐ |
| **Combat** | Sub-State Machine：Attack/Skill/Hit/Die | ⭐⭐⭐ |
| **Jump** | Trigger，Has Exit Time 关 | ⭐⭐ |
| **Parameters** | Speed, IsGrounded, Attack, ComboIndex | ⭐⭐⭐ |

### 3.2 脚本示例

```csharp
public class RpgAnim : MonoBehaviour
{
    Animator anim;
    int comboStep;

    void Awake() => anim = GetComponent<Animator>();

    void Update()
    {
        anim.SetFloat("Speed", velocity.magnitude);
        anim.SetBool("IsGrounded", grounded);

        if (Input.GetMouseButtonDown(0) && CanAttack())
        {
            comboStep = (comboStep + 1) % 3;
            anim.SetInteger("ComboIndex", comboStep);
            anim.SetTrigger("Attack");
        }
    }

    bool CanAttack()
    {
        var info = anim.GetCurrentAnimatorStateInfo(0);
        return !(info.IsName("Attack") && info.normalizedTime < 0.85f);
    }
}
```

**[重难点]** 连击：`ComboIndex` + `GetCurrentAnimatorStateInfo.normalizedTime` 防止动画未播完就打断。

## 四、2D 游戏（⭐⭐）

### 4.1 资源与状态机

| 方式 | 说明 | 频率 |
|------|------|------|
| **Sprite 序列** | 切图 → Create Animation Clip | ⭐⭐⭐ |
| **2D 骨骼** | 2D Animation 包 → 骨骼 Clip | ⭐⭐ |
| **State** | Idle / Run / Jump / Attack | ⭐⭐⭐ |
| **Parameters** | Speed, IsGrounded | ⭐⭐⭐ |

### 4.2 脚本示例

```csharp
public class Player2DAnim : MonoBehaviour
{
    Animator anim;
    SpriteRenderer sr;
    Rigidbody2D rb;

    void Awake()
    {
        anim = GetComponent<Animator>();
        sr = GetComponent<SpriteRenderer>();
        rb = GetComponent<Rigidbody2D>();
    }

    void Update()
    {
        anim.SetFloat("Speed", Mathf.Abs(rb.velocity.x));
        anim.SetBool("IsGrounded", IsGrounded());

        if (rb.velocity.x != 0)
            sr.flipX = rb.velocity.x < 0;  // ⭐⭐ 朝向，不另做 Left/Right State
    }

    public void OnHitFrame() { /* Animation Event：攻击帧伤害 */ }
}
```

**[易错点]** flipX 后**碰撞体偏移**未调整 → 攻击判定错位；Event 帧与 Collider 对齐。

## 五、Animation vs Animator 选型（⭐⭐⭐）

| 系统 | 适用场景 | 典型例子 | 频率 |
|------|---------|---------|------|
| **Animator** | 角色控制、多状态、混合、技能切换 | 射击/RPG/2D 角色 | ⭐⭐⭐ |
| **Animation Legacy** | 环境动画、简单 Prop、镜头漫游 | 门、旗帜、光影、Camera | ⭐⭐ |
| **结合** | Animator 管角色 + Legacy 管场景 | 角色 + 环境 | ⭐⭐ |

**[教材精华·宣雨松/Official]** 角色多状态 → Animator；环境/门/镜头 → Legacy；见 Mecanim FAQ 与 01 文档选型表。

## 十、注意要点与最佳实践

| 易错点 | 说明 |
|--------|------|
| 只导模型不建 Controller | 角色 T-Pose 不动 |
| 脚本每帧 Play 绕过状态机 | 难维护，违背 Mecanim 规范 |
| 2D flipX 碰撞错位 | 调整 Collider 或统一朝右制作 |
| Layer Weight 忘记 Set | 上半身层不生效 |
| 环境动画上 Animator | 简单 Prop 用 Legacy 即可 |

### 最佳实践

- 三类游戏统一：**导入 → Controller → Set 参数**
- 射击：Layer 分上下半身
- RPG：BlendTree Locomotion + SubSM Combat
- 2D：flipX 代替左右两套 State
