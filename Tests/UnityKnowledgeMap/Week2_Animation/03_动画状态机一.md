## 思维导图总览

```
动画状态机（一）
│
├── 【第一梯队】三要素 ★★★
│   ├── State（状态）────────── 某一时刻的行为，绑定 Motion/Clip
│   ├── Transition（过渡）────── State A → B 的路径 + Conditions
│   └── Parameters（参数）────── 脚本与状态机的接口（Float/Bool/Trigger…）
│
├── 【第二梯队】状态切换 ★★★
│   ├── Make Transition + Add Condition
│   ├── 单条件 / 多条件 AND / OR
│   ├── Has Exit Time（是否等当前 State 播完）
│   ├── Transition Duration（混合时长）
│   └── Fixed Duration vs Normalized
│
├── 【第三梯队】Any State ★★
│   ├── 从任意 State 过渡到指定 State
│   ├── 典型：Die / Stun / 全局 Reset
│   └── 不能作为 Transition 终点
│
├── 【第四梯队】子状态机 Sub-State Machine ★★
│   ├── 打包一组相关 State（如战斗模块）
│   ├── 对外像一个 State，内有 Entry/Exit
│   └── 参数与主 Controller 共享
│
├── 【第五梯队】动画层 Layer ★★★
│   ├── 多层并行播放
│   ├── Avatar Mask ─── 指定 Layer 影响哪些骨骼
│   ├── Layer Weight ─── 0~1 影响程度
│   └── Override / Additive 混合模式
│
├── 【调试】Solo / Mute ★
│   └── 编辑器中临时启用/禁用某条 Transition
│
└── 【注意要点】
    ├── 响应输入：Has Exit Time 关 + Trigger
    ├── 上下半身：Layer0 全身 + Layer1 Mask 上半身
    └── 脚本只 Set 参数，不选 Clip
```

## 一、状态机运行逻辑（官方标准 · 必背）

```
每帧：
  读当前 State
    → 检查从当前 State 出发的所有 Transition 的 Conditions
    → 满足则按 Duration 混合过渡到目标 State
    → 播放目标 State 的 Motion（Clip 或 BlendTree）
```

**[重难点]** 脚本职责是 **Set 参数**；**何时切换、切到哪个 State** 由 Controller 编辑器配置。不在代码里 `if (running) anim.Play("Run")`。

## 二、State 与 Transition（⭐⭐⭐）

### 2.1 创建切换

| 步骤 | 操作 | 频率 |
|:---:|------|------|
| 1 | Animator 窗口 → Create State → 分配 Clip | ⭐⭐⭐ |
| 2 | 右键起始 State → **Make Transition** → 点目标 State | ⭐⭐⭐ |
| 3 | 选中 Transition → **Add Condition** | ⭐⭐⭐ |
| 4 | 选 Parameter + 比较（Greater/Less/Equals/Trigger） | ⭐⭐⭐ |
| 5 | 右键窗口 → Save / Ctrl+S | ⭐⭐ |

```csharp
anim.SetFloat("Speed", speed);  // 驱动 Condition: Speed > 0.1
```

### 2.2 Condition 类型

| Parameter 类型 | 可用比较 | 示例 |
|----------------|---------|------|
| Float | Greater, Less, Equals | `Speed > 0.1` |
| Int | Equals, NotEqual, Greater, Less | `WeaponType == 2` |
| Bool | true / false | `IsGrounded` |
| Trigger | 自动触发一次 | `Jump` |

### 2.3 多条件

| 类型 | 说明 | 示例 |
|------|------|------|
| **AND** | Transition 上所有 Condition 都满足 | `Speed > 0.1 && IsGrounded` |
| **OR** | 同一 Transition 内多条 Condition 为 OR（Unity 规则：列表内为 AND，不同 Transition 为 OR 来源） | 多条 Transition 各带不同 Condition |

**[易错点]** 同一 State 出发多条 Transition 条件可能**同时满足** → 需设计互斥条件或优先级（Transition Order / 条件细分）。

---

## 三、Has Exit Time（⭐⭐⭐）

| 设置 | 行为 | 频率 |
|------|------|------|
| **不勾选** | 条件满足 → Transition **立即**开始 | ⭐⭐⭐ |
| **勾选** | 当前 State 播放到 Exit Time 百分比后才过渡 | ⭐⭐⭐ |

| 应用 | Has Exit Time | 原因 |
|------|---------------|------|
| Jump / 受击 / 响应输入 | **关** | 需要即时响应 |
| 攻击后摇 / 死亡播完 | **开** | 需要动作播完 |

**[重难点] Jump 误勾 Has Exit Time → 按键后跳跃延迟半拍，手感极差。**

**[教材精华·宣雨松/吴亚峰]** 玩家输入驱动动作：**Has Exit Time = false**；终结/过场类才考虑开启。

---

## 四、Transition 其他属性（⭐⭐）

| 属性 | 说明 |
|------|------|
| **Transition Duration** | 混合时长（秒或归一化） |
| **Transition Offset** | 目标 State 从哪个时间点开始 |
| **Interruption Source** | 过渡过程中是否可被其他 Transition 打断 |
| **Ordered Interruption** | 打断优先级 |

---

## 五、Any State（⭐⭐）

| 项目 | 说明 |
|------|------|
| **作用** | 从**任意** State 过渡到指定 State |
| **典型** | Die、Stun、Hit、全局 Reset |
| **限制** | **不能**作为 Transition 的**终点** |
| **API** | `anim.SetTrigger("Die")` + AnyState→Die |

```csharp
// 任意状态可切到死亡
anim.SetTrigger("Die");
```

**[易错点]** AnyState 连到过多 State → 过渡线爆炸，难维护；应只用于**真正全局**的少数动作。

---

## 六、Solo / Mute（⭐）

| 功能 | 说明 |
|------|------|
| **Solo** | 启用该 Transition，同来源其他视为 Mute |
| **Mute** | 禁用该 Transition |
| **用途** | **编辑器调试**切换冲突，非运行时 API |

## 七、子状态机 Sub-State Machine（⭐⭐）

### 7.1 概念

```
主状态机：Idle / Locomotion / Combat / Air …
    └── Sub-State Machine「Combat」：
            Attack / Skill / Hit / Block / Die
        （战斗模块独立一套 State，对外像一个 State）
```

| 特点 | 说明 |
|------|------|
| 对外 | 像一个 State，有 Entry / Exit 与主层连接 |
| 参数 | 与主 Controller **共享**，脚本 Set 参数写法不变 |
| 复用 | 多角色可共用含 SubSM 的 Controller |

### 7.2 创建

| 操作 | 说明 |
|------|------|
| 右键 → Create Sub-State Machine | 创建子状态机 |
| 双击进入 | 在内部 Create State / Transition |
| Entry / Exit | 与主层 State 连接 |

---

## 八、动画层 Layer（⭐⭐⭐）

### 8.1 Layer 作用

| 作用 | 说明 |
|------|------|
| 并行播放 | Layer0 全身走 + Layer1 上半身瞄准 |
| 降低复杂度 | 拆分 Locomotion 与 UpperBody |
| 独立权重 | SetLayerWeight 动态开关某层 |

### 8.2 创建与 Mask

| 步骤 | 操作 | 频率 |
|:---:|------|------|
| 1 | Layers → **Add Layer** | ⭐⭐⭐ |
| 2 | Create → **Avatar Mask** | ⭐⭐⭐ |
| 3 | Mask 勾选生效骨骼（Spine 以上 = 上半身） | ⭐⭐⭐ |
| 4 | Layer 设置 Mask | ⭐⭐⭐ |

### 8.3 Layer Weight 与 Blending

| 项目 | 说明 | 频率 |
|------|------|------|
| **Weight** | 0~1，该层影响程度 | ⭐⭐⭐ |
| **Override** | 覆盖下层同名骨骼 | ⭐⭐ |
| **Additive** | 叠加姿态 | ⭐ |

```csharp
anim.SetLayerWeight(1, isAiming ? 1f : 0f);
```

**[重难点] 射击/TPS 经典结构**

```
Layer 0（Weight=1，无 Mask）：Idle / Walk / Run — Locomotion
Layer 1（Weight=0~1，Mask 上半身）：Aim / Fire / Reload
```

**[易错点]**
- Layer Weight 始终 0 → 该层完全不生效
- 未绑 Mask → Layer1 覆盖全身，与 Layer0 冲突

## 九、核心 API 速查（⭐⭐⭐）

| API | 用途 | 频率 |
|-----|------|------|
| `SetFloat/SetBool/SetInteger/SetTrigger` | 驱动 Transition Condition | ⭐⭐⭐ |
| `SetLayerWeight(layerIndex, weight)` | 控制层权重 | ⭐⭐⭐ |
| `GetCurrentAnimatorStateInfo(layer)` | 读当前 State | ⭐⭐⭐ |
| `info.normalizedTime` | 判断攻击是否播完 | ⭐⭐⭐ |
| `info.IsName("Attack")` | 是否处于某 State | ⭐⭐⭐ |
| `ResetTrigger("Attack")` | 重置 Trigger | ⭐⭐ |

```csharp
void TryAttack()
{
    var info = anim.GetCurrentAnimatorStateInfo(0);
    if (info.IsName("Attack") && info.normalizedTime < 0.8f)
        return;  // 攻击未播完，不允许再次攻击
    anim.SetTrigger("Attack");
}
```

## 十、注意要点与最佳实践

| 易错点 | 说明 |
|--------|------|
| 条件满足不切换 | Trigger 未 Reset / Has Exit Time / 参数类型错 |
| Jump 延迟 | Has Exit Time 误开 |
| Layer 不生效 | Weight=0 / 未绑 Mask |
| 多条 Transition 抢切 | 条件未互斥 |
| AnyState 滥用 | 状态图难以维护 |

### 最佳实践

- 输入响应：Trigger + Has Exit Time **关**
- 需播完：Has Exit Time **开** + normalizedTime 检查
- 上下半身：Layer0 + Layer1 + Avatar Mask
- 复杂模块：Sub-State Machine 打包

## 十四、教材 vs 官方 · 对照速查

| 知识点 | 教材常见写法 | 官方/工程推荐 |
